We maintain the [ImFusion Public Demos Repository](https://github.com/ImFusionGmbH/public-demos) on Github.

It contains a couple of small example projects demonstrating how to integrate the ImFusion SDK into custom applications and how to extend the SDK with new functionality, for instance:

- [AnnotationHandle](https://github.com/ImFusionGmbH/public-demos/tree/release/AnnotationHandle): Explains how to add simple interactive handles to GlPointBasedAnnotation via a plugin.
- [BrushStandaloneApplication](https://github.com/ImFusionGmbH/public-demos/tree/release/BrushStandaloneApplication): Builds a standalone application with an interactive Brush tool for labeling medical images.
- [Example2D3DRegistration](https://github.com/ImFusionGmbH/public-demos/tree/release/Example2D3DRegistration): Demonstrates how to use and customize the x-ray 2D/3D registration algorithm in an example plugin.
- [ExampleAnatomyPlugin](https://github.com/ImFusionGmbH/public-demos/tree/release/ExampleAnatomyPlugin): Shows how you can use the AnatomyPlugin to represent AnatomicalStructures and combine them in a AnatomicalStructureCollection.
- [ExampleDicomBrowser](https://github.com/ImFusionGmbH/public-demos/tree/release/ExampleDicomBrowser): Explains how to build a simple DICOM browser using the ImFusionDicom plugin.
- [ExampleDicomExtension](https://github.com/ImFusionGmbH/public-demos/tree/release/ExampleDicomExtension): Implements a sample DicomExtension to read and write custom DICOM tags.
- [ExampleImageMath](https://github.com/ImFusionGmbH/public-demos/tree/release/ExampleImageMath): Contains basic examples on how to use our ImageMath plugin.
- [ExampleITK](https://github.com/ImFusionGmbH/public-demos/tree/release/ExampleITK): Shows how to use ITK with the ImFusion SDK.
- [ExampleMachineLearningInference](https://github.com/ImFusionGmbH/public-demos/tree/release/ExampleMachineLearningInference): Explains how to integrate a deep learning model, trained independently using Python, into the ImFusion framework.
- [ExampleMainWindowBaseApplication](https://github.com/ImFusionGmbH/public-demos/tree/release/ExampleMainWindowBaseApplication): Shows how to build a simple standalone application for displaying medical DICOM data using the ImFusion SDK.
- [ExampleOpenCV](https://github.com/ImFusionGmbH/public-demos/tree/release/ExampleOpenCV): Demonstrates integration with OpenCV for computer vision tasks.
- [ExampleOpenGL](https://github.com/ImFusionGmbH/public-demos/tree/release/ExampleOpenGL): Shows how to use OpenGL for rendering and image processing.
- [ExamplePlugin](https://github.com/ImFusionGmbH/public-demos/tree/release/ExamplePlugin): Provides a template for creating new plugins.
- [ExampleRGBDReconstruction](https://github.com/ImFusionGmbH/public-demos/tree/release/ExampleRGBDReconstruction): Demonstrates how to use the RGBD plugin for 3D reconstruction from RGB-D data.
- [ExampleStandaloneApplication](https://github.com/ImFusionGmbH/public-demos/tree/release/ExampleStandaloneApplication): Builds a standalone application using the SDK.
- [QMLRendererDemo](https://github.com/ImFusionGmbH/public-demos/tree/release/QMLRendererDemo): Explains how to natively integrate the ImFusionLib rendering stack into a QML application.
- [SlicerExtension](https://github.com/ImFusionGmbH/public-demos/tree/release/SlicerExtension): Basic example demonstrating how the ImFusion SDK can be integrated into 3D-Slicer via an Extension module..
- [StreamExamples](https://github.com/ImFusionGmbH/public-demos/tree/release/StreamExamples): Shows how to create Streams and to work with their data.
- [TotalSegmentatorAnatomyPlugin](https://github.com/ImFusionGmbH/public-demos/tree/release/TotalSegmentatorAnatomyPlugin): Shows how to integrate TotalSegmentator in a custom plugin and use it to create ImFusion anatomical structure descriptions.
- [TractographyPlugin](https://github.com/ImFusionGmbH/public-demos/tree/release/TractographyPlugin): Demonstrates how to create a custom data type (fiber tracts, used for instance in Diffusion Tensor Imaging) in the ImFusion SDK and integrate it into the visualization stack using the GlObject and DataDisplayHandler interfaces.
