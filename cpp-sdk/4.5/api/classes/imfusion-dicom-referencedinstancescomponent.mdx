## Overview

[DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) to store DICOM instances that are referenced by the dataset.

A DICOM dataset can reference a number of other DICOM datasets that are somehow related. The references in this component are determined by the ReferencedSeriesSequence.

This component is currently only created for DICOM Segmentation and RTStruct SOPs.

*override, virtual*

Returns a unique string identifier for this type of data component.

**Note:** To avoid conflicts make sure to include potential plugin names into this ID. Due to the modular architecture of the ImFusion library, uniqueness of IDs cannot be checked at compile time. However, [DataComponentFactory](/cpp-sdk/4.5/api/classes/imfusion-datacomponentfactory) will check the uniqueness of the ID during registration.

### isReferencing

```cpp
bool ImFusion::Dicom::ReferencedInstancesComponent::isReferencing (const SourceInfoComponent & sic) const
```

Returns true if the instances of the given [SourceInfoComponent](/cpp-sdk/4.5/api/classes/imfusion-dicom-sourceinfocomponent) are referenced by this component.

The instances and references have to only intersect for this to return true. This way, e.g. a segmentation would be considered referencing a CT if it only overlaps in a view slices.
