`#include <ImFusion/GUI/MainWindowBase.h>`

## Overview

Extends [ApplicationController](/cpp-sdk/4.5/api/classes/imfusion-applicationcontroller) with additional interfaces for graphical user interfaces.

Depending on the constructor arguments, instantiating this class can also take care of proper initialization (e.g. factories, OpenGL contexts, etc.) of the ImFusion SDK. In this case you do not need to call [Framework::init()](/cpp-sdk/4.5/api/namespaces/imfusion-framework) explicitly.

## Member Enumeration Documentation

### PostLoadAction

```cpp
enum class ImFusion::MainWindowBase::PostLoadAction
```

*strong*

Bitflag enumeration of available actions that the default implementation of [postLoad()](#postload) performs.

**Enumerator:**

- `None` — Don't perform any actions in [postLoad()](#postload).
- `SelectData` — Change the selection of the [DataWidget](/cpp-sdk/4.5/api/classes/imfusion-datawidget) to the newly added [Data](/cpp-sdk/4.5/api/classes/imfusion-data).
- `ShowViews` — Make views that can show the added [Data](/cpp-sdk/4.5/api/classes/imfusion-data) visible in case they are not.
- `StoreOriginalMatrix` — Store the [Data](/cpp-sdk/4.5/api/classes/imfusion-data)'s original pose matrix in its [TransformationStashDataComponent](/cpp-sdk/4.5/api/classes/imfusion-transformationstashdatacomponent).
- `CreateDataController` — Create and show a [DataController](/cpp-sdk/4.5/api/classes/imfusion-datacontroller) for the added [Data](/cpp-sdk/4.5/api/classes/imfusion-data) if possible.
- `All` — Perform all actions in [postLoad()](#postload).

```cpp

{
 // ...

    QString readCustomStyle()

        QFile stylesheet(":/style/customStyle.qss");
 if (stylesheet.open(QIODevice::ReadOnly))
 return stylesheet.readAll();
 else
 return QString();
    }

 void loadStyleSheet() override
 {
 MainWindowBase::loadStyleSheet(*qApp, MiniNavSystem::readCustomStyle());
    }
}
```

### eventFilter

```cpp
bool ImFusion::MainWindowBase::eventFilter (QObject * target, QEvent * event)
```

*override, protected*

[Filters](/cpp-sdk/4.5/api/namespaces/imfusion-filters) close events of algorithm controllers and data controllers.

On close event for a data controller, the controller is removed and deleted. For algorithm controllers which are QWidget the controller and its algorithm is removed and deleted. For controllers which are QDialog, reject() slot will be raised. They must handle deleting themselves and their algorithms either by overriding the reject() slot or in their run() method considering that the default reject slot() causes the exec() method to return 0. [AlgorithmController::closeAlgorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithmcontroller) method can be called to remove the algorithm and its controller.
