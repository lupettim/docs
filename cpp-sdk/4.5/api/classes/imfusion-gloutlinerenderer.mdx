`#include <ImFusion/GL/GlOutlineRenderer.h>`

## Overview

Light-weight wrapper to render a colored outline around any rendered [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject).

Instantiate this class with any [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) you want to render the outline of and then add this instance instead of the wrapped [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) to the view. Rendering the wrapped object and rendering the contour is performed independently, thus you can setup any combination of showing object and outline.

**Note:** [GlOutlineRenderer](/cpp-sdk/4.5/api/classes/imfusion-gloutlinerenderer) will only work with GlObjects that render directly to to framebuffer. It will not work if the [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) uses OIT to render semi-transparent fragments.

## Member Enumeration Documentation

### RenderMode

```cpp
enum class ImFusion::GlOutlineRenderer::RenderMode
```

*strong*

Enumeration for specifying what parts to show.

**Enumerator:**

- `ObjectOnly` — Only render object, not the outline.
- `OutlineOnly` — Only render outline, not the object.
- `Both` — Render both object and outline.

## Constructor & Destructor Documentation

### GlOutlineRenderer

```cpp
ImFusion::GlOutlineRenderer::GlOutlineRenderer (const std::vector < GlMesh * > & meshes)
```

Constructor for multiple GlMeshes.

If the [GlOutlineRenderer](/cpp-sdk/4.5/api/classes/imfusion-gloutlinerenderer) is created for multiple GlMeshes, the outline for each mesh will be computed from the parts which pass the depth test only. The order is considered in the following way. Assuming that two meshes are passed upon construction, then the outline of meshes[1] will override mesh[0].
