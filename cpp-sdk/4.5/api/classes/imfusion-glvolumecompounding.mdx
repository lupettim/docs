`#include <ImFusion/GL/GlVolumeCompounding.h>`

## Overview

Compounding of multiple volumes (SharedImageSets) with different modalities for the computation of the resulting voxel values ([Mode](#mode)): Mean, Median, Maximum, and Distance.

**Note:** While the first three modes are self-descriptive, the DISTANCE mode computes distance-maps of the input volumes and employs them to blend the resulting volume. This mode allows to reduce artifacts at the edges. The properties of the resulting volume are taken from the given reference volume (setReferenceVolume()) or inferred from the input volumes.

## Member Enumeration Documentation

### Mode

```cpp
enum ImFusion::GlVolumeCompounding::Mode
```

Compounding mode, i.e. how the resulting voxel values is computed.

**Enumerator:**

- `MEAN` — Compute mean.
- `MEDIAN` — Compute median (in case of even number of samples, will use the larger center one).
- `MAXIMUM` — Compute maximum.
- `DISTANCE` — Compute distance to border.

### WeightedMeanMode

```cpp
enum ImFusion::GlVolumeCompounding::WeightedMeanMode
```

The mode for weighted Mean.

**Enumerator:**

- `RADIAL` — Compute the weight by the distance to the center.
- `ALONGZ` — Compute the weight by the distance to the center in z direction.
- `COSINE` — Compute the weight by a smooth cosine along the border covering any overlap.

### InterpolationMode

```cpp
enum class ImFusion::GlVolumeCompounding::InterpolationMode
```

*strong*

Interpolation mode for texture lookup.

**Enumerator:**

- `Nearest` — Nearest-neighbor interpolation, useful for label maps.
- `Linear` — Use bi-/tri-linear interpolation.
