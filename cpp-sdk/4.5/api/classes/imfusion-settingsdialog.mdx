`#include <ImFusion/GUI/SettingsDialog.h>`

## Overview

Presents a [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui) dialog to configure the global settings.

This class inherently handles the settings in the ImFusion::Settings class, as well as [ViewInteraction](/cpp-sdk/4.5/api/classes/imfusion-viewinteraction) settings. You can configure this to also handle custom settings using the [addCustomSettings()](#addcustomsettings-1-2) functions. By default these will be presented with an auto-generated [PropertiesWidget](/cpp-sdk/4.5/api/classes/imfusion-propertieswidget), but you may also register a custom widget with [addCustomSettingsWidget()](#addcustomsettingswidget).

## Member Function Documentation

### addCustomSettings

```cpp
template<typename T> void ImFusion::SettingsDialog::addCustomSettings ()
```

*inline, static*

Add a [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) type *T* to the settings dialog.

**Note:** Only the parameters returned by T.configuration() within this call will be displayed by the [SettingsDialog](/cpp-sdk/4.5/api/classes/imfusion-settingsdialog).

**Note:** T.configuration() can set the "label" attribute to set the displayed name of the settings.

### addCustomSettings

```cpp
void ImFusion::SettingsDialog::addCustomSettings (std::unique_ptr < Properties > p)
```

*static*

Add a custom settings object to the dialog.

\warn p-&gt;name() is used as the name of the settings, as such you must take care that the other accesses to the settings are consistent. If you're not sure of this, you probably need the other overload.

**Note:** Only the parameters within *p* will be displayed by the [SettingsDialog](/cpp-sdk/4.5/api/classes/imfusion-settingsdialog) for the settings under p-&gt;name().

**Note:** Use the "label" attribute to set the displayed name of the settings.

Add a custom settings widget *W* for configurable type *T*.
