## Overview

The basic [ApplicationController](/cpp-sdk/4.5/api/classes/imfusion-applicationcontroller) provides essential interfaces of the SDK but no graphical user interface.

Depending on the constructor arguments, instantiating this class can also take care of proper initialization (e.g. factories, OpenGL contexts, etc.) of the ImFusion SDK. In this case you do not need to call [Framework::init()](/cpp-sdk/4.5/api/namespaces/imfusion-framework) explicitly.

While it is technically possible to have multiple [ApplicationController](/cpp-sdk/4.5/api/classes/imfusion-applicationcontroller) instances at the same time, such setups are discouraged and may be considered undefined/unsupported behavior.

**See also:** [What is the correct way to initialize the ImFusionLib and its plugins?](/cpp-sdk/4.5/api/guides/f-a-q)

### loadPlugins

```cpp
void ImFusion::ApplicationController::loadPlugins (const std::vector < std::string > & pluginFolders = std::vector < std::string >())
```

Load and register/initializes all ImFusion plugins from the specified folders.

If *pluginFolders* is empty, the following default search paths are used:

Windows platforms:

- IMFUSION_PLUGIN_PATH environment variable
- &lt;executable-path&gt;/plugins
- &lt;ImFusionLib.dll-path&gt;/plugins

Unix platforms:

- IMFUSION_PLUGIN_PATH environment variable
- &lt;ImFusionLib.so-path&gt;/../lib/ImFusionLib/plugins

Plugin names defined in the environment variable IMFUSION_PLUGIN_BLACKLIST as a semi-colon-separated list (e.g. "Foo.dll;Bar.dll") will not be loaded by this function. This mechanism can also be used to explicitly ignore libraries that might conflict with or crash the application.

****Deprecated**:** "Use PluginManager::registerPlugins() and PluginManager::initAllRegisteredPlugins() instead"

Will additionally call ImFusionPlugin::setApplicationController() on every loaded plugin.
