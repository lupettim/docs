## Overview

The [DicomWriter](/cpp-sdk/4.5/api/classes/imfusion-dicomwriter) class can store a [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) in a DICOM dataset.

The writer creates DicomIODs and DcmDatasets that correspond to the [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset). For classical single frame DICOM files, an [IOD](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod) and DcmDataset will be generated per 2D frame in the image. For enhanced multi-frame DICOM files, a single [IOD](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod) and DcmDataset is generated.

The [DicomWriter](/cpp-sdk/4.5/api/classes/imfusion-dicomwriter) will automatically pick the most appropriate [Dicom::IOD](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod) instance upon construction using the [Dicom::IOD_Registry](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod-registry). The [Dicom::IOD](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod) instance can also be changed later on using the resetIOD method.

## Member Enumeration Documentation

### Type

```cpp
enum class ImFusion::DicomWriter::Type
```

*strong*

[Type](#type) of DICOM storage to use.

**Enumerator:**

- `File` — If possible, write all image data to a single file.
- `Folder` — Write each frame as separate DICOM file to a folder.
- `Auto` — Automatically determine storage type based on image content.

### createDatasets

```cpp
void ImFusion::DicomWriter::createDatasets ()
```

Creates the actual DcmDataset instances and fills them with tags according to the image.

This might allocate a lot of memory depending of the size of the image. Calling this function again will replace the current dataset.

### datasets

List of DcmDatasets and corresponding IODs for each saved file.

This list will be empty until createDatasets was called.

### save

```cpp
bool ImFusion::DicomWriter::save (const Url & uri)
```

Saves the DICOM dataset to the given URI, which must have a scheme of either "file" or "pacs".

Automatically writes pixel data and applies compression. All intermediate directories of the given path are created automatically.

**Note:** If the URI is of scheme "file" and the [DicomWriter](/cpp-sdk/4.5/api/classes/imfusion-dicomwriter) is multi-file, a folder is expected, otherwise a file.

### save

```cpp

```

Convenience method to save the DICOM dataset to the file system without explicitly constructing an URI.

Just pass a normal file/folder path.

### setCompression

```cpp
void ImFusion::DicomWriter::setCompression (DicomWriter::Compression compression)
```

Sets the compression used when saving.

The default is LittleEndianImplicit (uncompressed). The compression will only be used during saving.

### setCompressionQuality

```cpp
void ImFusion::DicomWriter::setCompressionQuality (int quality)
```

Sets the quality setting of lossy compressions.

The quality goes from 1 to 100, with 1 being the lowest quality and highest compression and 100 being the highest quality and the lowest compression. Values will be clamped to 1 and 100. Has no effect on lossless or no compression.

### setFilenameFormat

```cpp
void ImFusion::DicomWriter::setFilenameFormat (const std::string & filenameFormat)
```

Sets the output filename format.

Syntax: prefix$0..0$suffix. E.g. dicom_$0000$.dcm will set m_maxdigits to 4 and the dicom files will be named dicom_0000.dcm, dicom_0001.dcm etc.
