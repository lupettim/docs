## Overview

template&lt;typename T&gt;

class ImFusion::Parameter&lt; T &gt;
The [Parameter](/cpp-sdk/4.5/api/classes/imfusion-parameter) class represents a single parameter of a [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) entity.

It is intended to be used as member of [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) classes and enables you to reduce the amount of boilerplate code needed to implement the pair of [configure()](#configure)/configuration() functions for each individual class member that is supposed to be serialized. In order for this to work each [Parameter](/cpp-sdk/4.5/api/classes/imfusion-parameter) must be [registered](/cpp-sdk/4.5/api/classes/imfusion-configurable) with the parent [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) instance. This can be automatized by passing it as additional argument to the corresponding constructor overload. Registration is preserved across copies/moves/assignments, and removed on destruction.

Parameters wrap a regular type T and make it serializable to/from the [Properties](/cpp-sdk/4.5/api/classes/imfusion-properties) interface. Since it provides a cast- and assignment operator, it can be used like a variable of type T directly:

```cpp
Parameter<int> count("count", 0);
int other = count;
count = 5;
```
 fragment
Any getter access is restricted to const access. To change the parameter you need to explicitly set it. This ensures the [signalValueChanged](/cpp-sdk/4.5/api/classes/imfusion-parameterbase) to be emitted whenever the param changes.

By convention, public [Parameter](/cpp-sdk/4.5/api/classes/imfusion-parameter) members use the p_ prefix for the variable name (instead of the common m_ prefix) in order to make their semantics easier to identify in code.

**Template Parameters:** | T | Underlying type of the parameter, must be supported by [Properties::setParam()](/cpp-sdk/4.5/api/classes/imfusion-properties). |
