`#include <ImFusion/GUI/DisplayWidgetMulti.h>`

## Overview

Specialized version of [DisplayWidget](/cpp-sdk/4.5/api/classes/imfusion-displaywidget) that manages specific views and provides an automatic layout.

Other than [DisplayWidget](/cpp-sdk/4.5/api/classes/imfusion-displaywidget), [DisplayWidgetMulti](/cpp-sdk/4.5/api/classes/imfusion-displaywidgetmulti) knows the specific properties of each view (e.g. 2D or 3D). It provides several convenience methods to add specific views with preset properties and overlays. Views are automatically separated into into 4 categories: 2D, 3D, MPR or [Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm).

[Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm) views are owned by [DisplayWidgetMulti](/cpp-sdk/4.5/api/classes/imfusion-displaywidgetmulti) and will be considered during layouting, however they are not considered by other functionality (e.g. showing [Data](/cpp-sdk/4.5/api/classes/imfusion-data)) and are alto not modified otherwise by this class. As the name suggests they are intended to be fully maintained by an Algorithm(Controller) and show algorithm-specific data, such as plots or debugging information.

## Member Typedef Documentation

### LayoutFunc

```cpp
using ImFusion::DisplayWidgetMulti::LayoutFunc = std::function <DisplayLayoutConfig(const std::vector < InteractiveView *>& visibleViews)>
```

Returns a layout configuration containing the visibleViews.

A well behaved [LayoutFunc](#layoutfunc) will only contain the visibleViews in the final configuration. It is also valid to ignore this list and return a layout containing other views of the [DisplayWidget](/cpp-sdk/4.5/api/classes/imfusion-displaywidget). This way a controller could enforce a specific fixed layout for a certain workflow. However, this should only be used for specific use cases because the user experience may degrade otherwise (e.g. user cannot hide certain views anymore). If certain view types are not supported at all by the [LayoutFunc](#layoutfunc), the [LayoutFeatures](#layoutfeatures) should be set accordingly when adding the layout.

**See also:** [addCustomLayout](#addcustomlayout)
