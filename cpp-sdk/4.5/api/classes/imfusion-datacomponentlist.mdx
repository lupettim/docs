## Overview

Class for holding a collection of DataComponents.

DataComponents are uniquely identified by their type and [DataComponentList](/cpp-sdk/4.5/api/classes/imfusion-datacomponentlist) will hold only a single instance of a specific [DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) type.

DataComponents inheriting from [LazyInstantiatedDataComponent](/cpp-sdk/4.5/api/classes/imfusion-lazyinstantiateddatacomponent) can never be deleted from a [DataComponentList](/cpp-sdk/4.5/api/classes/imfusion-datacomponentlist) due to their invariant that references remain valid over the lifetime of the parent [Data](/cpp-sdk/4.5/api/classes/imfusion-data). Therefore, the [assign()](#assign) and erase() may behave unexpected as they will not remove existing DataComponents of such types.

Example:

```cpp
{
 // store information at one place
 SharedImageSet* image = ...;

 // alternative 1:
 auto adc = std::make_unique<AnatomyDataComponent>();
    adc->add(Anatomy::ABDOMEN);
    image->components().add(std::move(adc));

 // alternative 2:
    image->components().getOrCreate<AnatomyDataComponent>().add(Anatomy::ABDOMEN);
}

{
 // retrieve information at some other place
 SharedImageSet* image = ...;
 if (AnatomyDataComponent* adc = image->components().get<AnatomyDataComponent>()) {
 if (adc->contains(Anatomy::ABDOMEN))
            ...
    }
}
```

### assign

```cpp
void ImFusion::DataComponentList::assign (const DataComponentList & rhs)
```

Copy assignment, removing all existing components where possible and cloning all DataComponents held by rhs.

**Note:** Due to certain DataComponent-invariants this list is not necessarily the same as *rhs* after assignment.

### add

```cpp
template<typename T> T * ImFusion::DataComponentList::add (std::unique_ptr < T > component)
```

Adds the given [DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) to the collection.

**Note:** If the list already contains a [DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) of the exact same type, its configuration will be overwritten with the one in *component*.

**Note:** Do not store a pointer to *component* after calling this method since it may be deleted. Use the returned pointer in case you want to keep a reference to the added [DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) (this may be a different instance)!

### add

```cpp

```

Adds all DataComponents in *other* to this one.

Any given [DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) of the exact same type will be overwritten.

Swaps the data component lists.
