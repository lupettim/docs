## Overview

Wraps different mesh processing functionality into a single algorithm.

See the Mesh::Processing namespace for direct access to the different processing options.

## Member Enumeration Documentation

### Mode

```cpp
enum ImFusion::MeshProcessingAlgorithm::Mode
```

Computation mode.

**Enumerator:**

- `NONE` — no mode set
- `SMOOTH` — smooth mesh
- `DECIMATE` — decimate mesh
- `FILL_HOLES` — fill holes in a mesh
- `COLOR` — change color of the mesh
- `CROP` — crop the mesh
- `CLEAN` — clean mesh
- `AUTO_ALIGN` — auto-align mesh
- `REMOVE_EDGES` — remove boundary layers of the mesh
- `INVERT_NORMALS` — invert the normals
- `MERGE_CLOSE_VERTICES` — merge close vertices

### StatusExt

```cpp
enum class ImFusion::MeshProcessingAlgorithm::StatusExt
```

*strong*

Enumeration listing possible errors during the computation.

**Enumerator:**

- `FillHolesNotSupported` — Poisson reconstruction is not supported in this version of the software.
- `ColoringFailed` — Input mesh does not have a property of vertex colors.
- `ComputationFailed` — Something went wrong during computation.

### takeOutput

```cpp
OwningDataList ImFusion::MeshProcessingAlgorithm::takeOutput ()
```

*override, virtual*

Return any new [Data](/cpp-sdk/4.5/api/classes/imfusion-data) that was created by the [Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm) during the last call to [compute()](#compute).

The default implementation will return an empty list.

**Note:** Since ownership of the data is transferred, you can call this method only once between calls to [compute()](#compute).
