## Overview

[Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm) to split overlapping labels into separate instances It is possible for a label to (admittedly, this is somewhat rare, the main setting this happens is when loading dicom seg files) contain an encoding that specifies how individual pixel values should be assigned to classes.

This allows for labels to overlap. This algorithm converts such an overlapping label into a set of individual (binary) labels, one for each instance.

**See also:** [LabelOverlapEncoder](/cpp-sdk/4.5/api/classes/imfusion-labeloverlapencoder)
