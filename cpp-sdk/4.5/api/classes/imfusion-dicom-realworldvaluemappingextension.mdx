### load

```cpp
std::vector < std::unique_ptr < Data > > ImFusion::Dicom::RealWorldValueMappingExtension::load (const FrameDescriptor & frame, SharedImageSet & sis) const
```

*override, virtual*

Loads the Real-World-Value-Mapping and creates a [RealWorldMappingDataComponent](/cpp-sdk/4.5/api/classes/imfusion-realworldmappingdatacomponent) accordingly.

Since DICOM defines this mapping on the unmodified storage values, make sure that this method is not called after the storage values have been changed (e.g. through [MemImage::prepare](/cpp-sdk/4.5/api/classes/imfusion-memimage)). For this reason, Real-World-Value-Mapping is not supported with non-linear modality LUTs.

### save

```cpp
void ImFusion::Dicom::RealWorldValueMappingExtension::save (const FrameDescriptor & frame, const SharedImageSet & sis) const
```

*inline, override, virtual*

Save data from the given [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) to DICOM.

This method will be called once for each frame of the [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset). Use [FrameDescriptor::iod](/cpp-sdk/4.5/api/structs/imfusion-dicom-framedescriptor) to access the DICOM tags and [FrameDescriptor::sharedImageIndex](/cpp-sdk/4.5/api/structs/imfusion-dicom-framedescriptor)/sliceIndex to figure out which frame data should be saved. The matrix parameters of frame should be ignored. Use the matrix of the [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) instead.

### showInWriterUI

```cpp
bool ImFusion::Dicom::RealWorldValueMappingExtension::showInWriterUI () const
```

*inline, override, virtual*

Whether this extension will show up in the UI and can be enabled/disabled by the user.

The save method will be called independently of this value. This is useful if the extension only implements load.
