## Overview

[IoAlgorithm](/cpp-sdk/4.5/api/classes/imfusion-ioalgorithm) for reading and writing DICOM files.

The specified location can be either a file/folder path or a URI of the following syntax:

- *Local file system:*file://path/to/file/or/folder
- *PACS instance:* pacs://&lt;Hostname&gt;:&lt;Port&gt;/&lt;HostAE&gt;?&lt;seriesUID&gt;
- *DicomWeb instance:*[https://](https://)&lt;Hostname&gt;:&lt;Port&gt;/&lt;dicomweb-endpoint&gt;/studies/&lt;studyUID&gt;/series/&lt;seriesUID&gt;

Depending on the mode passed during construction, this algorithm operate in the following fashion:

- *IoMode::Read*: The algorithm will use the [DicomLoader](/cpp-sdk/4.5/api/classes/imfusion-dicomloader) class to read DICOM data from the specified location. Multiple input locations are supported by separating each item by '##'. Furthermore, an optional filter query ?&lt;seriesUID&gt; can be appended in order to only load a specific series from a folder. Thus, a valid input location has the following format: &lt;path&gt; [?&lt;seriesUID&gt;] [##&lt;path&gt; [?&lt;seriesUID&gt;] ]\* Alternatively, the query can also be a key-value pair with the following format: &lt;path&gt; [?series=&lt;seriesUID&gt;[&key=&lt;value&gt;]\* ] [##&lt;path&gt; [?series=&lt;seriesUID&gt;[&key=&lt;value&gt;]\* ] ]\* Only the keys 'series', 'study', 'ignoreNonDicom' and 'recursive' are recognised. All other keys are ignored. The optional 'ignoreNonDicom' key specifies whether files without a DICOM header are ignored (defaults to true). The optional 'recursive' key determines if a &lt;path&gt; to a folder should be loaded recursively.
- *IoMode::Write*: The algorithm will use the function defined in DicomExport.h to store the [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) passed during construction to the specified location. Depending on the image type it can be stored as set of DICOM frame or (if supported) as a single Enhanced DICOM file.

This class automatically uses the DicomPlugin::defaultAuthorizationProvider for the internal [Dicom::DatasetCache](/cpp-sdk/4.5/api/classes/imfusion-dicom-datasetcache). The provider can be changed (or removed).

### compressionQuality

```cpp
int ImFusion::DicomIoAlgorithm::compressionQuality () const
```

Sets the quality setting of lossy compressions.

The quality goes from 1 to 100, with 1 being the lowest quality and highest compression and 100 being the highest quality and the lowest compression. Values will be clamped to 1 and 100. Has no effect on lossless compressions.
