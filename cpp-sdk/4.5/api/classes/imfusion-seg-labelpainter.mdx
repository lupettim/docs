Explicitly sets a mask.

Masked pixels will not be painted to If no mask is set, but there is one attached to the image itself, the image mask will be used

Defines whether we filter the input before applying the stencil.

This can be beneficial in modalities like Ultrasound because of the high noise. Only matters if adaptiveness &gt; 0

### stencil

```cpp
SharedImage * ImFusion::Seg::LabelPainter::stencil ()
```

*inline*

The stencil image that will be drawn onto the label map by calling [paint()](#paint) This pointer may be null if [paint()](#paint) has not yet been called on the current image.

It's a single image of unsigned char data with the same size as the label map Values are only 1 or 0, where 1 indicates that the value will be modified by painting Pairs nicely with the [GlIsolineRenderer](/cpp-sdk/4.5/api/classes/imfusion-glisolinerenderer)

### paint

```cpp
bool ImFusion::Seg::LabelPainter::paint (vec3 from, vec3 to, bool erase = false, bool onlyPreview = false, std::optional < vec3 > planeNormal = {})
```

this signal is emitted after the pointer returned by [stencil()](#stencil) changes

Draws a capsule in the labelMap "from" and "to" are defined in world coordinates, and represent the two ends of the cylinder part of the capsule The focus of the inputImage is used for sampling, and the focus of the labelMap is used for drawing If erase is true, values will be set to 0 instead of "label" If onlyPreview is true, the stencil (see [stencil()](#stencil)) will be updated, but not the labelMap If planeNormal is defined and the image is 3D, only the points in the slice defined by the point 'from' and the normal 'planeNormal' will be drawn If you just want to draw a sphere, you can call this with from == to

Returns false if the capsule is definitely outside of the image's boundaries However returning true is not a guarantee that the labelMap was actually modified
