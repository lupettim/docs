## Overview

Abstract base class for OpenGL overlays.

In contrast to a [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject), a [GlOverlay](/cpp-sdk/4.5/api/classes/imfusion-gloverlay) is rendered on top of a [GlView](/cpp-sdk/4.5/api/classes/imfusion-glview) and reside in viewport space. Furthermore, you can not add a [GlOverlay](/cpp-sdk/4.5/api/classes/imfusion-gloverlay) directly to a [GlView](/cpp-sdk/4.5/api/classes/imfusion-glview). Instead it must always be wrapped in a [InteractiveOverlay](/cpp-sdk/4.5/api/classes/imfusion-interactiveoverlay) and added to an [InteractiveView](/cpp-sdk/4.5/api/classes/imfusion-interactiveview).

**See also:** InteravtiveOverlay

## Member Function Documentation

### setSizeHint

```cpp
virtual void ImFusion::GlOverlay::setSizeHint (int width, int height)
```

*virtual*

Sets the preferred size of the overlay for a DPI scaling of 1.0.

The size hint should be the optimal size of the overlay to display all its information. If the overlay does not have a preferred size, the size hint should be set to to the minimalSizeHint.

### setMinimalSizeHint

```cpp
virtual void ImFusion::GlOverlay::setMinimalSizeHint (int width, int height)
```

*virtual*

Sets the minimal size of the overlay for a DPI scaling of 1.0.

The minimal size should reflect the size to display the content in an undistorted way. The actual size of the overlay can still be lower than this, in which case the overlay should scale its content.
