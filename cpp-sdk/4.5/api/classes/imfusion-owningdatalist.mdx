## Overview

Wrapper class to store a list of owned [Data](/cpp-sdk/4.5/api/classes/imfusion-data) instances.

Internally, this class is merely a [std::vector](http://en.cppreference.com/w/cpp/container/vector.html)&lt;[std::unique_ptr](http://en.cppreference.com/w/cpp/memory/unique_ptr.html)&lt;[Data](/cpp-sdk/4.5/api/classes/imfusion-data)&gt;&gt;. However, it provides a couple of convenience functions for filtering and extracting data. Use a plain [DataList](/cpp-sdk/4.5/api/classes/imfusion-datalist) to model a view on a list of [Data](/cpp-sdk/4.5/api/classes/imfusion-data) instances without ownership.

**See also:** [DataList](/cpp-sdk/4.5/api/classes/imfusion-datalist)

**Examples:** [ExampleAlgorithm.cpp](/cpp-sdk/4.5/api/examples/example-algorithm-8cpp), and [ExampleAlgorithm.h](/cpp-sdk/4.5/api/examples/example-algorithm-8h).

### OwningDataList

```cpp
template<typename T> ImFusion::OwningDataList::OwningDataList (std::vector < std::unique_ptr < T > > && data)
```

Constructor from a vector of data.

**Note:** This constructor is deliberately implicit to allow for easy assignment of [OwningDataList](/cpp-sdk/4.5/api/classes/imfusion-owningdatalist) queries (i.e. extract-accessors) to OwningDataLists. For instance, you can write owningDataList2 = owningDataList1.extractAll&lt;[SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset)&gt;(); to extract all SharedImageSets from owningDataList1 and assign them to owningDataList2.

**Template Parameters:** | T | [Data](/cpp-sdk/4.5/api/classes/imfusion-data) type stored in the vector, must inherit from [Data](/cpp-sdk/4.5/api/classes/imfusion-data). |
