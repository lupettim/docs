## Overview

[Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm) to parse through folder structures and run a workspace on individual files.

Note that is algorithm only parses the folder structures and obtains a file list. The loading and execution of a workspace file is in done in the controller, because this requires access to the application controller.

The workspace file to be executed on each found file may have contain the following placeholders: %(OrigFilePath) - full path of the input file %(OrigFileDirectory) - folder that contains the input file %(OrigFileStem) - input file name stem without extension %(OrigFileExt) - input file extension, including the period %(Index) - zero-based index of the current file %(IndexPadded) - zero-based index of the current file, padded with as many leading zeros as necessary

These placeholders allow a workspace file as follows:

```cpp
<propertyfile version="1.1">
   <param name="workspaceVersion">10</param>
   <property name="Algorithms">
       <property name="Basic Processing">
           <param name="mode">7</param>
           <param name="createNew">1</param>
           <param name="preserveValuesOnBitReduction">0</param>
           <param name="execute">1</param>
           <param name="input">-1</param>
       </property>
       <property name="ImFusionFile">
           <param name="location">%(OrigFileDirectory)/%(OrigFileStem)_gray.imf</param>
           <param name="blockId">-1</param>
           <param name="execute">1</param>
           <param name="input">0</param>
       </property>
   </property>
</propertyfile>
```
 fragment
The workspace above assumes the input file to contain one item, which will have index -1 when the workspace is executed. It is greyscale converted and saved into the original folder, with the original name plus the suffix _gray.imf.

**See also:** Workspaces API, [Placeholders in Workspace Files](/cpp-sdk/4.5/api/guides/workspace-placeholders-tutorial)
