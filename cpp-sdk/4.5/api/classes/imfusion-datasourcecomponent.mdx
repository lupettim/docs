## Overview

[Data](/cpp-sdk/4.5/api/classes/imfusion-data) component for keeping track of the data's origin.

This data component introduces the [DataSourceInfo](/cpp-sdk/4.5/api/structs/imfusion-datasourcecomponent-datasourceinfo) struct, which provides members for keeping track of the original filename and IoAlgorithm+configuration that was used to load it. The component itself keeps a vector of [DataSourceInfo](/cpp-sdk/4.5/api/structs/imfusion-datasourcecomponent-datasourceinfo) elements, one for each file (since data from different files may be combined into a single SIS).

If data from multiple files is combined into a single SIS, their DataSourceComponents can be combined by summation (operator+ or operator+=) and added to the new SIS.

## Classes

- struct [DataSourceInfo](/cpp-sdk/4.5/api/structs/imfusion-datasourcecomponent-datasourceinfo) — Simple struct for storing the original filename of a data, how it has been loaded and the history of its sources.

## Member Function Documentation

### id

```cpp
std::string ImFusion::DataSourceComponent::id () const
```

*inline, override, virtual*

Returns a unique string identifier for this type of data component.

**Note:** To avoid conflicts make sure to include potential plugin names into this ID. Due to the modular architecture of the ImFusion library, uniqueness of IDs cannot be checked at compile time. However, [DataComponentFactory](/cpp-sdk/4.5/api/classes/imfusion-datacomponentfactory) will check the uniqueness of the ID during registration.

### updateSources

```cpp
void ImFusion::DataSourceComponent::updateSources (const DataSourceInfo & other, const Data & refData)
```

Helper function to move all current infos to their history and add a new source info.

If there is no current info, the vector will be filled with the apprioriate number of instances, depending on the reference data. For instance, if the reference data is a [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) with 5 frames, a vector of size 5 will be created.
