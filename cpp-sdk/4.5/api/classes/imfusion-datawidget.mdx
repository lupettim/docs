`#include <ImFusion/GUI/DataWidget.h>`

## Overview

Graphical user interface for the DataModel/DataItemModel.

This class provides a Qt-based [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui) representing the underlying [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel). Furthermore, manages the visibility of its contents in a [DisplayWidgetMulti](/cpp-sdk/4.5/api/classes/imfusion-displaywidgetmulti), and can instantiate Algorithms in the parent [ApplicationController](/cpp-sdk/4.5/api/classes/imfusion-applicationcontroller). Internally, it uses the [DataDisplayModel](/cpp-sdk/4.5/api/classes/imfusion-datadisplaymodel) to augment the input [DataItemModel](/cpp-sdk/4.5/api/classes/imfusion-dataitemmodel) with the visibility information from the display and the [DataWidgetMenu](/cpp-sdk/4.5/api/classes/imfusion-datawidgetmenu) to show a context menu for the selected datasets.

You can configure convenience functionality such as the "quick selection" functionality that synchronizes item selection with visibility in the display, or the automatic linking of poses or windowing. Finally, you can customize the visual appearance by providing a custom [DataWidgetItemDelegate](/cpp-sdk/4.5/api/classes/imfusion-datawidgetitemdelegate).

**See also:** [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel), [DataItemModel](/cpp-sdk/4.5/api/classes/imfusion-dataitemmodel), [DefaultDataWidgetItemDelegate](/cpp-sdk/4.5/api/classes/imfusion-defaultdatawidgetitemdelegate), [DataWidgetMenu](/cpp-sdk/4.5/api/classes/imfusion-datawidgetmenu)

## Public Slots

- void **showContextMenu** (const QPoint &pos) — Opens a context menu at pos.
- void **setShowThumbnails** (bool enable, bool storeInGlobalSettings=true) — Sets if data thumbnails should be displayed or not.
- void **enableQuickSelection** (bool enable, bool storeInGlobalSettings=true) — Enables quick selection i.e. the data visibility changes together with the selection.
- void **setAutoLinkWindowing** (bool enable, bool storeInGlobalSettings=true) — Enables automatic creation of [LinkWindowing](/cpp-sdk/4.5/api/classes/imfusion-linkwindowing) links per modality.
- void **setAutoLinkPose** (bool enable, bool storeInGlobalSettings=true) — Enables automatic creation of [LinkPose](/cpp-sdk/4.5/api/classes/imfusion-linkpose) links when grouping [Data](/cpp-sdk/4.5/api/classes/imfusion-data).
- void **setEnableDeleteKeyboardShortcut** (bool enable) — Enables deleting the selected data when the user presses the &lt;delete&gt; key.
- void **setDataGroupingOption** ([DataGroupingModel::GroupingOption](/cpp-sdk/4.5/api/classes/imfusion-datagroupingmodel) option, bool storeInGlobalSettings=true) — Sets the grouping-option for grouping of data by patientID/seriesID in the TreeView. Optionally stores the option in the global settings.

## Constructor & Destructor Documentation

### DataWidget

```cpp
ImFusion::DataWidget::DataWidget (DataItemModel & dataItemModel, DisplayWidgetMulti & display, ApplicationController & appCtrl, QMainWindow * mainWindow = nullptr)
```

Creates the widget using data from dataItemModel.

**Parameters:**

- `dataItemModel` — Input data model to show.
- `display` — Display widget to apply dataset visibility changes to.
- `appCtrl` — Parent [ApplicationController](/cpp-sdk/4.5/api/classes/imfusion-applicationcontroller) to launch algorithms in.
- `mainWindow` — Optional QMainWindow, only used for showing image information in the status bar.

### DataWidget

```cpp
ImFusion::DataWidget::DataWidget (MainWindowBase * main)
```

Creates the widget from *main's* data model, display, and with *main* as application controller and main window.

Does not use the annotation model (because it is not guaranteed to be valid at this point).

If the data is nullptr all thumbnails are re-created.

### linkWindowing

```cpp
bool ImFusion::DataWidget::linkWindowing (const std::vector < SharedImageSet * > & images)
```

Links the windowing properties of the given images.

Each image can only be in a single linking group. E.g. it is not possible to first link A and B, and afterwards B and C. Returns true if a new link between the images could be created.

### unlinkWindowing

```cpp
bool ImFusion::DataWidget::unlinkWindowing (const std::vector < SharedImageSet * > & images)
```

Unlinks the windowing properties of the given images.

Removes each image in the list from a linked group. The given images can be in different linked groups. Empty or single item groups are removed automatically. Returns true if at least one link was removed.

### groupData

```cpp
DataGroup * ImFusion::DataWidget::groupData (const DataList & data)
```

Groups the given data and adds "Group Data" to the history of [AlgorithmController](/cpp-sdk/4.5/api/classes/imfusion-algorithmcontroller).

Contrary to DataModel::group, the grouping will be reproduced by a workspace.

### deleteData

```cpp
void ImFusion::DataWidget::deleteData (const DataList & data)
```

Deletes the given data from the [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel).

Only data that [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) allows to be deleted gets deleted. This might open a dialog to ask the user for confirmation (see [MainWindowBase::canBeDeleted](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase)). Contrary to [DataModel::remove](/cpp-sdk/4.5/api/classes/imfusion-datamodel), the deletion will be reproduced by a workspace.
