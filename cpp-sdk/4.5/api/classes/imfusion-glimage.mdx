`#include <ImFusion/GL/GlImage.h>`

## Overview

[Image](/cpp-sdk/4.5/api/classes/imfusion-image) representation stored as OpenGL texture.

This class is a wrapper around a low-level [GL::Texture](/cpp-sdk/4.5/api/classes/imfusion-gl-texture) that additionally inherits from [Image](/cpp-sdk/4.5/api/classes/imfusion-image) to hold additional metadata in the [ImageDescriptor](/cpp-sdk/4.5/api/structs/imfusion-imagedescriptor) and make it compatible with [SharedImage](/cpp-sdk/4.5/api/classes/imfusion-sharedimage).

Furthermore, it provides a couple of member functions such as conversion from/to MemImages. However, for the majority of use cases you will likely not need to call any of them directly:

- When doing low-level OpenGL processing the [GL::Texture](/cpp-sdk/4.5/api/classes/imfusion-gl-texture) class and its API is usually sufficient and more explicit.
- When doing high-level image processing you will use the [SharedImage](/cpp-sdk/4.5/api/classes/imfusion-sharedimage)[Set] API to implement automatic conversion from/to MemImages or other representations. When receiving a [GlImage](/cpp-sdk/4.5/api/classes/imfusion-glimage) instance from you will rarely need to do more with it than passing it to a [GL::Program](/cpp-sdk/4.5/api/classes/imfusion-gl-program) or assigning it to a different [SharedImage](/cpp-sdk/4.5/api/classes/imfusion-sharedimage)[Set].

By default, unless explicitly selected during construction, [GlImage](/cpp-sdk/4.5/api/classes/imfusion-glimage) will select the [internal OpenGL texture format](/cpp-sdk/4.5/api/classes/imfusion-gl-internalformat) automatically based on the provided input PixelType:

- Byte and UByte -&gt; R[GBA]8
- Short and UShort -&gt; R[GBA]16
- HFloat -&gt; R[GBA]16F
- Int, UInt, Float, Double -&gt; R[GBA]32F

Platform-specific caveats when using OpenGL ES:

- 3-channel images use the corresponding 4-channel internal format (RGB\* -&gt; RGBA\*) because 3-channel formats are not color-renderable. The padded values are initialized to normalized 1.
- 16-bit integer textures require an extension and use use R[GBA]16_EXT.

**Warning:** 32-bit integer data in normalized texture formats and double-precision input may lose precision, because not all values are exactly representable as 32-bit float. This limitation is due to OpenGL not supporting such types as normalized texture format.

[GlImage](/cpp-sdk/4.5/api/classes/imfusion-glimage) performs the following value domain conversions during upload/download: For non-integer (normalized) texture formats, signed integer uploads (Byte, Short, Int) are converted to the corresponding unsigned type by applying a half-range offset ([-2^(N-1) .. 2^(N-1)-1] &lt;-&gt; [0 .. 2^N-1]). This maps to unsigned normalized GPU range [0..1] without collapsing distinct codes. The inverse transform is applied on download. You can use [GlUtils::storageToNormalized()](/cpp-sdk/4.5/api/namespaces/imfusion-glutils) and [GlUtils::normalizedToStorage()](/cpp-sdk/4.5/api/namespaces/imfusion-glutils) to perform the same rescaling on the CPU for any input value.

**Note:** Be advised that OpenGL supports a maximum of 4 channels per element. If you try to create a [GlImage](/cpp-sdk/4.5/api/classes/imfusion-glimage) with more than 4 channels, any additional channel will be discarded and the resulting [GlImage](/cpp-sdk/4.5/api/classes/imfusion-glimage) will have 4 channels (i.e. you lose data). This will also happen implicitly when synchronizing through a [SharedImage](/cpp-sdk/4.5/api/classes/imfusion-sharedimage).

### makeCompatible

```cpp
bool ImFusion::GlImage::makeCompatible (GlImage ** other, unsigned int type = 0, bool use2D = false, int channels = -1) const
```

Creates or updates another image instance to be compatible to this one.

**Parameters:**

- `other` — Pointer location where the compatible image should be created or updated.
- `type` — If specified, the other image will be created with that type. The type is accordingly not compared to the current image's type.
- `use2D` — If set to true only a slice image will be created.
- `channels` — If the value is bigger or equal zero the resulting image will be created with this number of channels instead of the one of the current image. The number of channels is accordingly not compared to the current image's number of channels.

**Returns:** true if an image was (re-)created at the specified pointer location.
