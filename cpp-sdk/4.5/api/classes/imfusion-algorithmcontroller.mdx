## Overview

Specialization of the [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) interface to be used with [Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm) instances.

An [AlgorithmController](/cpp-sdk/4.5/api/classes/imfusion-algorithmcontroller) is supposed to provide a [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui) enabling the user to configure and control the underlying [Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm). In contrast to the Algorithm interface the AlgorithmController has access to the parent [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) instance and can therefore also access entities that usually would not be accessible by the Algorithm (e.g. [DisplayWidget](/cpp-sdk/4.5/api/classes/imfusion-displaywidget), [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel), other Algorithms, ...).

AlgorithmControllers receive different callbacks:

- [onSelectedDataChanged()](/cpp-sdk/4.5/api/classes/imfusion-controller) is called whenever the selection in the [DataWidget](/cpp-sdk/4.5/api/classes/imfusion-datawidget) changes.
- [onVisibleDataChanged()](/cpp-sdk/4.5/api/classes/imfusion-controller) is called whenever the visible data in the [DisplayWidgetMulti](/cpp-sdk/4.5/api/classes/imfusion-displaywidgetmulti) changes.
- [onAlgorithmParametersChanged()](#onalgorithmparameterschanged) is called whenever the underlying [Algorithm::signalParametersChanged](/cpp-sdk/4.5/api/classes/imfusion-algorithm) was emitted.
- [onAlgorithmOutputChanged()](#onalgorithmoutputchanged) is called whenever the underlying [Algorithm::signalOutputChanged](/cpp-sdk/4.5/api/classes/imfusion-algorithm) was emitted.

Furthermore, it extends the base [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) interface with extra functionality specific to Algorithms. If wrapped in a DefaultControllerContainer, it will decorate it with additional buttons for showInputData() and [showHelp()](#showhelp).

**Note:** AlgorithmControllers should be registered with the [AlgorithmControllerFactory](/cpp-sdk/4.5/api/classes/imfusion-algorithmcontrollerfactory) interface so that [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) can instantiate them automatically together with their [Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm).

**See also:** [Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm), [AlgorithmControllerFactory](/cpp-sdk/4.5/api/classes/imfusion-algorithmcontrollerfactory), [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase), [ControllerDecorator](/cpp-sdk/4.5/api/classes/imfusion-controllerdecorator), [DataController](/cpp-sdk/4.5/api/classes/imfusion-datacontroller)

## Member Enumeration Documentation

### InputDataVisibility

```cpp
enum class ImFusion::AlgorithmController::InputDataVisibility
```

*strong*

Enumeration of possible return values for showsInputData().

**Enumerator:**

- `None` — None of the input data is currently visible.
- `All` — All of the input data is currently visible.
- `Some` — A subset of the input data is currently visible.

### closeAlgorithm

```cpp
virtual void ImFusion::AlgorithmController::closeAlgorithm (bool executed = false)
```

*protected, virtual*

Removes the algorithm from the [ApplicationController](/cpp-sdk/4.5/api/classes/imfusion-applicationcontroller) and closes the controller.

**Warning:** Since this method deletes the algorithm and the controller, the this pointer becomes invalid immediately. Therefore this method should be called last. If you need to wait for certain events (e.g. Qt signals) before doing so you should override this function and implement the necessary custom logic.


*protected*

Adds the contents of *data* to the [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel) of the parent [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase).

If a dataset is annotation data ([Data::isAnnotationType()](/cpp-sdk/4.5/api/classes/imfusion-data)) it will take care of also adding it to the [AnnotationModel](/cpp-sdk/4.5/api/classes/imfusion-annotationmodel).
