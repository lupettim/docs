`#include <ImFusion/GL/IntensityMask.h>`

## Overview

Masks pixels with a specific value or values outside a specific range.

This mask type can work in two modes: either it considers only a single value as masked (default) or you can enabled a ranged mode where all pixel values outside a configured value range are considered as masked. Therefore, during mask query it requires the actual pixel value as input. However, comparison against the configured mask value (range) is only performed on the first channel and all other channels are ignored if present.

**Note:** The intensity values used during masking (i.e. when calling [maskValue()](#maskvalue)) are compared to the exact same values set during [setMaskedValue()](#setmaskedvalue) or [setMaskValueRange()](#setmaskvaluerange), respectively. In other words: there is no shift/scale transformation involved.

## Member Function Documentation

### setUseRange

```cpp
void ImFusion::IntensityMask::setUseRange (bool enable)
```

Configure whether the mask should operate in range mode (true) or single-value mode (false).

Depending on the mode use [setMaskedValue()](#setmaskedvalue) or [setMaskValueRange()](#setmaskvaluerange) to configure the thresholds.

### setMaskedValue

```cpp
void ImFusion::IntensityMask::setMaskedValue (float v)
```

Sets a specific pixel value that should be ignored.

The value should be given in storage domain of the type of the mask.

**Note:** Make sure to also call setUseRange(false) to enable single-value mode.

### setMaskValueRange

```cpp
void ImFusion::IntensityMask::setMaskValueRange (float min, float max)
```

Sets a half open range [min, max) of allowed pixel values.

Values outside this range will be masked (i.e. considered outside). The values should be given in storage domain of the type of the mask.

**Note:** Make sure to also call setUseRange(true) to enable ranged mode.

### maskValue

```cpp
uint8_t ImFusion::IntensityMask::maskValue (const vec3i & coord, const vec4f & color) const
```

*override, virtual*

Returns 0 if the given pixel is outside the mask (i.e.

invisible/to be ignored) or a non-zero value if it is inside the mask (i.e. visible/to be considered).

This function takes both the pixel coordinate and pixel value as input. Concrete implementations of the [Mask](/cpp-sdk/4.5/api/classes/imfusion-mask) interface are free to use only either input for their decision if it makes sense. Also, it is up to the implementation whether it supports multi-channel images or not. Subclasses only supporting grayscale images should only use the first channel.

**Parameters:**

- `coord` — Pixel coordinate to be tested against the mask, can be outside of the image.
- `color` — Pixel value (not normalized) to be tested against the mask. This parameter is only needed if [requiresPixelValue()](#requirespixelvalue) returns true, which can allow for optimizations where pixel lookup is expensive.

### requiresPixelValue

```cpp
bool ImFusion::IntensityMask::requiresPixelValue () const
```

*override, virtual*

Returns true if the [maskValue()](#maskvalue) rely on the pixel value.

If this method returns false, the [maskValue()](#maskvalue) method can be safely used with only the coordinate. This is useful to avoid an otherwise unnecessary and expensive pixel lookup.

### resizedMask

```cpp
std::unique_ptr < Mask > ImFusion::IntensityMask::resizedMask (const SharedImage & original, const Image & target) const
```

*override, virtual*

Creates a copy of the current mask, resized to the dimensions of the target image.

The default implementation calls createExplicitMask and resamples the result. Returns nullptr if the mask cannot be resized.

### setIncludeArguments

```cpp
int ImFusion::IntensityMask::setIncludeArguments (GL::Program & p, const std::string & prefix = "mask") const
```

*override, virtual*

Called by users of [GlMask](/cpp-sdk/4.5/api/classes/imfusion-glmask) in order to correctly setup the mask shader.

Subclasses should implement this function so that it correctly sets all needed uniforms and input textures to be used by the OpenGL shader instance. Make sure to use the given *prefix* argument instead of a hardcoded "mask" when defining uniform names to support multi-includes where the prefixes have been replaced with unique alternatives. Example:

```cpp
p.setArgument(prefix + "UseRange", m_useRange);
```
