## Overview

ImFusion::Algorithm that wraps around a [StructuresView](/cpp-sdk/4.5/api/classes/imfusion-seg-structuresview) and adds a couple of convenience functions.

This algorithm provides the following set of extra features over a [StructuresView](/cpp-sdk/4.5/api/classes/imfusion-seg-structuresview):

- You can optionally set a [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel) to use as storage backend. In this case, the algorithm will subscribe to all changes of the [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel) and configure the underlying [StructuresView](/cpp-sdk/4.5/api/classes/imfusion-seg-structuresview) to always show all label maps (images with Modality::LABEL) that are present in the [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel).
- When using a [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel) as storage backend, you can use the convenience functions such as [createStructure()](#createstructure)/deleteStructure(), that will change the underlying [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel) and create/delete label maps if needed. These functions have no effect if there is no [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel) assigned.

**See also:** [StructuresView](/cpp-sdk/4.5/api/classes/imfusion-seg-structuresview), [StructuresViewAlgorithmController](/cpp-sdk/4.5/api/classes/imfusion-seg-structuresviewalgorithmcontroller)

## Member Function Documentation

### detectLabels

```cpp
void ImFusion::Seg::StructuresViewAlgorithm::detectLabels ()
```

Calls [LabelDataComponent::detectLabels()](/cpp-sdk/4.5/api/classes/imfusion-labeldatacomponent) for every registered label map.

If a [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel) is set, it additionally checks whether any data sets of modality LABEL are in there, which are not yet registered (this can happen if the modality has changed after adding them to the [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel)).

### createStructure

```cpp
Structure * ImFusion::Seg::StructuresViewAlgorithm::createStructure (const SharedImageSet & referenceImage, bool reuseExistingLabelMapIfPossible)
```

Creates a new [Structure](/cpp-sdk/4.5/api/classes/imfusion-seg-structure) for the given reference image.

If reuseExistingLabelMapIfPossible is true, it will check whether there is already an existing label map for referenceImage and if there are still free label configurations available. If so, it will use that label map instead of creating a new one. Furthermore, this function will automatically create a [LinkPose](/cpp-sdk/4.5/api/classes/imfusion-linkpose) instance in the [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel) to keep the poses of label map and reference image in sync.

**Note:** This function has no effect if there is no [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel) configured.

### deleteStructure

```cpp
void ImFusion::Seg::StructuresViewAlgorithm::deleteStructure (Structure * structure)
```

Deletes the given structure from the underlying storage backend.

All pixels of the label map that have the structure's label value are set to 0. If this was the last structure of the label map, the whole label map will be deleted (from the [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel)).

**Note:** This function has no effect if there is no [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel) configured.
