## Overview

Factory singleton for creating [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) from string IDs.

Every [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) implements the typeName() method returning a unique string identifier for that specific type of [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject). GlObjects have to be registered with [GlObjectFactory](/cpp-sdk/4.5/api/classes/imfusion-globjectfactory) using this string ID and a factory method for instantiating a [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) of such type. During registration [GlObjectFactory](/cpp-sdk/4.5/api/classes/imfusion-globjectfactory) ensures the uniqueness of the registered GlObjects and otherwise raises an assertion/logs a warning.

## Member Function Documentation

### registerObject

```cpp
void ImFusion::GlObjectFactory::registerObject (const std::string & id, FactoryFunction factoryFunction, CompatibilityFunction factoryCompatibilityFunction)
```

Registers a [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) with the [GlObjectFactory](/cpp-sdk/4.5/api/classes/imfusion-globjectfactory).

**Note:** The string IDs of GlObjects have to be unique. If you register an ID a second time, the previously registered factory method will be overwritten. [Debug](/cpp-sdk/4.5/api/namespaces/imfusion-gl-debug) builds will also raise an assertion.

**Parameters:**

- `id` — Unique string ID for the [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) to register.
- `factoryFunction` — Method to use for instantiating the [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) to register which takes reference data and potential objects to wrap.
- `factoryCompatibilityFunction` — Method to use for checking if the [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) can be created with given data and potential objects to wrap.

**Parameters:**

- `factoryFunction` — Method to use for instantiating the [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) to register.
