`#include <ImFusion/Core/Filesystem/File.h>`

## Overview

Entity representing a file in the filesystem.

The file does not necessarily exist and [File](/cpp-sdk/4.5/api/classes/imfusion-filesystem-file) will not perform any checks whether the given file name represents a valid identifier for the current platform.

The underlying [Path](/cpp-sdk/4.5/api/classes/imfusion-filesystem-path) will always be absolute: if you construct an instance using a relative path the constructor will make it absolute using the currentWorkingDirectory() and all subsequent calls to member functions will correspond to that absolute path.

**Examples:** [Filesystem.cpp](/cpp-sdk/4.5/api/examples/filesystem-8cpp).

## Member Function Documentation

### isExecutable

```cpp
bool ImFusion::Filesystem::File::isExecutable () const
```

Checks if the file is executable (and exists).

On Windows, the file must be a 32- or 64-bit "Windows-based" application (according to GetBinaryType()) On Unix, the file must grant the executable permission (according to access())

### copyTo

```cpp

```

Copies this single file to *destination* using the provided copy options.

**Parameters:**

- `destination` — Target file name of the copy. If *destination* describes an existing directory, then behaves as [copyTo](#copyto)(destination / source.filename(), copyOptions).
- `copyOptions` — Optional flags to control the behavior of this function. [Flags](/cpp-sdk/4.5/api/classes/imfusion-flags) irrelevant to files are ignored and do not report an error.

### readText

```cpp
std::vector < std::string > ImFusion::Filesystem::File::readText () const
```

Read the entire content of the file in a text-based fashion and return it as vector of lines.

This function will use [std::ifstream](http://en.cppreference.com/w/cpp/io/basic_ifstream.html) in text mode internally and consume the content using [std::getline()](http://en.cppreference.com/w/cpp/string/basic_string/getline.html) until the end of the file is reached. The newline characters (\n) after each line are removed; no further post-processing is performed. The corresponding platform-dependent behavior of C-style text file I/O applies.

**Note:** Please resort to [std::ifstream](http://en.cppreference.com/w/cpp/io/basic_ifstream.html) if you need more advanced functionality such as random access or stream processing.

### writeBinary

```cpp
bool ImFusion::Filesystem::File::writeBinary (const ByteBufferView & data) const
```

Write the given *data* to the file in a binary fashion.

If the file does not exist, it will be created recursively. If the file already exists, all content will be overwritten.

**Note:** Please resort to [std::ofstream](http://en.cppreference.com/w/cpp/io/basic_ofstream.html) if you need more advanced functionality such as random access or text-based processing.

**Examples:** [Filesystem.cpp](/cpp-sdk/4.5/api/examples/filesystem-8cpp).

### writeText

```cpp

```

Write the given list of strings into the file in a text-based fashion inserting a newline character (\n) after each line.

If the file already exists, all content will be overwritten. This function will use [std::ofstream](http://en.cppreference.com/w/cpp/io/basic_ofstream.html) in text mode internally, thus the corresponding platform-dependent behavior of C-style text file I/O applies.
