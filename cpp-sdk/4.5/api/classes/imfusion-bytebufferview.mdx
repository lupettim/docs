## Overview

Const view onto a sized buffer of raw binary data.

This class can be used to represent any kind of raw data independent of encoding or other limitations. For instance, in contrast to a [std::string](http://en.cppreference.com/w/cpp/string/basic_string.html) it can properly represents 0-bytes (\0) at any location. It annotates the raw buffer pointer with the size of the data and offers convenience functions.

This most base class of the ByteBuffer-class hierarchy that represents const read access to a contiguous buffer of raw bytes and can for instance be used as argument type for non-mutating processing functions. Two specializations of this class exist:

- [ByteBufferMutableView](/cpp-sdk/4.5/api/classes/imfusion-bytebuffermutableview) additionally provides non-const access to the underlying buffer
- [ByteBuffer](/cpp-sdk/4.5/api/classes/imfusion-bytebuffer) additionally models ownership of the underlying data

ByteBuffer(Mutable)View does not own the underlying buffer. All copies/assignments of a ByteBuffer(Mutable)View will be shallow and not copy the underlying buffer. However, comparison of multiple ByteBuffer(Views) will always perform a deep comparison of the underlying data in a byte-wise fashion independent of the concrete specialized type.

**See also:** [ByteBufferStream](/cpp-sdk/4.5/api/classes/imfusion-bytebufferstream)
