### absoluteDataSourcePath

```cpp
const Filesystem::Url ImFusion::DataDescriptor::absoluteDataSourcePath () const
```

Returns the final location where the [DataSource](/cpp-sdk/4.5/api/structs/imfusion-datasource) can be (down)loaded.

This might be a file on the local filesystem (i.e. file: scheme), a download link of the remote project server or a link to some external storage. Remote projects will never return a file: url. Local projects might return something else than file: urls (e.g. http for DICOMweb).

### importPixelwiseLayers

```cpp
bool ImFusion::DataDescriptor::importPixelwiseLayers (std::vector < SharedImageSet * > layers, Project::LayersToModify layersToModify, Progress * progress = nullptr)
```

Import the given pixelwise layers.

The given layers don't need to match the pixelwise layers of this dataset, only the entries in the [LabelDataComponent](/cpp-sdk/4.5/api/classes/imfusion-labeldatacomponent) must match the names of the labels. E.g. if the [LabelDataComponent](/cpp-sdk/4.5/api/classes/imfusion-labeldatacomponent) of a given layer contains a label with the name "Left Kidney", this label will be assigned to the "Left Kidney" label as defined in the project. It does not matter in which layer this label is defined, it will be automatically mapped to the corresponding layer. If a label has a matching name but the value is not matching, then the value will be mapped to the value defined in the project. Basically the given layers can contain the labels in arbitary order and with arbitary values. As long as the names can be matched, the layers will be imported according to the project definition.

Labels that are not referenced by the [LabelDataComponent](/cpp-sdk/4.5/api/classes/imfusion-labeldatacomponent) are left unmodified. Therefore it's possible to only update specific labels. Labels that are referenced by the [LabelDataComponent](/cpp-sdk/4.5/api/classes/imfusion-labeldatacomponent) but are not present in the project, are ignored.

Returns true if all layers could be imported. If any of the layers is incompatible to the dataset (e.g. wrong dimension or spacing) or if not all labels can be matched to a project label, no pixelwise layer of this dataset is modified and false is returned.
