`#include <ImFusion/Base/Progress.h>`

## Overview

Represents the progress/status report of an individual task hosted by a [Progress](/cpp-sdk/4.5/api/classes/imfusion-progress).

Changes to a [Task](/cpp-sdk/4.5/api/classes/imfusion-progress-task) object are forwarded to the parent [Progress](/cpp-sdk/4.5/api/classes/imfusion-progress) handler. In case no parent [Progress](/cpp-sdk/4.5/api/classes/imfusion-progress) is provided, progress reports will not have any effect and effectively result in a no-op. This allows for more concise code in case a [Progress](/cpp-sdk/4.5/api/classes/imfusion-progress) is not guaranteed to be present:

```cpp
// The pointer to the Progress instance may be null
Progress* progressHandler = ...;
Progress::Task progress(progressHandler, 17, "Computing...");

[...]

// will effect in a no-op in case progressHandler was null
progress.setCurrentStep(2);
```
 fragment
Removes itself automatically from the parent on destruction.

By default tasks are not thread-safe and must only be accessed from the same thread they were created on. You can request a limited support for concurrent access, see [requestThreadSafety()](#requestthreadsafety).

Create a new progress report for an operation with the given number of steps.

**Parameters:**

- `parent` — Parent progress handler to use for reporting, nullptr is allowed
- `numSteps` — Total number of steps, should be &gt;= 0, will show a busy indicator if set to 0.
- `description` — Description text to be shown next to the progress indicator.

### requestThreadSafety

```cpp
bool ImFusion::Progress::Task::requestThreadSafety ()
```

Tells the task and its parent reporter that this task instance is going to be updated concurrently from multiple threads and they therefore need to install synchronization primitives.

A [Progress](/cpp-sdk/4.5/api/classes/imfusion-progress) is free to reject this request (cf. [Progress::onTaskRequestsThreadSafety()](/cpp-sdk/4.5/api/classes/imfusion-progress)). In this case the function returns false and disconnects from the reporter so that all subsequent task updates result in a no-op.

**Note:** This function must still be called from the original thread. After calling it you can update the tasks state from any other thread. However, this does not apply to the [Task](/cpp-sdk/4.5/api/classes/imfusion-progress-task) destructor.
