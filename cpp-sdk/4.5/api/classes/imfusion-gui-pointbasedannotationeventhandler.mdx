## Overview

Event handler for creating and editing [point-based annotations](/cpp-sdk/4.5/api/classes/imfusion-glpointbasedannotation).

This event handler can operate in two different modes:

- In "creation mode" the editing of the existing points is not possible and the only actions you can perform are to append additional points at the end, or to finish or abort the creation. Usually, this is the initial mode when starting with an empty annotation without any points.
- In "regular mode" the full set of editing actions are available.

If you need an interactively editable [GlPointBasedAnnotation](/cpp-sdk/4.5/api/classes/imfusion-glpointbasedannotation) you additionally instantiate the [PointBasedAnnotationEventHandler](/cpp-sdk/4.5/api/classes/imfusion-gui-pointbasedannotationeventhandler) and wrap the two together in a [TypedViewObject](/cpp-sdk/4.5/api/classes/imfusion-gui-typedviewobject).

## Member Enumeration Documentation

### Action

```cpp
enum class ImFusion::GUI::PointBasedAnnotationEventHandler::Action
```

*strong*

Enumeration of available actions this event handler can perform on the underlying annotation.

**Enumerator:**

- `CreateAppendPoint` — Append a point to the end while in "creation mode".
- `AbortCreation` — Abort "creation mode", remove all previously created points, and switch to "regular mode".
- `FinishCreation` — Finish "creation mode" and switch to "regular mode".
- `MovePoint` — Move an existing point while in "regular mode".
- `InsertPoint` — Insert a new point to the annotation while in "regular mode".
- `RemovePoint` — Remove an existing point while in "regular mode".
- `MoveAnnotation` — Move the entire annotation while in "regular mode".

## Member Function Documentation

### enableCreationMode

```cpp
bool ImFusion::GUI::PointBasedAnnotationEventHandler::enableCreationMode ()
```

Switches the event handler into "creation mode" where the only possible action for the user is to append new points at the end.

Returns false if "creation mode" cannot be enabled because the maximum number of points is already reached.

### actionMapper


*inline*

Get access to the mapping of input events to [Actions](#action) of this event handler.

You can use this to customize the mapping of mouse and keyboard buttons, or disable certain Actions altogether.

### handleInputEvent

```cpp
EventResult ImFusion::GUI::PointBasedAnnotationEventHandler::handleInputEvent (const InputEvent & event, const View & view)
```

*override, virtual*

This function is called by sources of input events in order to dispatch them to the handlers.

In case there are more than one InputEventHandlers dispatchers are intended call this function consecutively until the first handler indicates to stop by returning a result where [EventResult::stopPropagation()](/cpp-sdk/4.5/api/structs/imfusion-gui-eventresult) is true.
