`#include <ImFusion/GUI/Display.h>`

## Overview

Basic version of a display surface hosting a set of [Views](/cpp-sdk/4.5/api/classes/imfusion-gui-view), rendering them, and distributing input events to them.

A [Display](/cpp-sdk/4.5/api/classes/imfusion-gui-display) represents a rectangular viewport in OpenGL space into which it will render the views. The arrangement of the views inside the viewport is determined by the [DisplayLayout](/cpp-sdk/4.5/api/classes/imfusion-gui-displaylayout) member. The default implementation will simply arrange all views horizontally next to each other while subclasses can customize this behavior. Furthermore, you can always override the layouting from the outside.

[Display](/cpp-sdk/4.5/api/classes/imfusion-gui-display) provides the [handleInputEvent()](#handleinputevent) function to handle keyboard/mouse/touch events from the user. Those events are forwarded to the views according their current layouting position.

In order to bring the [Display](/cpp-sdk/4.5/api/classes/imfusion-gui-display) into your application you will usually need to wrap it inside an OpenGL-capable container widget corresponding to the used [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui) toolkit/windowing platform. That wrapper must take care of setting the viewport correctly and calling [render()](#render) from the correct OpenGL context. Furthermore, it needs to convert the platform-specific user input events to the corresponding [ImFusion InputEvents](/cpp-sdk/4.5/api/classes/imfusion-gui-inputevent) and pass them to [handleInputEvent()](#handleinputevent). The resulting generic [GUI::EventResult](/cpp-sdk/4.5/api/structs/imfusion-gui-eventresult) can then be used to yield the intended platform-specific effects. [QtDisplayWidget](/cpp-sdk/4.5/api/classes/imfusion-gui-qtdisplaywidget) provides a ready-to-use implementation for a QtWidgets-based application.

### render

```cpp
virtual void ImFusion::GUI::Display::render (std::string * outCriticalGlErrorMessage = nullptr) const
```

*virtual*

Perform OpenGL-based rendering of the [Display](/cpp-sdk/4.5/api/classes/imfusion-gui-display) and all its Views.

Call this function from the correct OpenGL context and your framebuffer [object] setup correctly. If a background color is set it will clear the framebuffer with it. Then it will call [View::render()](/cpp-sdk/4.5/api/classes/imfusion-gui-view) for each view.

At the end of the rendering the function will check the OpenGL error flag. After the third frame where a critical error occurred (e.g. out-of-memory) the [Display](/cpp-sdk/4.5/api/classes/imfusion-gui-display) will disable all rendering to avoid further corruption of the system. This shall give the application the opportunity to at least perform a graceful shutdown. Callers can be notified about this by passing a pointer-to-string to the optional *outCriticalGlErrorMessage* parameter.

### handleInputEvent

```cpp
EventResult ImFusion::GUI::Display::handleInputEvent (const InputEvent & event)
```

Entry point for the dispatching of input events from users.

Usually, this function is called by the platform-specific wrapper class of the [Display](/cpp-sdk/4.5/api/classes/imfusion-gui-display) (e.g. [QtDisplayWidget](/cpp-sdk/4.5/api/classes/imfusion-gui-qtdisplaywidget)) after converting the platform-specific event to the generic [InputEvent](/cpp-sdk/4.5/api/classes/imfusion-gui-inputevent) type. It will forward the event to the [DisplayLayout](/cpp-sdk/4.5/api/classes/imfusion-gui-displaylayout) and all Views and return an [EventResult](/cpp-sdk/4.5/api/structs/imfusion-gui-eventresult) instance, which the wrapper class shall handle accordingly.

The default implementation will switch on the type of *event* and forward to either handleKeyEvent(), handleMouseEvent(), handleTouchEvent(), or handleContextMenuEvent(). Thus, if you need to implement custom logic you should override those member functions.
