`#include <ImFusion/GUI/ImageView2D.h>`

## Overview

High-level wrapper class for 2D/MPR rendering of one or multiple images.

Serves as Qt wrapper of the [GlSliceView](/cpp-sdk/4.5/api/classes/imfusion-glsliceview) and [GlSlice](/cpp-sdk/4.5/api/classes/imfusion-glslice) classes for 2D/MPR rendering. Depending on the displayOptionsSource property of [ImageView2D](/cpp-sdk/4.5/api/classes/imfusion-imageview2d) the [DisplayOptions2d](/cpp-sdk/4.5/api/classes/imfusion-displayoptions2d) used during rendering are either individual for each image (stored in the images' DataComponents) or shared for all shown images (stored in [ImageView2D](/cpp-sdk/4.5/api/classes/imfusion-imageview2d)).

This class offers a couple of additional convenience interfaces/functionality over the underlying hierarchy of [GlSliceView](/cpp-sdk/4.5/api/classes/imfusion-glsliceview), [GlSlice](/cpp-sdk/4.5/api/classes/imfusion-glslice), and [GlSliceRenderer](/cpp-sdk/4.5/api/classes/imfusion-glslicerenderer):

- High-level interface to generically set the shown [Data](/cpp-sdk/4.5/api/classes/imfusion-data) (cf. [setVisibleData()](/cpp-sdk/4.5/api/classes/imfusion-gui-view), [showData()](/cpp-sdk/4.5/api/classes/imfusion-gui-view), [hideData()](/cpp-sdk/4.5/api/classes/imfusion-gui-view), and setDisplayOptionsSource())
- Automatic zoom to fit the viewport (cf. setZoomMode())
- Automatic configuration of view settings based on the shown data (cf. [setAutomaticViewSettingsEnabled()](#setautomaticviewsettingsenabled))
- Animation support for actions such as changing the view matrix

**Note:** Due to the potential external storage of display options, the Qt signals for changes of individual display options (i.e. windowChanged, levelChanged, gammaChanged, invertedChanged, and blendFactorChanged) are only emitted when using their corresponding Qt property setter. If the underlying [DisplayOptions2d](/cpp-sdk/4.5/api/classes/imfusion-displayoptions2d) have changed by somebody else in the background, this is not detected. If you need to be notified reliably in all occasions you need to register to the [DisplayOptions2d](/cpp-sdk/4.5/api/classes/imfusion-displayoptions2d) changed signal directly.

## Member Enumeration Documentation

### ZoomMode

```cpp
enum class ImFusion::ImageView2D::ZoomMode
```

*strong*

Enumeration to describe the zooming mode used for a 2D view.

**Enumerator:**

- `FitToView` — A zoom factor of 1 will fit the shown image to the viewport.
- `Fixed` — A zoom factor of 1 will show 1mm per viewport pixel.
- `OriginalPixelSize` — A zoom factor of 1 will show 1 image pixel (of the first visible image) per viewport pixel.

### DisplayOptionsSource

```cpp
enum class ImFusion::ImageView2D::DisplayOptionsSource
```

*strong*

Describes from where to take the DisplayOptions for rendering.

**Enumerator:**

- `View` — All visible data will be rendered using the same view's display options.
- `Data` — Every visible data will be rendered using its own display options.

### displayOptions

```cpp
const DisplayOptions2d & ImFusion::ImageView2D::displayOptions (const Data * data = nullptr) const
```

Return the DisplayOptions used for the given data wrt.

to the views' display options source setting. If the display options source is set to View, this function will return the view's display options. If the display options source is set to Modality, this function will return the global display options wrt. to *data*. If the display options source is set to [Data](/cpp-sdk/4.5/api/classes/imfusion-data), this function will return the display options of *data*. If *data* is 0 but required, it will use the first visible data of this view instead.

### setImage

```cpp
void ImFusion::ImageView2D::setImage (SharedImage * image)
```

Convenience method to display a single image.

Internally, creates a container [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) holding *image* and then calls [setVisibleData()](/cpp-sdk/4.5/api/classes/imfusion-gui-view). ### setImage

void ImFusion::ImageView2D::setImage (Image * image)

Convenience method to display a single image.
