## Overview

Set of images with arbitrarily sampled tracking data.

The class replaces the matrix assignment within [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) (each contained image uses its own matrix) by a look-up into [TrackingSequence](/cpp-sdk/4.5/api/classes/imfusion-trackingsequence) instances. Therefore, directly setting a matrix of an individual frame is deactivated, and the matrices of the individual images are ignored.

Resolving which matrix of which tracking sequence is associated with a particular image depends on several options:

1. In general, the main tracking sequence (trackingUsed) is used for look-up.
2. Lookup strategy based on timestamps:
  - If useTimestamps is false:
    - A 1:1 lookup is performed: the frame \(i\) is associated directly with tracking matrix \(T_i\).
  - If useTimestamps is true:
    - The image timestamp \(t_i - t_&#123;off&#125;\) is used to find the matrix with the closest timestamp, where \(t_&#123;off&#125;\) denotes [TrackingSequence::temporalOffset](/cpp-sdk/4.5/api/classes/imfusion-trackingsequence).
    - If that timestamp lies before the first or after the last in the tracking sequence, the first or last matrix is used, respectively.
    - Otherwise, the matrix is interpolated (using quaternions for the rotation) between the two closest pose matrices.
3. The final pose \(P\) for a given frame is computed using [TrackingSequence::registration](/cpp-sdk/4.5/api/classes/imfusion-trackingsequence) \(R\) and [TrackingSequence::calibration](/cpp-sdk/4.5/api/classes/imfusion-trackingsequence) \(C\) matrices:
  \[P = R \cdot T \cdot C
  \]
4. If there is a relative Tracking [Sequence](/cpp-sdk/4.5/api/classes/imfusion-dicom-sequence) with matrices \(R'\), \(T'_j\), and \(C'\), the final pose becomes:
  \[P_i = R \cdot \left( &#123;R'&#125; \cdot T'_j \cdot &#123;C'&#125; \right)^&#123;-1&#125; \cdot T_i \cdot C
      = R \cdot &#123;C'&#125;^&#123;-1&#125; \cdot &#123;T'_j&#125;^&#123;-1&#125; \cdot &#123;R'&#125;^&#123;-1&#125; \cdot T_i \cdot C
  \]

  - The matrix \(T'_j\) is determined as in step 2, unless [TrackingSequence::relativeToFirst](/cpp-sdk/4.5/api/classes/imfusion-trackingsequence) is true, in which case it is simply the first matrix \(T'_0\).

See [TrackingSequence](/cpp-sdk/4.5/api/classes/imfusion-trackingsequence) for details on the registration/calibration matrices and temporal offset. All matrices are natively stored in [Pose::TOWORLD](/cpp-sdk/4.5/api/namespaces/imfusion-pose) convention, and methods called "..FromWorld" will evualate a matrix inversion.

**Note:** It is advised to only include calibration and registration matrices in the main tracking sequence.

While the class does not strictly enforce identical type, pixel size, spacing, etc. across individual frames, unexpected behavior may result from noncompliance with these assumptions. In particular, all methods exposing size measures in any form (frame pixel or phyiscal size) only access the first frame and ignore all others. I/O operations with noncompliant images may not work.

## Member Function Documentation

### add

```cpp
void ImFusion::TrackedSharedImageSet::add (std::shared_ptr < SharedImage > img)
```

*override, virtual*

Add a shared image to the data set.

This method checks if the added images have [SharedImage::matrix](/cpp-sdk/4.5/api/classes/imfusion-sharedimage) different from identity, and if so, assumes that the user is deliberately adding these images with matrices and wants to retain them. In many cases it is not trivial to understand unambiguously what the users wants given the images and tracking samples already there, and to keep images and [TrackingSequence](/cpp-sdk/4.5/api/classes/imfusion-trackingsequence) instances synchronized, so the method will in such cases check the following conditions:

- The image set is not using timestamps.
- The image set is empty and there is no [TrackingSequence](/cpp-sdk/4.5/api/classes/imfusion-trackingsequence) there (it will be created), or there is exactly one [TrackingSequence](/cpp-sdk/4.5/api/classes/imfusion-trackingsequence) with as many matrices as images in the set. If both conditions are met, the matrix is added to the [TrackingSequence](/cpp-sdk/4.5/api/classes/imfusion-trackingsequence). It is taken as-is, and not inverted (note that [TrackedSharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-trackedsharedimageset) uses [Pose::TOWORLD](/cpp-sdk/4.5/api/namespaces/imfusion-pose) convention). Note that these criteria are stricter than [canSetMatricesIndividually](#cansetmatricesindividually).

### setMatrix

```cpp
void ImFusion::TrackedSharedImageSet::setMatrix (const mat4 & m, int which, bool updateAll = false)
```

*override, virtual*

Attempts to update tracking matrices.

- If `updateAll` is true, will update the image set's registration matrix so that querying the matrix of image `which` (focus if -1) will return `m` afterwards.
- If no timestamps are used, follows the same logic as [add(shared_ptr&lt;SharedImage&gt;)](#add) and adds the matrix to the (potentially newly created) tracking sequence if one-to-one mapping is unambiguous.
- If [canSetMatricesIndividually](#cansetmatricesindividually) is true, updates the tracking sequence's raw matrix so that querying the matrix of image \which will return `m` afterwards.
- Does nothing and issues a warning otherwise.
