## Overview

template&lt;class T&gt;

class ImFusion::Threading::Protected&lt; T &gt;
Thread-safe wrapper for a shared resource protected by a mutex.

This class provides synchronized access to a resource of type T by wrapping it with an internal mutex. Access to the resource is granted through [Locked&lt;T&gt;](/cpp-sdk/4.5/api/classes/imfusion-threading-protected-locked) objects, which ensure that the mutex is held for the duration of access.

The [Protected](/cpp-sdk/4.5/api/classes/imfusion-threading-protected) class manages both the resource and its protecting mutex, ensuring they have the same lifetime and are correctly paired.

**Template Parameters:** | T | The type of the resource to protect. Can be any copyable or movable type. |
| --- | --- |

**Note:** Protected&lt;shared_ptr&lt;T&gt;&gt; should not be used. Use SharedProtected&lt;T&gt; instead. Usage example:
```cpp
// Create a synchronized vector
Protected<std::vector<int>> syncVec(std::vector<int>{});

// Thread 1: Add elements safely
{
 auto locked = syncVec.lock();
    locked->push_back(1);
    locked->push_back(2);
}  // Lock released

// Thread 2: Read elements safely
{
 auto locked = syncVec.lock();
 for (int val : *locked) {
 std::cout << val << std::endl;
    }
}  // Lock released
```

### lock

```cpp
template<class T> Locked ImFusion::Threading::Protected < T >::lock ()
```

*inline, nodiscard*

Acquires exclusive access to the protected resource.

Returns a [Locked&lt;T&gt;](/cpp-sdk/4.5/api/classes/imfusion-threading-protected-locked) object that holds the mutex lock and provides access to the underlying resource. The lock is held until the [Locked](/cpp-sdk/4.5/api/classes/imfusion-threading-protected-locked) object is destroyed.

**Note:** This method will block if another thread currently holds the lock.

**Note:** CAUTION: Calling two locking functions in the same scope can lead to deadlocks. Use lockAll(...) instead.

**See also:** lockAll
