`#include <ImFusion/Base/HistogramDataComponent.h>`

## Overview

[DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) storing information on the intensity distribution of the image they are attached to.

You can use it to query the minimum/maximum intensity of an image as well as its intensity histogram. Most information is lazy-evaluated and cached. Thus, you can expect consecutive calls to [minmax()](#minmax-1-2) and/or [histogram()](#histogram) as being free as long as the corresponding image remains unchanged.

Example:

```cpp
SharedImageSet sis = ...;

// query minimum/maximum value of the first image in storage domain
vec2 minmaxStorage = HistogramDataComponent::get(sis).minmax(0);

// query 16-bin intensity histogram of the first image for the intensity range of [0..100]
auto histogram = HistogramDataComponent::get(sis).derivedHistogram(0, 16, vec2(0, 100));
```
 fragment

**Note:** [HistogramDataComponent](/cpp-sdk/4.5/api/classes/imfusion-histogramdatacomponent) does not support serialization since it will always be lazy-instantiated during runtime.

**See also:** [Image](/cpp-sdk/4.5/api/classes/imfusion-image), [Pixel Value Domains](/cpp-sdk/4.5/api/guides/pixel-value-domains)

## Member Function Documentation

### histogram

```cpp
const SharedImage * ImFusion::HistogramDataComponent::histogram (int index) const
```

Returns the intensity histogram for the image with the given index not considering any masks.

The binning of the histogram is based on the image content. The min/max bin correspond to the min/max value of the underlying image (i.e. [minmax()](#minmax-1-2)). The number of bins determined by [maxNumBuckets()](#maxnumbuckets).

**Note:** This histogram will be cached by [HistogramDataComponent](/cpp-sdk/4.5/api/classes/imfusion-histogramdatacomponent).

### histogramMaskWeighted

```cpp
std::unique_ptr < SharedImage > ImFusion::HistogramDataComponent::histogramMaskWeighted (int index) const
```

Returns the intensity histogram for the image with the given index while considering masks.

The binning of the histogram is based on the image content. The min/max bin correspond to the min/max value of the underlying image (i.e. [minmax()](#minmax-1-2)). The number of bins determined by [maxNumBuckets()](#maxnumbuckets).

**Note:** Due to the volatility of the mask, the computed histogram can *not* be cached and will be recomputed every time it is requested.

### derivedHistogram

```cpp
std::shared_ptr < const SharedImage > ImFusion::HistogramDataComponent::derivedHistogram (int index, int numBins, const vec2 & intensityRange) const
```

Returns the intensity histogram with the given parameters for image with the given index.

The histogram is derived from the original full range histogram.

**Parameters:**

- `index` — Index of the image to compute the histogram for.
- `numBins` — Number of bins of the histogram to compute.
- `intensityRange` — Intensity range of the histogram to compute in storage value domain.

**Note:** This histogram will be cached (in an LRU cache) by [HistogramDataComponent](/cpp-sdk/4.5/api/classes/imfusion-histogramdatacomponent).

### maxNumBuckets

```cpp
int ImFusion::HistogramDataComponent::maxNumBuckets (int index) const
```

Returns the maximum number of buckets that makes sense for a histogram for the image with the given index.

For discrete numeric type images this matches the distance between the min and max value. However, it will never exceed the maximum supported 1D texture size, which is also the number of bins used for float images.

### computeAutoWindowing

```cpp
vec2 ImFusion::HistogramDataComponent::computeAutoWindowing (double lowerLimit, double upperLimit, int index, bool considerMask = false) const
```

Deduces optimal window/level (returned as vec2) settings the image with the given index based on its histogram.

**Parameters:**

- `lowerLimit` — Percentage of lower values removed by the auto windowing.
- `upperLimit` — Percentage of upper values removed by the auto windowing.
- `index` — Index of the image to compute the auto windowing for.
- `considerMask` — Flag whether to consider a mask attached to the image when computing the auto windowing.

### computeAutoWindowingParameters

```cpp
vec2 ImFusion::HistogramDataComponent::computeAutoWindowingParameters (double window, double level, int index, bool considerMask = false) const
```

Inverse computation of computeAutoWindowing.

Computes the needed lower/upper limit (returned as vec2) needed to deduce the auto windowing with the provided parameters.

**Parameters:**

- `level` — Windowing level to use.
- `index` — Index of the image to compute the inverse auto windowing for.
- `considerMask` — Flag whether to consider a mask attached to the image when computing the inverse auto windowing.
