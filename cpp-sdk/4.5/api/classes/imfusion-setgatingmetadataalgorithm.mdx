## Overview

[Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm) to compute gating information to an image set.

Two separate modes are supported: 1) If a [VitalsDataComponent](/cpp-sdk/4.5/api/classes/imfusion-vitalsdatacomponent) is available, a peak detection in either the ECG (preferred) or pulseoximetry signal is attempted. If at least two peaks are found, gating information is extrapolated to the entire clip. 2) Otherwise, at least two frames exhibiting the same point in time in the cycle (typically end-diastolic frames) need to be defined in p_endDiastolicFrames. A gating signal is extrapolated from these peaks to the entire clip.

The parameter p_phaseOffset is just written directly into the gating data component to help with manual and semiautomatic image registration.
