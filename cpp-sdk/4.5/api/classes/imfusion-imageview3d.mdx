`#include <ImFusion/GUI/ImageView3D.h>`

## Overview

Qt widget for 3d rendering of one or multiple images.

Serves as Qt wrapper of the [GlVolumeView](/cpp-sdk/4.5/api/classes/imfusion-glvolumeview) and [GlVolumeRenderer](/cpp-sdk/4.5/api/classes/imfusion-glvolumerenderer) classes for 3D volume rendering. Depending on the displayOptionsSource property of [ImageView3D](/cpp-sdk/4.5/api/classes/imfusion-imageview3d) the [DisplayOptions3d](/cpp-sdk/4.5/api/classes/imfusion-displayoptions3d) used during rendering are either individual for each image (stored in the images' DataComponents) or shared for all shown images (stored in [ImageView3D](/cpp-sdk/4.5/api/classes/imfusion-imageview3d)).

**Note:** Due to the potential external storage of display options, the Qt signals for changes of individual display options (i.e. windowChanged, levelChanged, alphaChanged are only emitted when using their corresponding Qt property setter. If the underlying [DisplayOptions3d](/cpp-sdk/4.5/api/classes/imfusion-displayoptions3d) have changed by somebody else in the background, this is not detected. If you need to be notified reliably in all occasions you need to register to the [DisplayOptions3d](/cpp-sdk/4.5/api/classes/imfusion-displayoptions3d) changed signal directly.

**Enumerator:**

- `View` — All visible data will be rendered using the same view's display options.
- `Data` — Every visible data will be rendered using its own display options.

### displayOptions

```cpp
const DisplayOptions3d & ImFusion::ImageView3D::displayOptions (const Data * data = nullptr) const
```

Return the DisplayOptions used for the given data wrt.

to the views' display options source setting. If the display options source is set to View, this function will return the view's display options. If the display options source is set to [Data](/cpp-sdk/4.5/api/classes/imfusion-data), this function will return the display options of *data*. If *data* is 0 but required, it will use the first visible data of this view instead.

### setImage

```cpp
void ImFusion::ImageView3D::setImage (SharedImage * image)
```

Convenience method to display a single image.

Internally, creates a container [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) holding *image* and then calls [setVisibleData()](/cpp-sdk/4.5/api/classes/imfusion-gui-view). [InteractiveView](/cpp-sdk/4.5/api/classes/imfusion-interactiveview) will not take ownership of *image*. If *image* is 0, will remove all data from this view.

****Deprecated**:** This function has problematic ownership semantics. Use [setVisibleData()](/cpp-sdk/4.5/api/classes/imfusion-gui-view) instead.

### setImage

```cpp
void ImFusion::ImageView3D::setImage (Image * image)
```

Convenience method to display a single image.

Internally, creates a container [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) holding *image* and then calls [setVisibleData()](/cpp-sdk/4.5/api/classes/imfusion-gui-view). [InteractiveView](/cpp-sdk/4.5/api/classes/imfusion-interactiveview) will not take ownership of *image*, it must stay alive as long as the image is shown. If *image* is 0, will remove all data from this view.

****Deprecated**:** This function has problematic ownership semantics. Use [setVisibleData()](/cpp-sdk/4.5/api/classes/imfusion-gui-view) instead.
