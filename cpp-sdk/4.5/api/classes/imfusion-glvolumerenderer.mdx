`#include <ImFusion/GL/GlVolumeRenderer.h>`

## Overview

Interface for a class that can render volumetric data into a [GlVolumeView](/cpp-sdk/4.5/api/classes/imfusion-glvolumeview).

Implement the [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) interface in order to provide an automatically generated [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui) for configuration when using the [InteractiveViewOptionsOverlay](/cpp-sdk/4.5/api/classes/imfusion-interactiveviewoptionsoverlay).

**See also:** [ImageView3D](/cpp-sdk/4.5/api/classes/imfusion-imageview3d), [GlVolumeView](/cpp-sdk/4.5/api/classes/imfusion-glvolumeview), [InteractiveViewOptionsOverlay](/cpp-sdk/4.5/api/classes/imfusion-interactiveviewoptionsoverlay), [EntryExitPointRenderer](/cpp-sdk/4.5/api/classes/imfusion-entryexitpointrenderer)

## Classes

- struct [RenderTextures](/cpp-sdk/4.5/api/structs/imfusion-glvolumerenderer-rendertextures) — Helper struct to encapsulate all input/output textures needed/used during direct volume rendering.

### supportsOITIntegration

```cpp
virtual bool ImFusion::GlVolumeRenderer::supportsOITIntegration () const
```

*virtual*

Returns whether this renderer supports the order-independent integration of semi-transparent scene geometry.

This flag is queried by [GlVolumeView](/cpp-sdk/4.5/api/classes/imfusion-glvolumeview) to determine whether blending OIT fragments is performed by the volume renderer (return true) or whether it should be done by the [GlVolumeView](/cpp-sdk/4.5/api/classes/imfusion-glvolumeview) prior to letting the [GlVolumeRenderer](/cpp-sdk/4.5/api/classes/imfusion-glvolumerenderer) render (return false). The default implementation returns false.

**See also:** GlOrderIndependentTransparency
