`#include <ImFusion/Reg/DemonsImageRegistration.h>`

## Overview

Registers two images with the Demons algorithm The algorithm modifies the [GlDenseDeformation](/cpp-sdk/4.5/api/classes/imfusion-gldensedeformation) object attached to the moving image A number of variants of the standard demons algorithm (diffeomorphic vs additive, various gradient modes) are supported (cf.

T. Vercauteren et al, " Diffeomorphic demons: efficient non-parametric image registration", Neuroimage (2009)).

Masking:

- If m_constrainDeformationByFixedImageMask is true:
  - If a mask is defined on fixed [m_image1](/cpp-sdk/4.5/api/classes/imfusion-abstractimageregistration), it will be converted to an explicit mask and applied to the deformation field. This ensures that the moving [m_image2](/cpp-sdk/4.5/api/classes/imfusion-abstractimageregistration) retains its original values wherever the mask is active.
  - If a mask is defined on the moving image ([m_image2](/cpp-sdk/4.5/api/classes/imfusion-abstractimageregistration)), this mask is only used during the demons force calculation.
  - During force calculation, the force at a pixel is set to zero if either the moving or fixed image is masked at that pixel.
  - In case of using an image set with multiple images as fixed (or moving), all images must have the same mask (respectively).
- If m_constrainDeformationByFixedImageMask is false:
  - Masks on fixed [m_image1](/cpp-sdk/4.5/api/classes/imfusion-abstractimageregistration) and moving [m_image2](/cpp-sdk/4.5/api/classes/imfusion-abstractimageregistration) are only used to constrain the forces calculated by computeDemonsForces to non-masked locations.
  - Masks are not considered during vector field flow (diffeomorphic demons) or pyramid upsampling.

## Member Enumeration Documentation

### GradientType

```cpp
enum class ImFusion::DemonsImageRegistration::GradientType
```

*strong*

The Demons image algorithm requires the approximation of a gradient.

We support a number of options here. cf. T. Vercauteren et al, " Diffeomorphic demons: efficient non-parametric image registration", Neuroimage (2009)

**Enumerator:**

- `REFERENCE` — Use the gradient of the (fixed) reference image. Efficient, as can be computed once per image level.
- `MOVING` — Use finite difference gradient of the moving image, including the deformation. Must be computed per iteration.
- `SYMMETRIC` — Symmetric combination of the above options.
