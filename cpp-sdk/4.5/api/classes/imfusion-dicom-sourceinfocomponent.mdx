`#include <ImFusion/Dicom/DicomDataComponents.h>`

## Overview

[DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) to store the mapping of [SharedImage](/cpp-sdk/4.5/api/classes/imfusion-sharedimage) to the DICOM URIs (such as file names) used for loading.

**Note:** This [DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) is intended to be attached to each element (i.e. [SharedImage](/cpp-sdk/4.5/api/classes/imfusion-sharedimage)) of a [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) individually and therefore usually found in the [ElementwiseDataComponentList](/cpp-sdk/4.5/api/classes/imfusion-elementwisedatacomponentlist).

## Member Function Documentation

### id

```cpp
std::string ImFusion::Dicom::SourceInfoComponent::id () const
```

*override, virtual*

Returns a unique string identifier for this type of data component.

**Note:** To avoid conflicts make sure to include potential plugin names into this ID. Due to the modular architecture of the ImFusion library, uniqueness of IDs cannot be checked at compile time. However, [DataComponentFactory](/cpp-sdk/4.5/api/classes/imfusion-datacomponentfactory) will check the uniqueness of the ID during registration.
