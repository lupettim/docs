## Overview

Convenience class for a shader implementing thick and stippled line rendering.

[LineRenderer](/cpp-sdk/4.5/api/classes/imfusion-gl-linerenderer) is a stateless object which you can use to render high-quality lines with custom width and stipple pattern. The rendering is configured by passing an [Options](/cpp-sdk/4.5/api/structs/imfusion-gl-linerenderer-options) instance to the render function. The default usage is to call one of [renderLines()](#renderlines), [renderLineStrip()](#renderlinestrip) or [renderLineLoop()](#renderlineloop) with a set of vertices in world space:

```cpp
// renders a 3px wide red line strip consisting of *verticesWorld* using the given view state
GL::LineRenderer::cachedInstance().renderLineStrip(verticesWorld,
 GL::LineRenderer::Options(vec4(1, 0, 0), 3),
                                                   view.state(),
                                                   view.orderIndependentTransparency());
```
 fragment
If your geometry is static you can optimize your render loop by calling [makeVertexData()](#makevertexdata-1-2) once to write the vertex data into a VBO of your choice, and then only call [render()](#render) inside the render loop.

This class inherits from [GL::InstanceManager](/cpp-sdk/4.5/api/classes/imfusion-gl-instancemanager) so that you can request a cached instance of it from any valid OpenGL context to save on resources.

**Note:** The functionality partially overlaps with [FixedFunctionPipeline](/cpp-sdk/4.5/api/classes/imfusion-gl-fixedfunctionpipeline) since both can render thick and stippled lines:
- [FixedFunctionPipeline](/cpp-sdk/4.5/api/classes/imfusion-gl-fixedfunctionpipeline) is general purpose and allows for directly rendering any vertex buffer. However, it requires geometry shader support which is absent in OpenGL ES environments.
- [LineRenderer](/cpp-sdk/4.5/api/classes/imfusion-gl-linerenderer) also works in OpenGL ES contexts as it only uses basic vertex and fragment shaders. However, it therefore requires some preprocessing of the input vertices on the CPU.

**See also:** [GL::VertexBuffer](/cpp-sdk/4.5/api/classes/imfusion-gl-vertexbuffer), [GL::FixedFunctionPipeline](/cpp-sdk/4.5/api/classes/imfusion-gl-fixedfunctionpipeline)

## Member Function Documentation

### makeVertexData

```cpp
bool ImFusion::GL::LineRenderer::makeVertexData (const std::vector < vec3 > & vertices, Primitive primitive, GL::VertexBuffer & outputVBO)
```

*static*

Converts the given vertices to a suitable internal representation and uploads them to a [VertexBuffer](/cpp-sdk/4.5/api/classes/imfusion-gl-vertexbuffer) which can then be used with [render()](#render).

**Parameters:**

- `vertices` — Vertices of the line in world coordinates
- `primitive` — Line type to render, must be Primitive::Lines, Primitive::LineLoop, or Primitive::LineStrip.
- `outputVBO` — Output vertex buffer into which the resulting vertex data should be written

**Returns:** true if sufficient input vertices were provided; false if *vertices* does not contain enough data or *primitive* is invalid, in which case *outputVBO* will remain untouched.
