## Overview

Median Filter.

**Note:** The median filter performs a sorting of the pixel neighborhood and assigns the median pixel value to the center pixel.

**Note:** In this implementation, an additional threshold can be defined and only pixels with a value below or above will be median filtered.

**Note:** Adapted from [http://graphics.cs.williams.edu/papers/MedianShaderX6/median.pix](http://graphics.cs.williams.edu/papers/MedianShaderX6/median.pix) and [http://graphics.cs.williams.edu/papers/MedianShaderX6/median5.pix](http://graphics.cs.williams.edu/papers/MedianShaderX6/median5.pix)

## Member Enumeration Documentation

### Submode

```cpp
enum ImFusion::GlMedianFilter::Submode
```

Filter submode.

**Enumerator:**

- `FILTER_PEPPER` — Apply the filter only to dark pixels that lie beneath the threshold. E.g. for removal of 'dead' pixels.
- `FILTER_SALT` — Apply the filter only to bright pixels that lie above the threshold. E.g. for removal of overdriven pixels.
- `FILTER_ALL` — Apply the filter only to pixels when the absolute intensity difference to the neighborhood median pixel lies above the threshold.
The threshold value is multiplied by a factor of 0.1 in the shader. Apply the filter to entire image. Doesn't take threshold into account.

### setKernelSize

```cpp
bool ImFusion::GlMedianFilter::setKernelSize (int size, double k)
```

*override, virtual*

**Parameters:**

- `size` — Size of quadratic kernel. Currently supported sizes are 3 and 5.
- `k` — Threshold value. Note that this is the slider value multiplied by 0.01

**Warning:** Filtering a 3D image using the 5x5x5 kernel is costly. Computation time is about 1min for a 512^3 image.
