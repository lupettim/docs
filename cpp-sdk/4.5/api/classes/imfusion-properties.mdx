## Overview

Storage container for serialization of arbitrary types, internally backed by strings.

[Properties](/cpp-sdk/4.5/api/classes/imfusion-properties) storage is hierarchical in the sense that a [Properties](/cpp-sdk/4.5/api/classes/imfusion-properties) instance can have any number of nested subproperties. Parameters of subproperties can be set and retrieved by a path, for example [param](#param-1-2)("Views/window", val) will retrieve the value for window in the subproperties Views. The same works with [setParam()](#setparam-1-2), which will always create all subproperties on the way. The path separator is always /, which must not be used in the naming of parameters or subproperties.

The following types are supported for parameter serialization:

- most fundamental types except void and long double
- [std::string](http://en.cppreference.com/w/cpp/string/basic_string.html) and const char\*
- [Filesystem::Path](/cpp-sdk/4.5/api/classes/imfusion-filesystem-path) and [Filesystem::Url](/cpp-sdk/4.5/api/classes/imfusion-filesystem-url)
- Most Eigen typedefs in Mat.h (vecX, vecXf, vecXi, matX, matXf in sizes 2-6)
- [std::vector](http://en.cppreference.com/w/cpp/container/vector.html) containers of the previous types
- [std::optional](http://en.cppreference.com/w/cpp/utility/optional.html) containers of the previous types (including [std::vector](http://en.cppreference.com/w/cpp/container/vector.html))
- [std::set](http://en.cppreference.com/w/cpp/container/set.html) and [std::unordered_set](http://en.cppreference.com/w/cpp/container/unordered_set.html) of fundamental types and string types

If you want to store a custom type in [Properties](/cpp-sdk/4.5/api/classes/imfusion-properties) make it implement the [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) interface, implement the pair of configure()/configuration() functions and then serialize the type into a subproperty.

**Note:** Since parameter serialization is implemented through explicit template specialization on the library side, using an unsupported type will yield a linker error instead of a compiler error.

**See also:** Object Serialization, [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable)

**Examples:** [DicomExtensionExample.cpp](/cpp-sdk/4.5/api/examples/dicom-extension-8cpp), [DicomExtensionExample3.cpp](/cpp-sdk/4.5/api/examples/dicom-extension3-8cpp), [ExampleAlgorithm.cpp](/cpp-sdk/4.5/api/examples/example-algorithm-8cpp), [ExampleAlgorithm.h](/cpp-sdk/4.5/api/examples/example-algorithm-8h), [Serialization.cpp](/cpp-sdk/4.5/api/examples/serialization-8cpp), [SqrtAlgorithm.cpp](/cpp-sdk/4.5/api/examples/sqrt-algorithm-8cpp), and [SqrtAlgorithm.h](/cpp-sdk/4.5/api/examples/sqrt-algorithm-8h).

## Member Enumeration Documentation

### Comparison

```cpp
enum class ImFusion::Properties::Comparison
```

*strong*

Bitflag enumeration to define which aspects to consider during comparison of two [Properties](/cpp-sdk/4.5/api/classes/imfusion-properties) objects.

**Enumerator:**

- `ValuesOnly` — Compare only the param values (will always be compared).
- `Order` — Compare the order of the parameters.
- `Defaults` — Compare the optional default values of the parameters.
- `Types` — Compare the parameter types.
- `Attributes` — Compare the optional parameter attributes.
- `SubProperties` — Recursively compare sub properties.
- `All` — Compare all aspects of the [Properties](/cpp-sdk/4.5/api/classes/imfusion-properties) object.

### CopyOptions

```cpp
enum class ImFusion::Properties::CopyOptions
```

*strong*

Bitflag enumeration to configure behavior of [copyFrom()](#copyfrom).

**Enumerator:**

- `Default` — Overwrite existing parameters but not existing subproperties.
- `ReplaceSubProperties` — Overwrite existing subproperties instead of adding new ones.
- `KeepType` — Keep the original ParamType when overwriting existing parameters.
- `KeepAttributes` — Keep the original attributes when overwriting existing parameters.
- `ExistingParamsOnly` — Only copy parameters that exist in the target. Implies ReplaceSubProperties.

### setParamAttributes

```cpp
bool ImFusion::Properties::setParamAttributes (const std::string & name, const std::vector < Attribute > & attributes)
```

Set special attributes to a single parameter.

The attributes are specified in the form: &lt;attribute&gt;: &lt;value&gt;, &lt;attribute&gt;: &lt;value&gt;. Whitespace in between the elements (e.g. around the :) is automatically trimmed. Attributes must not contain whitespace. The value can be either a number or a string. Strings must be quoted with single quotes (') and are allowed to contain ,.

The supported attributes depend on the type of the parameter. Currently supported attributes are:

- General:
  - label (string): text to use for the label when displaying this property in a default [PropertiesWidget](/cpp-sdk/4.5/api/classes/imfusion-propertieswidget).
  - useroption ('r', 'w', or 'rw'): if this is set for a parameter of an [IoAlgorithm](/cpp-sdk/4.5/api/classes/imfusion-ioalgorithm), the parameter will be displayed in the global settings dialog.
  - read-only (bool): if set, the user cannot change the value
  - tooltip (string): tooltip that describes the parameter
  - hidden (bool): if set, the [PropertiesWidget](/cpp-sdk/4.5/api/classes/imfusion-propertieswidget) will not show a control for this parameter
  - invalid (bool): if set, the control’s value is highlighted as invalid in the [PropertiesWidget](/cpp-sdk/4.5/api/classes/imfusion-propertieswidget).
  - Double, Integer:
  - min (double or int): minimum allowed value
  - max (double or int): maximum allowed value
  - step (double or int): step by which the value is increased
  - prefix (string): prefix displayed before the value (e.g. a unit)
  - suffix (string): suffix displayed after the value (e.g. a unit)
  - withSlider (bool): add a slider in the user interface
- Double:
  - decimals (int): number of decimals displayed. Setting this might alter min/max value (see [https://doc.qt.io/qt-5/qdoublespinbox.html#decimals-prop](https://doc.qt.io/qt-5/qdoublespinbox.html#decimals-prop))
- Enum, EnumString:
  - values (comma separated string, with or without whitespace around comma): [String](/cpp-sdk/4.5/api/namespaces/imfusion-string) representation of enumerators.
  - keys (comma separated string, with or without whitespace around comma): Integer values assigned to enumerators. If not specified, integer values 0,1,2,3,... are assumed.
- Path:
  - type: specifies the dialog type and must be one of: OpenDirectory, OpenFile or SaveFile. The default is OpenFile.
  - caption: sets the title of the dialog (the default is a generic caption depending on the type)
  - filter: string of wildcard patterns to filter available files, separated by ;;, e.g.: Text files (\*.txt);;XML files (\*.xml) (see also QFileDialog)
  - dontUseNativeDialog: when set to true use the Qt dialog instead of the native dialog. The default is false.
- [String](/cpp-sdk/4.5/api/namespaces/imfusion-string):
  - textBox (bool): if set, uses a text box instead of a line edit, making it more user-friendly for long or multi-line text. The default is false.
