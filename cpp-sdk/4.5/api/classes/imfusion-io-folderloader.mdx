## Member Enumeration Documentation

### Options

```cpp
enum class ImFusion::IO::FolderLoader::Options
```

*strong*

Bitflag enumeration of options to customize the behavior of [FolderLoader](/cpp-sdk/4.5/api/classes/imfusion-io-folderloader).

**Enumerator:**

- `None` — Use default options.
- `AllowSpawnGUI` — If set, allow for the loader to spawn a [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui).

### Status

```cpp
enum class ImFusion::IO::FolderLoader::Status
```

*strong*

Known codes indicating the status/result of the last call to [load()](#load).

**Enumerator:**

- `Success` — Loading was successful.
- `NotFound` — The specified path does not exist.
- `NotFolder` — Path exists, but points to something that is not a folder.
- `MissingAppController` — Contained workspaces, but no [ApplicationController](/cpp-sdk/4.5/api/classes/imfusion-applicationcontroller) was passed.
- `Error` — Other unknown error occurred when loading.
- `Unknown` — [load()](#load) was not called

## Member Function Documentation

### load

```cpp
OwningDataList ImFusion::IO::FolderLoader::load (const Filesystem::Path & folderPath, ApplicationController * appController, Flags < Options > loadOptions = Options::None)
```

Attempts to load the folder at *folderPath*.

Returns an [OwningDataList](/cpp-sdk/4.5/api/classes/imfusion-owningdatalist) with the loaded data if successful or an empty Optional on failure. After calling this function you can query details information on the result using status().

**Parameters:**

- `folderPath` — Path to the file to load on the filesystem.
- `appController` — Optional pointer to an [ApplicationController](/cpp-sdk/4.5/api/classes/imfusion-applicationcontroller) instance needed to enable workspace loading support.
- `loadOptions` — Enables you to customize the behavior or configure what file/directory types are accepted.
