## Overview

template&lt;class T&gt;

template&lt;bool isConst = false&gt;

class ImFusion::Threading::Protected&lt; T &gt;::Locked&lt; isConst &gt;
RAII lock guard that provides exclusive access to a synchronized resource.

This class combines a lock acquisition with access to the protected resource. The lock is acquired on construction and automatically released on destruction, following the RAII ([Resource](/cpp-sdk/4.5/api/namespaces/imfusion-resource) Acquisition Is Initialization) pattern.

[Locked](/cpp-sdk/4.5/api/classes/imfusion-threading-protected-locked) objects provide pointer-like semantics via operator\* and operator-&gt; to access the underlying resource while the lock is held.

**Template Parameters:** | U | The type of the resource being protected |
| --- | --- |

Usage example:

```cpp
Protected<std::vector<int>> syncVec;
{
 auto locked = syncVec.lock();  // Lock acquired here
    locked->push_back(42);        // Safe access while locked
}  // Lock automatically released here
```
 fragment

**Note:** This class is non-copyable and non-movable.

*inline*

Constructs a [Locked](/cpp-sdk/4.5/api/classes/imfusion-threading-protected-locked) instance and acquires the mutex.

**Parameters:**

- `value` — Shared pointer to the resource to protect. Must not be null.
- `sharedMutex` — Shared pointer to the mutex protecting the resource. Must not be null.

**Note:** The lock is acquired immediately upon construction and held until the [Locked](/cpp-sdk/4.5/api/classes/imfusion-threading-protected-locked) object is destroyed.
