## Member Function Documentation

### computeDescPadded

```cpp
void ImFusion::FFT::computeDescPadded (int dim, double paddingPercentage)
```

Compute the padded image for an [FFT](/cpp-sdk/4.5/api/classes/imfusion-fft) of dimension `dim`. The image type will be `FLOAT`.

**Parameters:**

- `paddingPercentage` — Set to e.g. 1.0 for 100% of padding. The image will be padded along \dim dimensions. Its size will be at least ``(1+paddingPercentage) of the original image since the dimensions have to be multiples of small prime numbers for efficient [FFT](/cpp-sdk/4.5/api/classes/imfusion-fft) computation.

**Note:** This requires `m_descIn` to be set.

## Member Data Documentation

### m_largestValidPrimeRank

```cpp
unsigned int ImFusion::FFT::m_largestValidPrimeRank
```

*protected*

A specific [FFT](/cpp-sdk/4.5/api/classes/imfusion-fft) implementation can only deal with images whose dimensions are a multiple of small primes.

The rank of the largest of those primes needs to be specified (e.g. rank 0 for 2, 1 for 3, 2 for 5, 3 for 7, 4 for 11, etc.).
