`#include <ImFusion/GUI/InteractiveView.h>`

## Overview

Abstract base class for a view that can be hosted by a [DisplayWidget](/cpp-sdk/4.5/api/classes/imfusion-displaywidget).

An [InteractiveView](/cpp-sdk/4.5/api/classes/imfusion-interactiveview) is a wrapper around a [GlView](/cpp-sdk/4.5/api/classes/imfusion-glview) adding a [ViewInteraction](/cpp-sdk/4.5/api/classes/imfusion-viewinteraction), overlays, as well as a set of properties and signals. Each view has a certain size on the screen which is determined by the layout in [DisplayWidget](/cpp-sdk/4.5/api/classes/imfusion-displaywidget). Before rendering the actual view content with [GlView](/cpp-sdk/4.5/api/classes/imfusion-glview), an [InteractiveView](/cpp-sdk/4.5/api/classes/imfusion-interactiveview) draws a rectangular border around the content. The color of the border can be set with setDrawBorder.

[InteractiveView](/cpp-sdk/4.5/api/classes/imfusion-interactiveview) takes care of automatically layouting the hosted overlays, which are always drawn on top of the view content. Overlays can be placed at different anchors (e.g. corners, edges) of the view. If several overlays reside on the same Anchor, the view will position them in a way that they don't overlap.

### sceneEvent

```cpp
virtual bool ImFusion::InteractiveView::sceneEvent (QEvent * qtEvent, const GUI::InputEvent & inputEvent)
```

*virtual*

Called by the [DisplayWidget](/cpp-sdk/4.5/api/classes/imfusion-displaywidget) if a user input event has happened on this view.

The default implementation will forward the event to the sub-objects in the following order:

- InteractiveOverlays
- InteractiveObjects
- [ViewInteraction](/cpp-sdk/4.5/api/classes/imfusion-viewinteraction)

  **Returns:** true if the event has been handled and should no longer be passed to other objects.

### handleOverlayEvent

```cpp
virtual bool ImFusion::InteractiveView::handleOverlayEvent (QEvent * qtEvent, const GUI::InputEvent & inputEvent)
```

*protected, virtual*

The default implementation will call [InteractiveOverlay::sceneEvent()](/cpp-sdk/4.5/api/classes/imfusion-interactiveoverlay) and set m_mouseEventTarget/m_touchEventTarget if needed.

Furthermore, it will evaluate m_lastMouseEventOverlay and post enter/leave events to the overlays. Returns true if the event affects any overlay, false otherwise.

### handleObjectEvent

Auxiliary method for forwarding the scene event to all registered objects.

### handleInteractionEvent

```cpp
virtual bool ImFusion::InteractiveView::handleInteractionEvent (QEvent * qtEvent, const GUI::InputEvent & inputEvent)
```

*protected, virtual*

Auxiliary method for forwarding the scene event to the view interaction.

The default implementation will call [ViewInteraction::sceneEvent()](/cpp-sdk/4.5/api/classes/imfusion-interaction) and set m_mouseEventTarget if needed. It will also handle the ContextMenuEvent. Returns true if the event affects any overlay, false otherwise.
