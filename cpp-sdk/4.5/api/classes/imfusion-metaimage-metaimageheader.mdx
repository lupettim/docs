## Member Enumeration Documentation

### ByteOrder

```cpp
enum class ImFusion::MetaImage::MetaImageHeader::ByteOrder
```

*strong*

**Enumerator:**

- `BigEndian` — the most significant byte (MSB) in a multi-byte structure is stored first
- `LittleEndian` — the least significant byte (LSB) in a multi-byte structure is stored first

### load

```cpp
bool ImFusion::MetaImage::MetaImageHeader::load (const std::string & mhdFileName, bool strictSyntax = false, bool ignoreHalfPixelOffset = false)
```

Load a [MetaImage](/cpp-sdk/4.5/api/namespaces/imfusion-metaimage) header.

**Parameters:**

- `filename` — The absolute path to the [MetaImage](/cpp-sdk/4.5/api/namespaces/imfusion-metaimage) file. The extension of the given filename, either '.mhd' or '.mha', determines the type of the [MetaImage](/cpp-sdk/4.5/api/namespaces/imfusion-metaimage) file.
- `strictSyntax` — determines whether or not to fail when unexpected data is encountered (such as trailing non-whitespace characters after numbers).
- `ignoreHalfPixelOffset` — should be set if no half-pixel offset to the Position/Orientation provided should be applied. Returns true on success.

### save

```cpp
bool ImFusion::MetaImage::MetaImageHeader::save (const std::string & mhdFileName) const
```

Save a [MetaImage](/cpp-sdk/4.5/api/namespaces/imfusion-metaimage) header file.

Automatically appends .mhd extension if not present. Automatically creates the directory the file is in if not present. Does not save intensity shift+scale parameters if m_keepShiftAndScale is not set. Removes half-pixel offset from matrix when saving Position/Orientation Returns true on success.

### loadSharedImageSet

```cpp
std::unique_ptr < SharedImageSet > ImFusion::MetaImage::MetaImageHeader::loadSharedImageSet (bool prepare = false, SharedImageSet * outContainer = nullptr, bool * outSuccess = nullptr, Progress * prog = nullptr, const std::string & mhdFileName = "")
```

*protected*

Load [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) content, return zero pointer on failure.

Multi-file 3D data is interpreted as one slice per file. Optionally the data will be prepared (i.e. normalized) after loading, and a custom derived data container instance will be used if provided.

**Parameters:**

- `prepare` — If (prepare || m_msb) call [MemImage::prepare](/cpp-sdk/4.5/api/classes/imfusion-memimage).

### saveSharedImageSet

```cpp
bool ImFusion::MetaImage::MetaImageHeader::saveSharedImageSet (const SharedImageSet * is, std::unique_ptr < const MemImage > & unscaledImageBuffer) const
```

*protected*

Save selected [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) content (without header).

Location to save in is taken from m_rawFiles. If is.size() &gt; 1, then multifile mode is forced, we do not currently support multi-frame single-file [MetaImage](/cpp-sdk/4.5/api/namespaces/imfusion-metaimage) files.

**Parameters:**

- `unscaledImageBuffer` — should point to the result of img-&gt;unscaled() if m_keepScaleAndOffset is false where img is the first selected image.
