`#include <ImFusion/Base/DisplayOptions3d.h>`

## Overview

Configures the display options for 3D volume rendering of images.

3D display options are mainly consumed by [GlVolumeRenderer](/cpp-sdk/4.5/api/classes/imfusion-glvolumerenderer) which implements 3D rendering of image data. Usually, [DisplayOptions3d](/cpp-sdk/4.5/api/classes/imfusion-displayoptions3d) are attached as a [DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) to the image for which they define the rendering options. However, classes like [ImageView3D](/cpp-sdk/4.5/api/classes/imfusion-imageview3d) may use special (e.g. view-specific) display options for rendering their images and therefore use the same interface.

**Note:** Window/level parameters are defined in original pixel value domain (e.g. Hounsfield Units). Conversion of pixel values to display intensities is performed using the LINEAR_EXACT VOI LUT function as described in section C.11.2.1.3.2 of the DICOM standard. [http://dicom.nema.org/medical/dicom/current/output/chtml/part03/sect_C.11.2.html#sect_C.11.2.1.3](http://dicom.nema.org/medical/dicom/current/output/chtml/part03/sect_C.11.2.html#sect_C.11.2.1.3)

**See also:** [GlVolumeRenderer](/cpp-sdk/4.5/api/classes/imfusion-glvolumerenderer), [ImageView3D](/cpp-sdk/4.5/api/classes/imfusion-imageview3d), [DisplayOptions2d](/cpp-sdk/4.5/api/classes/imfusion-displayoptions2d)

## Member Enumeration Documentation

### MaskingMode

```cpp
enum class ImFusion::DisplayOptions3d::MaskingMode
```

*strong*

Enumeration of options for how masked image pixels should be rendered.

**Enumerator:**

- `Ignore` — Ignore the mask, render the pixels normally.
- `Discard` — Discard masked pixels (where the mask value is 0).
- `MultiplicativeBlend` — Multiply masked pixels with the maskingColor() (where the mask value is 0).
- `DiscardZerosMultiplicativeOnes` — Discard pixels where the mask value is 0, multiply pixels with maskingColor() where the mask value is 1.
