`#include <ImFusion/GUI/DataDisplayModel.h>`

## Overview

This class extends a [DataItemModel](/cpp-sdk/4.5/api/classes/imfusion-dataitemmodel) with data visibility information from a [DisplayWidgetMulti](/cpp-sdk/4.5/api/classes/imfusion-displaywidgetmulti).

Instances of this class wrap a [DataItemModel](/cpp-sdk/4.5/api/classes/imfusion-dataitemmodel) and add one additional column. The columns are added at the end on each rows and do not override any existing columns. The DataVisibleRole of a column represent if the data in a row is visible in the [DisplayWidgetMulti](/cpp-sdk/4.5/api/classes/imfusion-displaywidgetmulti) or not. This is also represented by the Qt::CheckStateRole.

**Note:** While this behaves like a QAbstractProxyModel it cannot be implemented as one because QAbstractProxyModel is expected to have a 1:1 mapping to the source model. Appending a column at the end would work but adding it in the front breaks editing.

## Member Function Documentation

### data

```cpp
QVariant ImFusion::DataDisplayModel::data (const QModelIndex & index, int role = Qt::DisplayRole) const
```

*override*

Returns true or false for DataVisibleRole for the additional columns.

The CheckStateRole of an additional column will returned Checked if the DataVisibleRole is true or false otherwise. All other columns are directed to the source model. The [DataItemModel::DataRole](/cpp-sdk/4.5/api/classes/imfusion-dataitemmodel) for the additional columns will return the Data\* represented by the row.

### setData

```cpp
bool ImFusion::DataDisplayModel::setData (const QModelIndex & index, const QVariant & value, int role = Qt::EditRole)
```

*override*

Adds or removes the [Data](/cpp-sdk/4.5/api/classes/imfusion-data) from the Display if role is DataVisibleRole.

If value is true the [Data](/cpp-sdk/4.5/api/classes/imfusion-data) is added to the Display, otherwise it's removed. The CheckStateRole will be redirected to the DataVisibleRole and the value changed to true if it was Qt::Checked or false for Qt::Unchecked.
