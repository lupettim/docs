`#include <ImFusion/GUI/View.h>`

## Overview

[Base](/cpp-sdk/4.5/api/structs/imfusion-dicom-base) interface for a rendering view (e.g.

single anatomical plane) that can be hosted inside a [Display](/cpp-sdk/4.5/api/classes/imfusion-gui-display).

A [View](/cpp-sdk/4.5/api/classes/imfusion-gui-view) takes care of rendering a scene consisting of any number of compatible datasets, scene objects and overlays, and of receiving and dispatching incoming input events to the individual elements. This is achieved by combining the following interfaces:

- It wraps one [GlView](/cpp-sdk/4.5/api/classes/imfusion-glview) that implements the actual rendering of the datasets using OpenGL.
- Any number of [ViewObject](/cpp-sdk/4.5/api/classes/imfusion-gui-viewobject) instances that describe additional objects of the rendered scene. In contrast to the plain [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) that can be attached directly to the [GlView](/cpp-sdk/4.5/api/classes/imfusion-glview), the [ViewObject](/cpp-sdk/4.5/api/classes/imfusion-gui-viewobject) supports attaching input event handlers so that the user can interact with the scene object.
- Any number of [ViewOverlay](/cpp-sdk/4.5/api/classes/imfusion-gui-viewoverlay) instances that are rendered on top of the scene. They reside in viewport space and are automatically layouted wrt. to their assigned anchor in a fashion that they don't overlap.

The generic interface to display datasets inside a [View](/cpp-sdk/4.5/api/classes/imfusion-gui-view) is [setVisibleData()](#setvisibledata). The default implementation will use the [DataDisplayHandler](/cpp-sdk/4.5/api/classes/imfusion-datadisplayhandler) interface to determine if and how dataset can be is visualized in a concrete view type. A [DataDisplayHandler](/cpp-sdk/4.5/api/classes/imfusion-datadisplayhandler) will then usually forward to member functions of the concrete [GlView](/cpp-sdk/4.5/api/classes/imfusion-glview) or create a suitable [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) renderer instance and register it with the view.

Usually, the viewport() size of a [View](/cpp-sdk/4.5/api/classes/imfusion-gui-view) is automatically determined by the layouting algorithm of the parent [Display](/cpp-sdk/4.5/api/classes/imfusion-gui-display) from which it also receives the input events. Incoming input events are dispatched in the following logic until one [EventResult](/cpp-sdk/4.5/api/structs/imfusion-gui-eventresult) indicates to stop propagation:

1. all registered overlays() in reverse order
2. all registered [objects()](#objects) in reverse order
3. all registered eventHandlers() in original order

You can optionally define a background color, as well as a rectangular border that should be rendered around the content.


Set the list of datasets that should be shown in this view.

The default implementation relies on [DataDisplayHandler](/cpp-sdk/4.5/api/classes/imfusion-datadisplayhandler) interface to determine if and how a [Data](/cpp-sdk/4.5/api/classes/imfusion-data) instance is visualized in this view. ### updateEventFocus

```cpp
void ImFusion::GUI::View::updateEventFocus (const InputEvent & event, const void * newTarget)
```

*protected*

Update the internal bookkeeping of "focus" event handlers:

- If *event* is a mouse press event, m_mouseEventFocusObject is set to *newTarget*.
- If *event* is a mouse release event, m_mouseEventFocusObject is set to nullptr.
- If *event* is a touch begin event, m_touchEventFocusObject is set to *newTarget*.
- If *event* is a touch end event, m_touchEventFocusObject is set to nullptr.

  **See also:** [filterEventFocus()](#filtereventfocus)

### filterEventFocus

```cpp
bool ImFusion::GUI::View::filterEventFocus (const InputEvent & event, const void * target) const
```

*protected*

Checks if the object *target* should receive the given *event* based on the event type and current "focus" object.

- Returns true if current focus object is nullptr or matches *target*.
- Returns true if *event* is not a mouse-move or touch-update event.
- Returns false if *event* is a mouse-move or touch-update event and *target* does not match the current focus object.

  **See also:** [updateEventFocus()](#updateeventfocus)

Pointer to the event receiver that shall receive [TouchEvent::Type::Update](/cpp-sdk/4.5/api/classes/imfusion-gui-touchevent) events exclusively.

The default implementation will set this to any object/overlay/interaction that handles ([EventResult::stopPropagation](/cpp-sdk/4.5/api/structs/imfusion-gui-eventresult) == true) a TouchBegin event. The default implementation will reset this to nullptr on any [TouchEvent::Type::End](/cpp-sdk/4.5/api/classes/imfusion-gui-touchevent) or [TouchEvent::Type::Cancel](/cpp-sdk/4.5/api/classes/imfusion-gui-touchevent) event.
