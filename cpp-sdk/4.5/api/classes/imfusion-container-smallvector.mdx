## Overview

template&lt;typename T, [std::size_t](http://en.cppreference.com/w/cpp/types/size_t.html) N = 8&gt;

class ImFusion::Container::SmallVector&lt; T, N &gt;
A small-buffer-optimized variant of [std::vector](http://en.cppreference.com/w/cpp/container/vector.html) that stores up to N elements without heap allocation.

When the number of elements grows beyond N a heap allocation is used so that [SmallVector](/cpp-sdk/4.5/api/classes/imfusion-container-smallvector) behaves like a regular [std::vector](http://en.cppreference.com/w/cpp/container/vector.html). Use this container when the number of elements is usually small and short-lived, but can occasionally grow dynamically. Typical examples are temporary working sets, per-frame buffers, or collections built in tight loops where avoiding frequent heap allocations improves performance.

Prefer [SmallVector](/cpp-sdk/4.5/api/classes/imfusion-container-smallvector) over [std::vector](http://en.cppreference.com/w/cpp/container/vector.html) if:

- Most instances contain only a few elements (often &lt;= N), and
- You want to reduce allocator traffic and improve locality.

Prefer [std::vector](http://en.cppreference.com/w/cpp/container/vector.html) if:

- Sizes are typically large or highly variable, or
- You need full STL API compatibility beyond the subset provided here.

Example:

```cpp
// The majority of use cases have a maximum of 4 channels, in this case no heap allocation is performed.
Container::SmallVector<double, 4> channelData;
for (int c = 0; c < img.channels(); ++c)
    channelData.push_back(img.value(x, y, z, c));

processChannels(channelData);
```

| N | Maximum number of elements stored inline without heap allocation |
