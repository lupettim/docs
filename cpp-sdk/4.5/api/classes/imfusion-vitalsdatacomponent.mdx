## Overview

[DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) for a collection of time dependent vital signs like ECG, heart rate or pulse oximeter measurements.

## Classes

- struct [Sample](/cpp-sdk/4.5/api/structs/imfusion-vitalsdatacomponent-sample) — A single temporal sample.
- class [TimeSeries](/cpp-sdk/4.5/api/classes/imfusion-vitalsdatacomponent-timeseries) — A scalar time series.

## Public Types

- enum class Kind : int &#123; **ECG** , **PulseOximeter** , **HeartRate** , **Other** &#125; — Type of vital signs represented.
- using **TimePoint** = std::chrono::system_clock::time_point
- using **Duration** = TimePoint::duration

## Member Function Documentation

### id

```cpp
std::string ImFusion::VitalsDataComponent::id () const
```

*override, virtual*

Returns a unique string identifier for this type of data component.

**Note:** To avoid conflicts make sure to include potential plugin names into this ID. Due to the modular architecture of the ImFusion library, uniqueness of IDs cannot be checked at compile time. However, [DataComponentFactory](/cpp-sdk/4.5/api/classes/imfusion-datacomponentfactory) will check the uniqueness of the ID during registration.
