### registerPointsOnData

```cpp
template<class DataType, class PodType, class ManipulatorType> void ImFusion::PointCorrespondencesAlgorithm::registerPointsOnData (int priority = 1)
```

*inline, static*

Registers custom [PointsOnData](/cpp-sdk/4.5/api/classes/imfusion-pointsondata) and manipulator types into the factory.

The priority parameter determines which factory will be used when multiple factories are available. Higher priority values mean the factory will be used first. In case of equal priorities, the factory registered first will be used. As a general suggestion, priority should be set to the inheritance depth of the class, with 1 being a class that directly inherits from [Data](/cpp-sdk/4.5/api/classes/imfusion-data), 2 a class that inherits from B, where B inherits directly from [Data](/cpp-sdk/4.5/api/classes/imfusion-data), and so on.

**Template Parameters:** | DataType | Derived type of base class [Data](/cpp-sdk/4.5/api/classes/imfusion-data) the registered [PointsOnData](/cpp-sdk/4.5/api/classes/imfusion-pointsondata) and [PointsOnDataManipulator](/cpp-sdk/4.5/api/classes/imfusion-pointsondatamanipulator) should be used for |
| --- | --- |
| PodType | Must be of type [PointsOnData](/cpp-sdk/4.5/api/classes/imfusion-pointsondata) that takes data of type DataType as construction parameter |
| [Manipulator](/cpp-sdk/4.5/api/classes/imfusion-manipulator) | Must be of type [PointsOnDataManipulator](/cpp-sdk/4.5/api/classes/imfusion-pointsondatamanipulator) that takes [PointsOnData](/cpp-sdk/4.5/api/classes/imfusion-pointsondata) of type PodType as construction parameter |
