## Overview

Tiling-based layouting class for arranging Views inside a [Display](/cpp-sdk/4.5/api/classes/imfusion-gui-display) in a non-overlapping fashion.

This class serves as runtime representation to configure the placement and size of views inside a [DisplayBase](/cpp-sdk/4.5/api/classes/imfusion-gui-displaybase). It does so by recursively subdividing the available viewport either horizontally or vertically. Each tile is then consumed by exactly one [View](/cpp-sdk/4.5/api/classes/imfusion-gui-view). Furthermore, it provides a handleInputEvent() function so that users can interactively change the layout with their mouse.

The [DisplayLayout](/cpp-sdk/4.5/api/classes/imfusion-gui-displaylayout) uses a local relative coordinate system where all edge positions are defined in [0..1] range independent of the actual pixel viewport size of the parent [Display](/cpp-sdk/4.5/api/classes/imfusion-gui-display).

**Note:** Manually specifying a [DisplayLayout](/cpp-sdk/4.5/api/classes/imfusion-gui-displaylayout) through a series of recursive subdivisions can be unintuitive and cumbersome. We recommend using the [DisplayLayoutConfig](/cpp-sdk/4.5/api/classes/imfusion-gui-displaylayoutconfig) class instead. It describes the layout in a table/grid-like fashion and is much simpler to manually specify. Use loadConfiguration() and [retrieveConfiguration()](#retrieveconfiguration) to convert between the two.

**See also:** [DisplayBase](/cpp-sdk/4.5/api/classes/imfusion-gui-displaybase), [DisplayLayoutConfig](/cpp-sdk/4.5/api/classes/imfusion-gui-displaylayoutconfig), [DisplayAutoLayouter](/cpp-sdk/4.5/api/classes/imfusion-gui-displayautolayouter)

## Member Function Documentation

### retrieveConfiguration

```cpp
virtual DisplayLayoutConfig ImFusion::GUI::DisplayLayout::retrieveConfiguration ()
```

*virtual*

Generates a layout configuration from the current layout.

This is not necessarily the same configuration that was loaded before. More specifically the configuration is generated by creating one row/column per edge and won't contain any sub-configuration.

### resizeView

```cpp
virtual void ImFusion::GUI::DisplayLayout::resizeView (GUI::View * view, float ratio, Edge side, bool split)
```

*virtual*

Moves the specified edge of a view.

**Parameters:**

- `view` — the view to be moved
- `ratio` — the new relative position of the edge defined by side
- `side` — the edge that should be moved
- `split` — if true the edge is split (if possible) before moving
