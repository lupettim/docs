## Member Enumeration Documentation

### OverlayType

```cpp
enum ImFusion::GUI::RadiologyViewOverlays::OverlayType
```

Enumeration to select default overlays to add to a view.

**Enumerator:**

- `FullscreenButton` — Button to maximize/minimize the view; requires [DisplayAutoLayouter](/cpp-sdk/4.5/api/classes/imfusion-gui-displayautolayouter).
- `ViewOrderButton` — Button to move the view to a different position in the view layout; requires [DisplayAutoLayouter](/cpp-sdk/4.5/api/classes/imfusion-gui-displayautolayouter).
- `DataSelectionButton` — Button to select the visible data from the [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel).
- `ViewOptionsButton` — Button to show a popup for configuring view options.
- `DisplayOptionsButton` — Button to show a popup for configuring display options.
- `PatientInfo` — Text overlay showing patient information.
- `HorizontalScale` — Horizontal ruler overlay showing the image scale; only applicable to SliceViews.
- `VerticalScale` — Vertical ruler overlay showing the image scale; only applicable to SliceViews.
- `OrientationMesh` — Overlay showing the orientation of the shown data (e.g. in form of a cube).
- `Histogram` — Shows the intensity distribution of the shown image as histogram; only applicable to SliceViews.
- `PixelInfo` — Shows image pixel information underneath the mouse pointer; only applicable to SliceViews.
- `FpsCounter` — Shows an estimate of the frames per second of the renderer.
- `OffscreenIndicator` — Dynamic overlay showing data that is outside of the current view area; only applicable to SliceViews.

### OrientationMeshType

```cpp
enum class ImFusion::GUI::RadiologyViewOverlays::OrientationMeshType
```

*strong*

Enumeration of available meshes to show in the OrientationOverlay.

**Enumerator:**

- `Cube` — Shows a cube where the faces have the letters A/P/L/R/F/H.
- `Head` — Shows a depiction of a human head.
- `Body` — Shows a depiction of a human body.
- `CubeQuadruped` — Shows a cube where the faces have the letters D/V/LE/RT/RC/CD.

### RadiologyViewOverlays

```cpp
ImFusion::GUI::RadiologyViewOverlays::RadiologyViewOverlays (DisplayBase * display, DisplayAutoLayouter * autoLayouter, DataModel * dataModel)
```

Creates a new [RadiologyViewOverlays](/cpp-sdk/4.5/api/classes/imfusion-gui-radiologyviewoverlays) instance.

All parameters are optional for this class in general but required for certain overlay types.

**Parameters:**

- `display` — Pointer to the parent display, required for the ViewOptionsButton and DisplayOptionsButton to be available.
- `autoLayouter` — Pointer to the [DisplayAutoLayouter](/cpp-sdk/4.5/api/classes/imfusion-gui-displayautolayouter) of the parent display, required for the FullscreenButton and ViewOrderButton to be available.
- `dataModel` — Pointer to an [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel) used by the parent application, required for the DataSelectionButton to be available.

### setPatientInfoFormatString

```cpp
void ImFusion::GUI::RadiologyViewOverlays::setPatientInfoFormatString (const std::string & formatString)
```

Defines the content of the PatientInfo overlay based on a format string.

The format string can contain placeholders using the syntax &#123;placeholder&#125;, where "placeholder" is one of the following: patient_name, birthdate, study_date, study_time, modality, patient_sex.

Example (also the default formatting string after construction):

```cpp
setPatientInfoFormatString("{modality}: {patient_name} ({patient_sex}) - {study_date} {study_time}")
```
