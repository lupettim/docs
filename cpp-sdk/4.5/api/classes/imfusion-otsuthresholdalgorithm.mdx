`#include <ImFusion/Base/OtsuThresholdAlgorithm.h>`

## Overview

Computes the Otsu threshold of an image.

In Mode::CenterAtZero, the algorithm subtracts the computed threshold from a float-converted copy of the input so that the threshold is centered at zero. In Mode::BinaryMask, it returns a binary mask with value 1 for pixels above or equal to the threshold and 0 otherwise.

In Mode::CenterAtZero, the output keeps the input modality. In Mode::BinaryMask, the output modality is set to [Data::LABEL](/cpp-sdk/4.5/api/classes/imfusion-data). The input must be a single-frame, single-channel image of any scalar pixel type.

A public static helper is also provided to compute the Otsu threshold directly from a valid image.

**Note:** Sets status to [Status::Error](/cpp-sdk/4.5/api/classes/imfusion-algorithm) if the input has no intensity variation (constant image), since no meaningful threshold can be computed in that case.

### computeOtsuThreshold

```cpp
double ImFusion::OtsuThresholdAlgorithm::computeOtsuThreshold (const SharedImageSet & image, int numBins = 256)
```

*static*

Computes the Otsu threshold of a valid image.

The threshold is returned in the original pixel value domain.

**Parameters:**

- `image` — Input image. Must contain exactly one single-channel image with non-zero intensity range.
- `numBins` — Number of histogram bins used to estimate the threshold. Must be at least 2.

**Returns:** Otsu threshold in the original pixel value domain.
