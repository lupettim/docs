`#include <ImFusion/Core/Threading/StoppableThread.h>`

## Overview

[Extension](/cpp-sdk/4.5/api/classes/imfusion-dicom-extension) of a [std::thread](http://en.cppreference.com/w/cpp/thread/thread.html) to allow for signalling the thread to abort its execution.

This class adds functionality for cooperative abortion of a running thread similar to the interface of boost::thread and the upcoming std::jthread: Other threads can signal the [StoppableThread](/cpp-sdk/4.5/api/classes/imfusion-threading-stoppablethread) that they request abortion by calling [requestStop()](#requeststop). In order for this to work the [StoppableThread](/cpp-sdk/4.5/api/classes/imfusion-threading-stoppablethread) must regularly check during its execution, whether abortion was requested using isStopRequested(), and if so stop its execution.

Example:

```cpp
StoppableThread stoppableThread([]() {
 while (true) { // continuously running loop performing computations
        [...]

 // periodically check whether abortion was requested
 if (StoppableThread::isStopRequested())
 break;
    }
});

// signal the thread to stop
stoppableThread.requestStop();
// should join in finite time
stoppableThread.join();
```
 fragment

**Note:** For lifetime reasons, a [StoppableThread](/cpp-sdk/4.5/api/classes/imfusion-threading-stoppablethread) can not be detached and will always implicitly join on destruction.

- **StoppableThread** (StoppableThread &&)=delete — [StoppableThread](/cpp-sdk/4.5/api/classes/imfusion-threading-stoppablethread) is not movable because we can't notify the running thread that his stopSignal ref has changed.

### requestStop

```cpp
void ImFusion::Threading::StoppableThread::requestStop (bool stop = true)
```

Signal to the thread that it should abort its execution.

The thread needs to check regularly for this signal in order for this to work. If you set *stop* to false you can reset the abortion flag in case future computations should no longer be aborted.

### join

```cpp
void ImFusion::Threading::StoppableThread::join ()
```

Wait for the thread to finish its execution.

Calling [join()](#join) on a non-joinable thread will have no effect.

**Note:** Calling this function will *not* request the thread to stop.
