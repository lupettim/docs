## Overview

[DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) storing general information on the image origin.

Modeled after the DICOM patient-study-series hierarchy, it stores information on the patient, study and series the data set belongs to.

- enum class PatientSex &#123; **Unknown** , **Male** , **Female** , **Other** &#125; — Gender of the patient.

- enum class AnatomicalOrientationType &#123; **Unknown** , **Biped** , **Quadruped** &#125; — DICOM Anatomical Orientation Type.

## Member Function Documentation

### id

```cpp
std::string ImFusion::ImageInfoDataComponent::id () const
```

*override, virtual*

Returns a unique string identifier for this type of data component.

**Note:** To avoid conflicts make sure to include potential plugin names into this ID. Due to the modular architecture of the ImFusion library, uniqueness of IDs cannot be checked at compile time. However, [DataComponentFactory](/cpp-sdk/4.5/api/classes/imfusion-datacomponentfactory) will check the uniqueness of the ID during registration.
