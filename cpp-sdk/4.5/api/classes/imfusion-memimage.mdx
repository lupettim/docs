## Overview

Abstract base class for an image residing in main memory.

Use the templated [TypedImage](/cpp-sdk/4.5/api/classes/imfusion-typedimage) to instantiate a MemImage for a concrete pixel type.

**See also:** [Using Image Data](/cpp-sdk/4.5/api/guides/image-data-tutorial), [ImageProcessing](/cpp-sdk/4.5/api/namespaces/imfusion-imageprocessing)

**Examples:** [SqrtAlgorithm.cpp](/cpp-sdk/4.5/api/examples/sqrt-algorithm-8cpp).

## Member Enumeration Documentation

### Ownership

```cpp
enum class ImFusion::MemImage::Ownership
```

*strong*

Enumeration to describe image data pointer ownership.

**Enumerator:**

- `NotOwning` — [MemImage](/cpp-sdk/4.5/api/classes/imfusion-memimage) will not take ownership of the pointer, pointer must remain valid for the lifetime of the image.
- `Owning` — [MemImage](/cpp-sdk/4.5/api/classes/imfusion-memimage) will take ownership of the pointer, will delete it on destruction.

### create


*static*

Factory method to instantiate a [MemImage](/cpp-sdk/4.5/api/classes/imfusion-memimage) from an [ImageDescriptor](/cpp-sdk/4.5/api/structs/imfusion-imagedescriptor).

**Note:** This method does *not* initialize the underlying buffer Will internally dispatch to [TypedImage&lt;T&gt;::create()](/cpp-sdk/4.5/api/classes/imfusion-typedimage) based on the pixel type of *descriptor*.

### create

```cpp
std::unique_ptr < MemImage > ImFusion::MemImage::create (const ImageDescriptor & descriptor, void * data, Ownership own = Ownership::NotOwning)
```

*static*

Factory method to instantiate a [MemImage](/cpp-sdk/4.5/api/classes/imfusion-memimage) from an [ImageDescriptor](/cpp-sdk/4.5/api/structs/imfusion-imagedescriptor) and pointer to existing image data.

Will internally dispatch to [TypedImage&lt;T&gt;::create()](/cpp-sdk/4.5/api/classes/imfusion-typedimage) based on the pixel type of *descriptor*.

**Parameters:**

- `descriptor` — [ImageDescriptor](/cpp-sdk/4.5/api/structs/imfusion-imagedescriptor) of the new image.
- `data` — Pointer to image data, must match the descriptor layout.
- `own` — Flag whether the newly created image shall take ownership of *data*.

### valueDouble

```cpp
virtual double ImFusion::MemImage::valueDouble (int x, int y, int z = 0, int c = 0) const
```

*pure virtual*

Return the pixel intensity at the coordinate without interpolation casted to double.

**Note:** Try to avoid calling this function repeatedly because the coordinate-to-index computation is slow. When accessing entire regions of the image prefer use the index-based overload incrementing the index by a suitable stride after each function call instead of always fully recomputing the index.

### valueDouble

```cpp
virtual double ImFusion::MemImage::valueDouble (double x, double y, double z = 0, int c = 0, bool clamp = true, bool * valid = nullptr) const
```

*pure virtual*

Return the pixel intensity at the given coordinate using linear interpolation casted to double.

**Note:** Be aware that this function is rather slow, use it only if you really need interpolation. Consider using the set of free functions in [Math::Interpolation](/cpp-sdk/4.5/api/namespaces/imfusion-math-interpolation) as they provide more control on the interpolation behavior.
