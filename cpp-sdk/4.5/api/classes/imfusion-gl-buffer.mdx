## Overview

Wrapper class for OpenGL [Buffer](/cpp-sdk/4.5/api/classes/imfusion-gl-buffer) Objects.

OpenGL buffers serve many different purposes of which this base class implements the common interface. However, this class is not meant to be used directly. Instead you should use one of the specializations such as [VertexBuffer](/cpp-sdk/4.5/api/classes/imfusion-gl-vertexbuffer), [ShaderStorageBuffer](/cpp-sdk/4.5/api/classes/imfusion-gl-shaderstoragebuffer), etc. instead for improved type safety. Furthermore, each specialization provides an extended API that is tailored to its buffer type.

**Note:** As with most OpenGL objects, this class is bind-sensitive: You need to call bind() to bind a [Buffer](/cpp-sdk/4.5/api/classes/imfusion-gl-buffer) to the current OpenGL context before calling any modifying member functions.

**See also:** [https://www.khronos.org/opengl/wiki/Buffer_Object](https://www.khronos.org/opengl/wiki/Buffer_Object)

### Usage

```cpp
enum class ImFusion::GL::Buffer::Usage
```

*strong*

Enumeration for usage pattern hints to provide to the GPU when allocating data.

**Enumerator:**

- `StreamDraw` — The data is modified once and used at most a few times; it is modified by the application, and used as the source for [GL](/cpp-sdk/4.5/api/namespaces/imfusion-gl) drawing and image specification commands.
- `StreamRead` — The data is modified once and used at most a few times; it is modified by reading data from the [GL](/cpp-sdk/4.5/api/namespaces/imfusion-gl) device, and used to return that data when queried by the application.
- `StreamCopy` — The data is modified once and used at most a few times; it is modified by reading data from the [GL](/cpp-sdk/4.5/api/namespaces/imfusion-gl) device, and used as the source for [GL](/cpp-sdk/4.5/api/namespaces/imfusion-gl) drawing and image specification commands.
- `StaticDraw` — The data is modified once and used many times; the data is modified by the application, and used as the source for [GL](/cpp-sdk/4.5/api/namespaces/imfusion-gl) drawing and image specification commands.
- `StaticRead` — The data is modified once and used many times; it is modified by reading data from the [GL](/cpp-sdk/4.5/api/namespaces/imfusion-gl) device, and used to return that data when queried by the application.
- `StaticCopy` — The data is modified once and used many times; it is modified by reading data from the [GL](/cpp-sdk/4.5/api/namespaces/imfusion-gl) device, and used as the source for [GL](/cpp-sdk/4.5/api/namespaces/imfusion-gl) drawing and image specification commands.
- `DynamicDraw` — The data is modified repeatedly and used many times; it is modified by the application, and used as the source for [GL](/cpp-sdk/4.5/api/namespaces/imfusion-gl) drawing and image specification commands.
- `DynamicRead` — The data is modified repeatedly and used many times; it is modified by reading data from the [GL](/cpp-sdk/4.5/api/namespaces/imfusion-gl) device, and used to return that data when queried by the application.
- `DynamicCopy` — The data is modified repeatedly and used many times; it is modified by reading data from the [GL](/cpp-sdk/4.5/api/namespaces/imfusion-gl) device, and used as the source for [GL](/cpp-sdk/4.5/api/namespaces/imfusion-gl) drawing and image specification commands.


*protected*

Upload data into the buffer starting at the given offset.

**Parameters:**

- `offset` — Offset in bytes into the buffer object's data store where start writing data.
- `data` — Pack of data arrays that should be copied to the buffer. If you provide multiple spans they will be concatenated and end up next to each other in the buffer without any padding.
