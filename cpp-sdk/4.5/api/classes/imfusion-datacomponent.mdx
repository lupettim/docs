## Overview

template&lt;typename T&gt;

class ImFusion::DataComponent&lt; T &gt;
Main specialization of [DataComponentBase](/cpp-sdk/4.5/api/classes/imfusion-datacomponentbase) for regular types.

If you need serialization, make your class T inherit from [DataComponent&lt;T&gt;](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) and implement the copy- constructor (T(const T&)), the copy assignment operator (T& operator=(const T&)), and the equality comparison operator (bool operator==(const T&) const / friend bool operator==(const T&, const T&)). The [clone()](#clone), [assign()](#assign), and [equals()](#equals) methods will forward to your class methods.

You also still need to implement [id()](/cpp-sdk/4.5/api/classes/imfusion-datacomponentbase), [configure()](/cpp-sdk/4.5/api/classes/imfusion-configurable), and [configuration()](/cpp-sdk/4.5/api/classes/imfusion-configurable) to make your class serializable.

Example:

```cpp
class FooComponent : public DataComponent<FooComponent>
{
public:
 // copy-constructor and copy-assignment operator are auto-generated by the compiler
 bool operator==(const FooComponent& rhs) const { return rhs.i == i; }

 // you still need to implement these methods to make the DataComponent serializable
 std::string id() const override;
 void configure(const Properties*) override;
 void configuration(Properties*) const override;

private:
 int i = 0;
};
```
