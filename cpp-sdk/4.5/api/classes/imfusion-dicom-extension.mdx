## Overview

Interface for adding custom functionality to [DicomLoader](/cpp-sdk/4.5/api/classes/imfusion-dicomloader) and [DicomWriter](/cpp-sdk/4.5/api/classes/imfusion-dicomwriter) An [Extension](/cpp-sdk/4.5/api/classes/imfusion-dicom-extension) might be called in a thread that is not the main thread.

It therefore most not rely on global resources like OpenGL. It will not be called from different threads simultaneously. See Extensions for an example.

**Examples:** [DicomExtensionExample.cpp](/cpp-sdk/4.5/api/examples/dicom-extension-8cpp), [DicomExtensionExample2.cpp](/cpp-sdk/4.5/api/examples/dicom-extension2-8cpp), and [DicomExtensionExample3.cpp](/cpp-sdk/4.5/api/examples/dicom-extension3-8cpp).

### load

```cpp
virtual std::vector < std::unique_ptr < Data > > ImFusion::Dicom::Extension::load (const FrameDescriptor & frame, SharedImageSet & sis) const
```

*pure virtual*

Load the data from DICOM to the given [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset).

This method will be called once for each frame that was generated by the [Dicom::IOD](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod) which created the [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset). Use [FrameDescriptor::iod](/cpp-sdk/4.5/api/structs/imfusion-dicom-framedescriptor) to access the DICOM tags and [FrameDescriptor::sharedImageIndex](/cpp-sdk/4.5/api/structs/imfusion-dicom-framedescriptor)/sliceIndex to figure out which frame data should get loaded. The matrix parameters of frame should be ignored. Use the matrix of the [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) instead. [DicomLoader](/cpp-sdk/4.5/api/classes/imfusion-dicomloader) will basically just iterate overall all slices and call this method. E.g. for a [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) containing two volumes with 4 slices, this method would be called with the following indices:

| sharedImageIndex | sliceIndex |
| --- | --- |
| 0 | 0 |
| 0 | 1 |
| 0 | 2 |
| 0 | 3 |
| 1 | 0 |
| 1 | 1 |
| 1 | 2 |
| 1 | 3 |

Derived classes might also need to differentiate between classic single frame DICOMs (Dicom::MultiFrameIOD) and enhanced multiframe DICOMs ([Dicom::EnhancedMultiFrameImageIOD](/cpp-sdk/4.5/api/classes/imfusion-dicom-enhancedmultiframeimageiod)). This should be done by dynamic_casting the frame.iod to [Dicom::EnhancedMultiFrameImageIOD](/cpp-sdk/4.5/api/classes/imfusion-dicom-enhancedmultiframeimageiod) and assume a classic DICOM otherwise. Use frame.index to access Dicom::EnhancedMultiFrameImageIOD::perFrameSequence.

### save

```cpp
virtual void ImFusion::Dicom::Extension::save (const FrameDescriptor & frame, const SharedImageSet & sis) const
```

*pure virtual*

Save data from the given [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) to DICOM.

This method will be called once for each frame of the [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset). Use [FrameDescriptor::iod](/cpp-sdk/4.5/api/structs/imfusion-dicom-framedescriptor) to access the DICOM tags and [FrameDescriptor::sharedImageIndex](/cpp-sdk/4.5/api/structs/imfusion-dicom-framedescriptor)/sliceIndex to figure out which frame data should be saved. The matrix parameters of frame should be ignored. Use the matrix of the [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) instead.

### showInWriterUI

```cpp
virtual bool ImFusion::Dicom::Extension::showInWriterUI () const
```

*inline, virtual*

Whether this extension will show up in the UI and can be enabled/disabled by the user.

The save method will be called independently of this value. This is useful if the extension only implements load.
