## Overview

Helper class to manage instances of the same base shader with different abstract includes and/or defines.

Both [abstract shader includes](/cpp-sdk/4.5/api/classes/imfusion-gl-abstractinclude) and defines change the actual source code of an OpenGL shader. Often the same base shader is used for different combinations of shader includes/defines. [ProgramCache](/cpp-sdk/4.5/api/classes/imfusion-gl-programcache) class facilitates managing these shader instances for combinations of multiple abstract includes. Based on the type of the abstract include, it will return (and if needed lazy-instantiate it) the correct shader variant that has the corresponding include code expanded.

```cpp
// Instantiate the program cache with the shader base version
GL::ProgramCache cache(GL::Program::createVertexFragment("...", "...");

// Retrieve shader variant that has additionally `SOME_DEFINE` defined
GL::Program& p1 = cache.getProgram("#define SOME_DEFINE");

// Retrieve shader variant with two additional abstract shader includes and with `SOME_DEFINE`.
GL::Program& p2 = cache.getProgram({abstractInclude1, abstractInclude2}, "#define SOME_DEFINE");
```
 fragment

**Warning:** Make sure not to alter the abstract includes and/or defines of shaders returned by this class. Doing so may lead to subtle bugs or even crashes as [ProgramCache](/cpp-sdk/4.5/api/classes/imfusion-gl-programcache) does not track any changes to these shader settings done from the outside.

**See also:** [GL::Program](/cpp-sdk/4.5/api/classes/imfusion-gl-program), [GL::AbstractInclude](/cpp-sdk/4.5/api/classes/imfusion-gl-abstractinclude)

- `referenceProgram` — Reference shader, must not be null.
