`#include <ImFusion/Base/ReplaceValuesAlgorithm.h>`

## Overview

Replace in-place a set of image values with others values.

The input image is expected to have either a UBYTE or USHORT type. The algorithm runs on the GPU unless one of the conditions is met:

- the parameter ForceCPU is set to true,
- the input has more than four channels or a shift/scale,
- there is a reference image.

An additional set of compatible reference images may be provided in the constructor; when this is the case, the values will be read from the reference images but replaced in the main ones.

For convenience purposes, a default value can be set, in which case all not explicitly defined non-zero input values will be assigned this value.

**Warning:** Any shift/scale will be burned in the memory buffer, i.e. the output image will have a zero-shift and a unit-scale.

Example:

```cpp
SharedImageSet inputImage; // example: 1d image with values [0, 0, 0, 1]
SharedImageSet referenceImage; // example: 1d image with values [1, 1, 2, 3]

ReplaceValuesAlgorithm replaceValuesAlgorithm(&inputImage, &referenceImage);
replaceValuesAlgorithm.setReplaceValues({1, 3}, {2, 0});  // 1 will be replaced by 2, 3 will be replaced by 0
replaceValuesAlgorithm.compute();
// now inputImage = [2, 2, 0, 0]
```
