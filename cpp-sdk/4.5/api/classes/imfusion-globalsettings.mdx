`#include <ImFusion/Core/GlobalSettings.h>`

## Overview

Class for storing and retrieving persistent global settings.

This class allows you to load and store single values, as well as entire [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) objects in a persistent manner across application sessions.

This class must be initialized with [init()](#init-1-2) before any usage. Typically this would be done on program startup. In order for the changes to settings to be persisted, sync() must be called. This is typically done on program exit, but can also be called at any time.

There are several interfaces provided for loading and storing settings:

- Objects of types that inherit from *[Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable)* can be loaded and stored directly:
  ```cpp
  class MyConfigurable: public Configurable
  {
  public:
   Parameter<int> mySetting{"mySetting", 10, this};
   Parameter<std::string> myOtherSetting{"myOtherSetting", "default", this};
  } myObject;

  // configure myObject with settings stored for MyConfigurable if they exist
  GlobalSettings::load(myObject);

  // store the configuration of myObject in the settings for MyConfigurable
  GlobalSettings::store(myObject);
  ```
   fragment
- [Settings](/cpp-sdk/4.5/api/classes/imfusion-settings) for [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) types can also be loaded and stored without an instance:
  ```cpp
  // load settings for MyConfigurable
  auto properties = GlobalSettings::load<MyConfigurable>();

  // store settings for MyConfigurable
  GlobalSettings::store<MyConfigurable>(properties.value());
  ```
   fragment

  **Note:** The names used for the settings of [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) types are determined automatically based on the type name. Overloads are provided to specify a custom name if needed.
- Single values can be loaded and stored with a name:
  ```cpp
  // load an integer value
  auto value = GlobalSettings::loadValue<int>("myIntValue");

  // store a double value with a default
  GlobalSettings::storeValue("myDoubleValue", 42.0, 0.0);
  ```
   fragment

  **Note:** The names can be nested using the / character (cf. path spec of [Properties](/cpp-sdk/4.5/api/classes/imfusion-properties)).

  All operations on this class are thread-safe, but the user is responsible for the thread-safety of the [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) objects themselves. The sync operation is also safe across multiple processes through an advisory file lock.

The setting are stored in the following locations by default depending on the platform:

- Linux: $HOME/.config/&lt;org&gt;/&lt;app&gt;.json
- Mac: $HOME/Library/Preferences/&lt;org&gt;/&lt;app&gt;.json
- Windows: APPDATA%/&lt;org&gt;/&lt;app&gt;.json

Where *org* and *app* must be specified by calling [init()](#init-1-2) once in the application.

**See also:** [Properties](/cpp-sdk/4.5/api/classes/imfusion-properties) for more information on serialization.
