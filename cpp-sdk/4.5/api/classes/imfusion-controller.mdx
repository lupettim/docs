`#include <ImFusion/GUI/Controller.h>`

## Overview

[Base](/cpp-sdk/4.5/api/structs/imfusion-dicom-base) interface for a [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui) element within [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase).

A [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) enables the user to configure and control the underlying entity such as an [Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm) or a [Data](/cpp-sdk/4.5/api/classes/imfusion-data) instance hosted by a [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase). Furthermore, a [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) has access to the parent [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) instance and can therefore also access entities that usually would not be accessible by the underlying entity (e.g. [DisplayWidget](/cpp-sdk/4.5/api/classes/imfusion-displaywidget), [DataModel](/cpp-sdk/4.5/api/classes/imfusion-datamodel), other Algorithms, ...). Furthermore, it provides the following callbacks for convenience:

- [onSelectedDataChanged()](#onselecteddatachanged) is called whenever the selection in the [DataWidget](/cpp-sdk/4.5/api/classes/imfusion-datawidget) changes.
- [onVisibleDataChanged()](#onvisibledatachanged) is called whenever the visible data in the [DisplayWidgetMulti](/cpp-sdk/4.5/api/classes/imfusion-displaywidgetmulti) changes.

The [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) interface by itself is just an interface providing the necessary link to the parent [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) instance and does *not* provide any "visible" [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui). This is done in order to avoid a dependency to a particular [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui) framework. However, usually all concrete implementations of this interface will provide a concrete [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui) and therefore depend on a particular [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui) framework (Qt in the case of all ImFusion Controllers).

Depending on the application a [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) can be placed in different locations within the [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui), which can be customized by the programmer through the [supportedLocations()](#supportedlocations) and [preferredLocation()](#preferredlocation) member functions. The actual placement within the application [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui) is to be performed from the outside by the hosting [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) and the [ControllerDecorator](/cpp-sdk/4.5/api/classes/imfusion-controllerdecorator) class/interface. Custom applications specializing [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) are free to implement their own container widgets supporting additional locations.

**Note:** Due to technical constraints [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) requires a two-step initialization procedure! After construction, [setMainWindowBase()](#setmainwindowbase) must be called to provide the parent [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) instance and complete the initialization procedure. Usually, programmers do not need to call this function manually since this is already done by [MainWindowBase::addController()](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase). Implementers of Controllers should put initialization code into the pure virtual [init()](#init) function since it's called by [setMainWindowBase()](#setmainwindowbase) once the parent main window is known. Users must not call any other functions on Controllers before their initialization is complete.

**See also:** [AlgorithmController](/cpp-sdk/4.5/api/classes/imfusion-algorithmcontroller), [DataController](/cpp-sdk/4.5/api/classes/imfusion-datacontroller), [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase), [ControllerDecorator](/cpp-sdk/4.5/api/classes/imfusion-controllerdecorator)

## Member Enumeration Documentation

### int

```cpp
enum class ImFusion::Controller::Location : int
```

*strong*

Enumeration of possible placement/decoration options of a [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller).

**Enumerator:**

- `Unknown` — Exact location could not be determined (e.g. when [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) is not yet initialized).
- `Naked` — [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) should not be wrapped/placed by [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) at all (e.g. [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) hosts its own window).
- `PrimaryDock` — [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) should be placed in MainWindowBase::algorithmDock() if available.
- `SecondaryDock` — [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) should be placed in [MainWindowBase::secondaryAlgorithmDock()](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) if available.
- `Detached` — [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) should be placed in a detached floating window.
- `ToolBar` — [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) should be placed in MainWindowBase::controllerBar() if available.
- `UserLocation` — First unreserved enum value that can be used for custom or application-specific purposes.

## Member Function Documentation

### setMainWindowBase

```cpp
void ImFusion::Controller::setMainWindowBase (MainWindowBase * mw)
```

Complete the initialization by providing the parent [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) instance hosting the controller by setting m_main and m_disp and eventually call [init()](#init).

This function is called by [MainWindowBase::addController()](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) so that you normally do not need to call it manually from the outside except in case you want to use a [Controller](/cpp-sdk/4.5/api/classes/imfusion-controller) without adding it to [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase).

**Note:** Controllers do not support exchanging their main window. Therefore, this function must not be called multiple times with different arguments!
