Inherits [ImFusion::Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable), and [ImFusion::Utils::NotCopyable](/cpp-sdk/4.5/api/classes/imfusion-utils-notcopyable).

## Overview

Interface for describing algorithms that can be made available in the ImFusion Suite through [AlgorithmFactory](/cpp-sdk/4.5/api/classes/imfusion-algorithmfactory).

The [Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm) interface provides the following functionality:

- They can be registered with the [AlgorithmFactory](/cpp-sdk/4.5/api/classes/imfusion-algorithmfactory). Define a [createCompatible()](#createcompatible) function serving both as predicate function checking if input data is suitable, as well as factory function to instantiate the algorithm. If an algorithm was instantiated by a factory, the [factoryInfo()](#factoryinfo) struct is populated.
- The inherit from [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) so that their settings/parameters can be easily serialized.
- You must implement the [compute()](#compute) function to execute the algorithm on the input data provided at creation. Afterwards, users can call [takeOutput()](#takeoutput) to receive [Data](/cpp-sdk/4.5/api/classes/imfusion-data) that was newly created during [compute()](#compute). Use the [status()](#status) interface to signal whether the last call to [compute()](#compute) was successful or not.
- The [Action](/cpp-sdk/4.5/api/structs/imfusion-algorithm-action) interface enables you to define a generic set of actions a user can execute on your algorithm without knowing about the concrete derived type.

If you would like to provide a [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui) for your [Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm) you need to use the [AlgorithmController](/cpp-sdk/4.5/api/classes/imfusion-algorithmcontroller) interface. For rapid prototyping you can use the [DefaultAlgorithmController](/cpp-sdk/4.5/api/classes/imfusion-defaultalgorithmcontroller) to automatically generate a [GUI](/cpp-sdk/4.5/api/namespaces/imfusion-gui) from the [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) interface and set of registered actions.

**Note:** The constructor of derived classes might perform a check for a valid license and throw a [MissingLicenseException](/cpp-sdk/4.5/api/classes/imfusion-missinglicenseexception) in case of failure.

**See also:** [Writing Algorithms](/cpp-sdk/4.5/api/guides/algorithm-tutorial), [Algorithms](/cpp-sdk/4.5/api/guides/algorithms), [AlgorithmUtils](/cpp-sdk/4.5/api/namespaces/imfusion-algorithmutils), [AlgorithmController](/cpp-sdk/4.5/api/classes/imfusion-algorithmcontroller)

**Examples:** [ExampleAlgorithm.cpp](/cpp-sdk/4.5/api/examples/example-algorithm-8cpp), [ExampleAlgorithm.h](/cpp-sdk/4.5/api/examples/example-algorithm-8h), [SqrtAlgorithm.cpp](/cpp-sdk/4.5/api/examples/sqrt-algorithm-8cpp), and [SqrtAlgorithm.h](/cpp-sdk/4.5/api/examples/sqrt-algorithm-8h).

## Member Enumeration Documentation

### Status

```cpp
enum ImFusion::Algorithm::Status
```

[Status](#status) codes.

**Enumerator:**

- `Unknown` — Returned before compute has been called or when algorithm does not implement status interface.
- `Success` — Computation was successful.
- `Error` — Generic catch all error status.
- `InvalidInput` — Some input arguments do not meet the requirements.
- `IncompleteInput` — Not all required input has been set.
- `OutOfMemoryHost` — Insufficient main memory, e.g. a [std::bad_alloc](http://en.cppreference.com/w/cpp/memory/new/bad_alloc.html).
- `OutOfMemoryGPU` — Insufficient memory on the GPU.
- `UnsupportedGPU` — This should be returned when for instance the OpenGL version is not high enough or CUDA/OpenCL is not available.
- `UnknownAction` — This is returned by runAction if an action with the requested id does not exist.
- `AbortedByUser` — User canceled execution.
- `User` — [Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm) specific status codes can be defined above this status code.

## Member Function Documentation

### createCompatible

```cpp
bool ImFusion::Algorithm::createCompatible (const DataList & data, Algorithm ** a = nullptr)
```

*static*

Factory function to check algorithm compatibility with input data and optionally instantiate it.

Every subclass of [Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm) must implement this function in order to be usable by [AlgorithmFactory](/cpp-sdk/4.5/api/classes/imfusion-algorithmfactory).

**Parameters:**

- `data` — Input data for the algorithm
- `a` — Optional output pointer for instantiation through factory. If *a* is not null, this function will instantiate the algorithm and store the pointer there.

**Returns:** true if the algorithm is compatible with the input data.

### takeOutput

```cpp
virtual OwningDataList ImFusion::Algorithm::takeOutput ()
```

*virtual*

Return any new [Data](/cpp-sdk/4.5/api/classes/imfusion-data) that was created by the [Algorithm](/cpp-sdk/4.5/api/classes/imfusion-algorithm) during the last call to [compute()](#compute).

The default implementation will return an empty list.

**Note:** Since ownership of the data is transferred, you can call this method only once between calls to [compute()](#compute).
