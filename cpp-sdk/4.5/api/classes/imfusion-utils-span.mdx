## Overview

template&lt;typename ElementType, [std::size_t](http://en.cppreference.com/w/cpp/types/size_t.html) Extent&gt;

class ImFusion::Utils::Span&lt; ElementType, Extent &gt;
Describes a non-owning view into contiguous sequence of elements.

The main use case of a span is to pass contiguous arrays of objects as function parameters so that the callee can work with/on the elements without requiring a particular container type to be used by the caller. Spans can be constructed conveniently from many different containers such as arrays, [std::vector](http://en.cppreference.com/w/cpp/container/vector.html), and [std::string](http://en.cppreference.com/w/cpp/string/basic_string.html).

A span can either have a static extent, in which case the number of elements in the sequence is known at compile-time and encoded in the type, or a dynamic extent. For a [Span](/cpp-sdk/4.5/api/classes/imfusion-utils-span)s, pointers, iterators, and references to elements of s are invalidated when an operation invalidates a pointer in the range [s.data(), s.data() + s.size()).

**Template Parameters:** | ElementType | Type of the wrapped elements, must be a complete type. |
| --- | --- |
| Extent | The number of elements in the sequence, or SpanDynamicExtent if dynamic. |

**Note:** This class is a polyfill of [std::span](http://en.cppreference.com/w/cpp/container/span.html) from C++20 providing the same interface and functionality.
