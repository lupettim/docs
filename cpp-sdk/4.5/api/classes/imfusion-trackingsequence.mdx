`#include <ImFusion/Base/TrackingSequence.h>`

## Overview

[Sequence](/cpp-sdk/4.5/api/classes/imfusion-dicom-sequence) of rigid tracking data with optional timestamps, quality values and flags.

By convention, TrackingSequences are either owned by the data model and thus shown in the data widget, or by some other class, for instance [TrackedSharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-trackedsharedimageset).

All transformations map to the world coordinate system. The method matrix() without any arguments contains the registration matrix of the entire tracking sequence. This allows using common suite features, for instance linking the tracking sequence with images, moving everything coherently. Querying a specific matrix M (matrix(int) with an index or [matrix(double)](#matrix-1-2) with a time stamp) will obtain the following sequence of transformations: \(M = R \cdot M[i] \cdot C\), where R is the registration matrix, C the calibration matrix, and M[i] the specific matrix queried for.

**Note:** In older versions of the SDK this class was called "TrackingStream" and was only later renamed to [TrackingSequence](/cpp-sdk/4.5/api/classes/imfusion-trackingsequence) in order to avoid confusion with the ImFusion::Stream interface. Therefore, at some places that have not yet been adapted, documentation or member names may still refer to a "tracking stream" even though it means a TrackingSequence.

## Member Enumeration Documentation

### FilterMode

```cpp
enum ImFusion::TrackingSequence::FilterMode
```

Set spatial filtering of the tracking data (0=None, 1=Position, 2=Position+Rotation).

The set of filtered matrices are computed in this method, and the original ones backed up. If a filter is set, new tracking data can only be added to the end of the sequence and old tracking data can only be removed from the beginning of the sequence. After adding or removing (one or multiple) samples this method must be called. Adding and removing samples must not be done together in between two calls.

```cpp
double ImFusion::TrackingSequence::quality (double time, bool checkDistance, bool ignoreRelative = false) const
```

Return the quality value at a particular time.

If checkDistance is enabled, a quality of zero is returned when the time is before or after the range of the timestamps, or within a gap. If ignoreRelative is enabled, the quality is calculated from this tracking sequence only and not multiplied with the tracking quality of the reference tracker (only relevant if relative tracking sequence is set).
