## Overview

Configures the display options for 2D slice rendering of images.

2D display options are mainly consumed by [GlSlice](/cpp-sdk/4.5/api/classes/imfusion-glslice) which implements 2D rendering of image data. Usually, [DisplayOptions2d](/cpp-sdk/4.5/api/classes/imfusion-displayoptions2d) are attached as a [DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) to the image for which they define the rendering options. However, classes like [ImageView2D](/cpp-sdk/4.5/api/classes/imfusion-imageview2d) may use special (e.g. view-specific) display options for rendering their images and therefore use the same interface.

**Note:** Window/level parameters are defined in original pixel value domain (e.g. Hounsfield Units). Conversion of pixel values to display intensities is performed using the LINEAR_EXACT VOI LUT function as described in section C.11.2.1.3.2 of the DICOM standard. [http://dicom.nema.org/medical/dicom/current/output/chtml/part03/sect_C.11.2.html#sect_C.11.2.1.3](http://dicom.nema.org/medical/dicom/current/output/chtml/part03/sect_C.11.2.html#sect_C.11.2.1.3)

**See also:** [GlSlice](/cpp-sdk/4.5/api/classes/imfusion-glslice), [ImageView2D](/cpp-sdk/4.5/api/classes/imfusion-imageview2d), [DisplayOptions3d](/cpp-sdk/4.5/api/classes/imfusion-displayoptions3d)

## Member Enumeration Documentation

### MaskingMode

```cpp
enum class ImFusion::DisplayOptions2d::MaskingMode
```

*strong*

Enumeration of options for how masked image pixels should be rendered.

**Enumerator:**

- `Ignore` — Ignore the mask, render the pixels normally.
- `Discard` — Discard masked pixels (where the mask value is 0).
- `AdditiveBlend` — Add maskingColor() to masked pixels (where the mask value is 0).
- `MultiplicativeBlend` — Multiply masked pixels with the maskingColor() (where the mask value is 0).
- `DiscardZerosAdditiveOnes` — Discard pixels where the mask value is 0, add maskingColor() to pixels where the mask value is 1.
- `DiscardZerosMultiplicativeOnes` — Discard pixels where the mask value is 0, multiply pixels with maskingColor() where the mask value is 1.

In all other cases it will be initialized to match a value range of [0..255].
