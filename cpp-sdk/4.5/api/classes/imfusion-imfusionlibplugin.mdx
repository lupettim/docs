## Overview

Specialized interface for a plugin for the ImFusionLib.

In addition to the base interface it provides the [setApplicationController()](#setapplicationcontroller) member function that can be called by host applications with an [ApplicationController](/cpp-sdk/4.5/api/classes/imfusion-applicationcontroller) so that plugins get access to it.

**Note:** In contrast to the legacy ImFusionPlugin interface, you must conduct license checks and factory registration yourself in [init()](/cpp-sdk/4.5/api/classes/imfusion-pluginbase). A typical plugin initialization function has the following structure:
```cpp
PluginBase::Status MyPlugin::init() {
 // conduct license check if necessary at the beginning
 if (!isLicensed("<feature code for plugin>"))
 return Status::LicenseCheckFailed;

 // instantiate necessary factories and register them with FactoryRegistry
    m_algFactory = std::make_unique<MyAlgorithmFactory>();
    m_algCtrlFactory = std::make_unique<MyAlgorithmControllerFactory>();
 registerFactories(m_algFactory.get(), m_algCtrlFactory.get(), nullptr);

 // perform registration of further factories etc.
    [...]

 return Status::Success;
}
```
