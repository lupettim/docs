## Overview

template&lt;typename ObjectType, typename EventHandlerType = ViewObject::EventHandler&gt;

class ImFusion::GUI::TypedViewObject&lt; ObjectType, EventHandlerType &gt;
Specialization of a [ViewObject](/cpp-sdk/4.5/api/classes/imfusion-gui-viewobject) where the concrete types of the underlying [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) and [EventHandler](/cpp-sdk/4.5/api/classes/imfusion-gui-viewobject-eventhandler) are defined by the template parameters.

This can be useful if you know them in advance, don't need to rely on polymorphism, and want to avoid manually casting repeatedly.

## Member Function Documentation

### handleInputEvent

```cpp
template<typename ObjectType, typename EventHandlerType> EventResult ImFusion::GUI::TypedViewObject < ObjectType, EventHandlerType >::handleInputEvent (const InputEvent & event, const View & view)
```

*override, virtual*

Gets called by the parent *view* in order to handle user input events.

It returns an [EventResult](/cpp-sdk/4.5/api/structs/imfusion-gui-eventresult) to indicated if and how event propagation should continue.

**Parameters:**

- `event` — User input event to handle.
- `view` — Parent [View](/cpp-sdk/4.5/api/classes/imfusion-gui-view) where the event originates from. This additional payload enables you to add the same [ViewObject](/cpp-sdk/4.5/api/classes/imfusion-gui-viewobject) to multiple views and still implement view-specific handling logic.
