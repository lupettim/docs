`#include <ImFusion/GUI/SelectionWidget.h>`

## Overview

Widget and scrollbar for handling selection and playback of data.

If more than one data is selected, the selection of all of them can be handled simultaneously as long as they have the same size as the first selected [Data](/cpp-sdk/4.5/api/classes/imfusion-data).

## Classes

- class [PlaybackWrapper](/cpp-sdk/4.5/api/classes/imfusion-selectionwidget-playbackwrapper) — This class provides an interface for creating wrappers that allow playback functionality to a type.

## Member Enumeration Documentation

### InputMode

```cpp
enum class ImFusion::SelectionWidget::InputMode
```

*strong*

Enumeration of modes how to determine which datasets are used as input data to operate on by the [SelectionWidget](/cpp-sdk/4.5/api/classes/imfusion-selectionwidget).

**Enumerator:**

- `Manual` — The datasets are solely defined by the user through calling [setInputData()](#setinputdata).
- `VisibleData` — Listen to the [DisplayWidget](/cpp-sdk/4.5/api/classes/imfusion-displaywidget) of the parent [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) and use the currently visible datasets.
- `DataWidgetSelection` — Listen to the [DataWidget](/cpp-sdk/4.5/api/classes/imfusion-datawidget) of the parent [MainWindowBase](/cpp-sdk/4.5/api/classes/imfusion-mainwindowbase) and use its current selection.

### LockMode

```cpp
enum class ImFusion::SelectionWidget::LockMode
```

*strong*

Enumeration of modes how the selection changes with the focus.

**Enumerator:**

- `None` — Focus has no effect on the rest of the selection.
- `Single` — Both *first* and *last* are locked to the focus.
- `First` — The *first* parameter of the selection is locked to the focus.
- `Last` — The *last* parameter of the selection is locked to the focus.

## Member Function Documentation

### setInputData

```cpp
void ImFusion::SelectionWidget::setInputData (const std::vector < Selectable * > & input)
```

Explicitly define what datasets the [SelectionWidget](/cpp-sdk/4.5/api/classes/imfusion-selectionwidget) should operate on.

If *input* has multiple datasets they must have the same size in order to be manipulated at the same time. If inputMode() is not InputMode::Manual the input data will be changed automatically according to the setting.

### inputData

```cpp
const std::vector < Selectable * > & ImFusion::SelectionWidget::inputData ()
```

*inline*

Returns the datasets the [Selection](/cpp-sdk/4.5/api/classes/imfusion-selection) widget is currently operating on.

**Note:** The return value may be a subset of the datasets originally passed to [setInputData()](#setinputdata) because all input datasets must have the same size in order to be manipulated at the same time.

### isPlaying

```cpp
bool ImFusion::SelectionWidget::isPlaying ()
```
