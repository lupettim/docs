## Overview

Calculates a deformable registration between two meshes by calculating a deformable registration between distance volumes.

Internally, an instance of the [DemonsImageRegistration](/cpp-sdk/4.5/api/classes/imfusion-demonsimageregistration) algorithm is used to register the "fixed" distance volume to the "moving" distance volume. As this registration computes the *inverse* of the mapping from the fixed to the moving volume, this directly yields a registration of the "moving" [Mesh](/cpp-sdk/4.5/api/classes/imfusion-mesh) to the "fixed" [Mesh](/cpp-sdk/4.5/api/classes/imfusion-mesh).

## Member Function Documentation

### compute

```cpp
void ImFusion::VolumeBasedMeshRegistrationAlgorithm::compute ()
```

*override, virtual*

Computes the deformable registration from meshMoving to meshFixed.

If a moving mesh has been provided, the result of this deformation is applied to the "moving" mesh in place. If a [PointCloud](/cpp-sdk/4.5/api/classes/imfusion-pointcloud) on the "moving" mesh was provided, this is also deformed in-place. The registration comprises a scaling and rigid initialization, followed by an image-based deformable registration.

### initialize

```cpp
Status ImFusion::VolumeBasedMeshRegistrationAlgorithm::initialize ()
```

Initializes the registration by rescaling and preforming a linear registration.

This function will in general affect both the pose and truncation of the distance volumes, thus running this more than once is not recommended as state is not restored between calls.


*virtual*

Return an error measure between the segmentation mesh and the deformed reference mesh.

This measure is given (in mm) by the maximum displacement of the projections of the vertices of the moving mesh onto the fixed mesh This is only possible if two meshes were provided at construction, [std::numeric_limits&lt;double&gt;::max()](http://en.cppreference.com/w/cpp/types/numeric_limits/max.html) is returned otherwise.

### p_memoryLimitMB

```cpp
Parameter < std::optional <size_t> > ImFusion::VolumeBasedMeshRegistrationAlgorithm::p_memoryLimitMB = {"memoryLimitMB", std::nullopt, this}
```

Memory limit that aborts creation of internal helper objects (e.g.

distance volume, label map) if they cumulatively surpass this limit.

**Warning:** This algorithm keeps an internal counter of the amount of memory allocated during its lifetime. All allocations made by the algorithm (including those made before the limit was set) are accounted for, and the counter is decremented when the corresponding deformations are released.

**Warning:** Only the GPU paths are reliably compatible with this, e.g. setting p_subVoxelPrecision=false may result in larger volumes being allocated.

### m_movingMeshWithoutDeformation

```cpp
std::optional < std::unique_ptr < Mesh > > ImFusion::VolumeBasedMeshRegistrationAlgorithm::m_movingMeshWithoutDeformation
```

*protected*

The algorithm uses the displacement values from an (inverse) deformation to update the moving mesh and landmarks.

We store the meshes/landmarks without the added displacement values here in order to be able to update the moving mesh/landmarks appropriately.
