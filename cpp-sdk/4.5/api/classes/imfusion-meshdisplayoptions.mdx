## Overview

[DataComponent](/cpp-sdk/4.5/api/classes/imfusion-datacomponent) to store the configuration how meshes should be rendered by default (through MeshDisplayHandler).

[MeshDisplayOptions](/cpp-sdk/4.5/api/classes/imfusion-meshdisplayoptions) are attached to the [DataComponentList](/cpp-sdk/4.5/api/classes/imfusion-datacomponentlist) of its parent [Mesh](/cpp-sdk/4.5/api/classes/imfusion-mesh) so that they can be accessed easily from everywhere. MeshDisplayHandler will query them to configure its renderers.

**See also:** MeshDisplayHandler, MeshRendererSurface, MeshRendererIntersection, MeshRendererNormals

## Member Enumeration Documentation

### RenderMode2d

```cpp
enum ImFusion::MeshDisplayOptions::RenderMode2d
```

Enumeration of options how a mesh should be shown in MPR views.

**Enumerator:**

- `Intersection` — Render intersection with MPR plane only.
- `Surface` — Additionally render the mesh using MeshRendererSurface projected onto the MPR without any clipping.
- `SurfaceWithClipping` — Additionally render the mesh using MeshRendererSurface clipped against the MPR plane.

## Member Function Documentation

### id

```cpp
std::string ImFusion::MeshDisplayOptions::id () const
```

*inline, override, virtual*

Returns a unique string identifier for this type of data component.

**Note:** To avoid conflicts make sure to include potential plugin names into this ID. Due to the modular architecture of the ImFusion library, uniqueness of IDs cannot be checked at compile time. However, [DataComponentFactory](/cpp-sdk/4.5/api/classes/imfusion-datacomponentfactory) will check the uniqueness of the ID during registration.
