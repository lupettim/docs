## Overview

template&lt;typename T&gt;

class ImFusion::SubProperty&lt; T &gt;
The [SubProperty](/cpp-sdk/4.5/api/classes/imfusion-subproperty) class represents a nested record of a [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) entity implementing value semantics.

It is intended to be used as member of [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) classes and enables you to reduce the amount of boilerplate code needed to implement the pair of [configure()](#configure)/configuration() functions for each nested class member that is supposed to be serialized. In order for this to work each [SubProperty](/cpp-sdk/4.5/api/classes/imfusion-subproperty) must be [registered](/cpp-sdk/4.5/api/classes/imfusion-configurable) with the parent [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) instance. This can be automatized by passing it as additional argument to the corresponding constructor overload. Registration is preserved across copies/moves/assignments, and removed on destruction.

SubProperties wrap a regular type T that must implement the [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) interface or alternatively an [std::optional](http://en.cppreference.com/w/cpp/utility/optional.html) of it and make it serializable to/from the [Properties](/cpp-sdk/4.5/api/classes/imfusion-properties) interface. The nested object is stored as a [subproperty](/cpp-sdk/4.5/api/classes/imfusion-properties) inside the given [Properties](/cpp-sdk/4.5/api/classes/imfusion-properties).

Depending on T the behavior of [configure()](#configure) is as follows:

- If T is a regular value type it will always configure the payload from the subproperty if present.
- If T is of type [std::optional](http://en.cppreference.com/w/cpp/utility/optional.html)&lt;S&gt; configure will set the payload to the empty Optional if the subproperty is absent, and instantiate a default-constructed S if the subproperty is present. Therefore, S must be default-constructible.
- If T is of type [std::unique_ptr](http://en.cppreference.com/w/cpp/memory/unique_ptr.html)&lt;S&gt; configure will *not* change the null-state of the payload: This means that if T is originally not null but the subproperty is absent the payload will remain unchanged. Also, if T is originally null but the subproperty is present the payload will remain nullptr. This is an active design decision to allow S to be a polymorphic and/or not default-constructible type.

**Note:** Support for [SubProperty&lt;std::unique_ptr&lt;T&gt;&gt;](/cpp-sdk/4.5/api/classes/imfusion-subproperty) is deprecated and will be removed in a future release. Use [PolymorphicSubProperty](/cpp-sdk/4.5/api/classes/imfusion-polymorphicsubproperty) instead.

The [SubProperty](/cpp-sdk/4.5/api/classes/imfusion-subproperty) class provides the arrow operator, so that you can easily access members of the nested elements:

```cpp
struct InnerStruct : public Configurable {
 Parameter<int> p_count = {"count", 0, this};
}
struct OuterStruct : public Configurable {
    SubProperty<InnerStruct> p_innerStruct = {"InnerStruct", InnerStruct(), this};
}

OuterStruct o;
o.p_innerStruct->p_count = 42;
```
 fragment
The signal [signalValueChanged](/cpp-sdk/4.5/api/classes/imfusion-parameterbase) is only emitted whenever the nested object is changed using setValue(), [configure()](#configure), or if its [Configurable::signalParametersChanged](/cpp-sdk/4.5/api/classes/imfusion-configurable) was emitted.

By convention, public SubProperties members use the p_ prefix for the variable name (instead of the common m_ prefix) in order to make their semantics easier to identify in code.

**Note:** Due to the way deserialization is implemented, you can not have multiple [SubProperty](/cpp-sdk/4.5/api/classes/imfusion-subproperty) instances with the same name next to each other.

**Template Parameters:** | T | Underlying type of the parameter, must be copyable and implement the [Configurable](/cpp-sdk/4.5/api/classes/imfusion-configurable) interface. If T is nullable (pointer/optional) those requirements apply to its payload. |
