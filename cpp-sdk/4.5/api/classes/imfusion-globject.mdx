## Overview

Abstract base class for an renderable OpenGL object that can be hosted in a [GlView](/cpp-sdk/4.5/api/classes/imfusion-glview).

In contrast to a [GlOverlay](/cpp-sdk/4.5/api/classes/imfusion-gloverlay), a [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject) reside in world space and is rendered as part of the scene of the hosting [GlView](/cpp-sdk/4.5/api/classes/imfusion-glview). If you do not require user interaction with the [GlObject](/cpp-sdk/4.5/api/classes/imfusion-globject), you can add it directly to a [GlView](/cpp-sdk/4.5/api/classes/imfusion-glview). Alternatively, you can wrap it in an [InteractiveObject](/cpp-sdk/4.5/api/classes/imfusion-interactiveobject) pairing it with a [Manipulator](/cpp-sdk/4.5/api/classes/imfusion-manipulator) and then add it to the corresponding [InteractiveView](/cpp-sdk/4.5/api/classes/imfusion-interactiveview).

**See also:** [InteractiveObject](/cpp-sdk/4.5/api/classes/imfusion-interactiveobject)

**Examples:** [MyCustomGlObject.h](/cpp-sdk/4.5/api/examples/my-custom-gl-object-8h).

## Member Enumeration Documentation

### RenderingOrderGroup

```cpp
enum ImFusion::GlObject::RenderingOrderGroup
```

Enumeration to determine rendering order of GlObjects in [GlSliceView](/cpp-sdk/4.5/api/classes/imfusion-glsliceview) and [GlVolumeView](/cpp-sdk/4.5/api/classes/imfusion-glvolumeview).

Objects are to be rendered in ascending order according to their group.

**Enumerator:**

- `RenderingGroupRest` — This group will be rendered first.
- `RenderingGroupMesh` — This group will be rendered after the rest since meshes might be transparent.
- `RenderingGroupManipulator` — This group will be rendered last since to avoid manipulators to be hidden by other objects.
