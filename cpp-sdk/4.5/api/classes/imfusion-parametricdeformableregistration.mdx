## Overview

Registers two images with a non-linear deformation based on TPSs.

The algorithm modifies the [GlTPSDeformation](/cpp-sdk/4.5/api/classes/imfusion-gltpsdeformation) object attached to image2. Because TPS deformations are generally not invertible, only one image can have a deformation at a time.

## Member Function Documentation

### compute

```cpp
void ImFusion::ParametricDeformableRegistration::compute ()
```

*override, virtual*

Starts the registration.

A optimizer has to be set before hand. During the registration the deformation of the moving image will be updated with temporary results. Upon return the deformation of the moving image is one that maximises the similarity measure.

### evaluate

```cpp
double ImFusion::ParametricDeformableRegistration::evaluate (int n, const double * x, double * dx = 0)
```

*override, virtual*

Assembles the parameters and returns similarity of images.

This method is mainly called during the optimization by the optimizer. n specifies the number of parameters in the arrays x and dx. x[i] contains the relative change to parameter i. Derived classes overriding this method must be able to apply these parameters to their transformation model. Since the parameters are only relative changes to the initial value, it is sometimes required to store the initial transformation in [compute()](#compute) before the optimization is started. The parameters will be optimized to maximize the return value of this method. If dx is not 0, derived classes can output the derivative of parameter i to each dx[i]. However, derived classes are not required to provide such a value.

Example:

```cpp
double evaluate(int n, const double* x, out double* dx = 0) {
 if (n != 6)
 return 0.0;
 else {
        mat4 relativeMatrix = Pose::eulerToMat(x);
 m_image2->setMatrix(m_initialMatrix * relativeMatrix); // move image to new location
 return return m_sm->compute(m_transformer); // automatically picks up the new transformation of image2
    }
}
```

### reset

```cpp
void ImFusion::ParametricDeformableRegistration::reset ()
```

*override, virtual*

Resets the deformation of moving to its initial deformation.

If the moving image had a GlFFDeformation upon creation of this algorithm, the deformation will be resetted to this state. Otherwise only the displacements will be set to 0 but the number of subdivisions and the spacing will be kept.
