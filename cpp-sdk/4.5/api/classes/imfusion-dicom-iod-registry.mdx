## Overview

A registry that assigns a [IOD](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod) implementation to a SOP class UID.

Both the [DicomLoader](/cpp-sdk/4.5/api/classes/imfusion-dicomloader) and [DicomWriter](/cpp-sdk/4.5/api/classes/imfusion-dicomwriter) use an instance of this class to determine how DICOM files should be loaded or saved. The [IOD_Registry](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod-registry) offers some static methods that can extend the content of the default [IOD_Registry](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod-registry).

The most common use-case of the [IOD_Registry](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod-registry) is enabled or disabling certain extensions. E.g. to enable the OverlayExtension:

```cpp
auto registry = Dicom::IOD_Registry::createDefault();
auto overlayExt = registry->findExtension<Dicom::OverlayExtension>();
if (overlayExt)
    overlayExt->setEnabled(true);
DicomLoader loader(..., std::move(registry));
```

*static*

Register a factory for [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset) subclasses.

Certain DICOMs should be loaded as a specialized sub-class of [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset), e.g. as a UltrasoundSweep or ConeBeamData. Factories should only return a new object if the given DICOM dataset fullfills all requirements of the corresponding specialized [SharedImageSet](/cpp-sdk/4.5/api/classes/imfusion-sharedimageset). Otherwise they should return nullptr.

### registerLoaderIOD

```cpp

```

*static*

Registers a new [IOD](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod) for loading a [Dicom](/cpp-sdk/4.5/api/namespaces/imfusion-dicom) dataset.

The function should return a valid [IOD](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod) implementation that can load the given dataset or 0 otherwise. The sopClassUID is provided for convenience. Loaders are queried in reverse order of registration i.e. the last registered loader is queried first.

### registerWriterIOD

```cpp
template<typename T> void ImFusion::Dicom::IOD_Registry::registerWriterIOD (const std::string & name, bool isMultiframe, std::function < std::string (SharedImageSet &)> canWrite)
```

*static*

Registers a new [IOD](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod) for writing a [Dicom](/cpp-sdk/4.5/api/namespaces/imfusion-dicom) dataset.

Each writer is uniquely identified by its name. The canWrite method should return a SOP class UID if the registered writer can save the given image or an empty string otherwise. The function is allowed to return different SOP class UID for different images. The isMultiframe flag defines if the [IOD](/cpp-sdk/4.5/api/classes/imfusion-dicom-iod) can store multiple frames in one dataset or not.
