## Overview

[Base](/cpp-sdk/4.5/api/structs/imfusion-dicom-base) class and interface for images.

This class is merely a wrapper around an [ImageDescriptor](/cpp-sdk/4.5/api/structs/imfusion-imagedescriptor) that offers a couple of convenience functions for the user. It does not store any image data, which is only done by the derived classes.

Older iterations of the SDK did not yet have the dedicated [ImageDescriptor](/cpp-sdk/4.5/api/structs/imfusion-imagedescriptor) class and were therefore using the [Image](/cpp-sdk/4.5/api/classes/imfusion-image) class as descriptor. New code should refrain from this and use [ImageDescriptor](/cpp-sdk/4.5/api/structs/imfusion-imagedescriptor) directly to avoid confusing semantics of function signatures.

**See also:** [Using Image Data](/cpp-sdk/4.5/api/guides/image-data-tutorial)

## Member Enumeration Documentation

### Type

```cpp
enum ImFusion::Image::Type
```

Pixel/voxel data type, equivalent to OpenGL defines.

**Note:** This enumeration is redundant with PixelType and may become deprecated eventually. Prefer interfaces using PixelType when writing new code.

### Location

```cpp
enum ImFusion::Image::Location
```

[Location](#location) where the data resides.

**Enumerator:**

- `NONE` — Actual data is not available.
- `MEMORY` — CPU main memory.
- `OPENGL` — OpenGL texture.
- `OPENCL` — OpenCL image or buffer.
- `CUSTOM` — Custom image location.
