## Overview

template&lt;typename PosT, typename TexCoordT = detail::NullAtribT, typename ColorT = detail::NullAtribT&gt;

struct ImFusion::GL::FixedFunctionPipeline::VertexData&lt; PosT, TexCoordT, ColorT &gt;
Utility struct to build a [VertexBuffer](/cpp-sdk/4.5/api/classes/imfusion-gl-vertexbuffer) to be used with [FixedFunctionPipeline](/cpp-sdk/4.5/api/classes/imfusion-gl-fixedfunctionpipeline) in a type-safe fashion.

You can either fill a [VertexBuffer](/cpp-sdk/4.5/api/classes/imfusion-gl-vertexbuffer) with the data using [assign()](#assign) or create a new one using [createBuffer()](#createbuffer). This interface also provides an implicit conversion to [std::unique_ptr&lt;VertexBuffer&gt;](http://en.cppreference.com/w/cpp/memory/unique_ptr.html) to minimize code overhead.

**Note:** Do instantiate this class manually but use the [FixedFunctionPipeline::makeVertexData()](/cpp-sdk/4.5/api/classes/imfusion-gl-fixedfunctionpipeline) factory function:
```cpp
std::vector<vec2> vertices = ...;
std::vector<vec2> texCoords = ...;
std::unique_ptr<VertexBuffer> vbo = FixedFunctionPipeline::makeVertexData(vertices).useTexCoords(texCoords);
```
