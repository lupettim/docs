## Member Function Documentation

### isOwnedByProject

```cpp
bool ImFusion::DataSource::isOwnedByProject (const Project & p) const
```

Returns whether this [DataSource](/cpp-sdk/4.5/api/structs/imfusion-datasource) is stored directly in the project.

For local projects that means the file is part of the project directory. For remote projects the file can be directly downloaded from the Labels server.

### fromImage

```cpp
std::optional < DataSource > ImFusion::DataSource::fromImage (const SharedImageSet & sis, bool storedInProject)
```

*static*

Generates a [DataSource](/cpp-sdk/4.5/api/structs/imfusion-datasource) from the [DataSourceComponent](/cpp-sdk/4.5/api/classes/imfusion-datasourcecomponent) of the given image.

If storedInProject is true and [DataSourceComponent](/cpp-sdk/4.5/api/classes/imfusion-datasourcecomponent) is missing (e.g. when the image was not loaded from a file) a default [DataSource](/cpp-sdk/4.5/api/structs/imfusion-datasource) for an ImFusionFile is returned.

## Member Data Documentation

### m_path

```cpp
std::string ImFusion::DataSource::m_path
```

Specifies *where* the data can be (down)loaded from.

If empty, the file can be downloaded directly from the server or loaded directly from the filesystem inside the project path. Use [DataDescriptor::absoluteDataSourcePath](/cpp-sdk/4.5/api/classes/imfusion-datadescriptor) to get the final path. The path can also be a URL.

### m_loader

```cpp
std::string ImFusion::DataSource::m_loader
```

Specifies *how* the data can be *loaded* into an image (i.e.

the id of the [IoAlgorithm](/cpp-sdk/4.5/api/classes/imfusion-ioalgorithm)) If empty, loading will guess which [IoAlgorithm](/cpp-sdk/4.5/api/classes/imfusion-ioalgorithm) to use which is not recommended because the same data might be loaded differently depending on the available plugins.
