`#include <ImFusion/Base/OAuth2.h>`

## Overview

Configuration for the OAuth2 Device Code Flow.

The endpoints depend on the OAuth2 server. For keycloak those are defined in [https://](https://)&lt;keycloak&gt;/realms/&lt;realm&gt;/.well-known/openid-configuration.

## Public Attributes

- [std::string](http://en.cppreference.com/w/cpp/string/basic_string.html) **authEndpoint** — URL that will be used to start the device flow.
- [std::string](http://en.cppreference.com/w/cpp/string/basic_string.html) **tokenEndpoint** — URL to retrieve the access token at the end of the flow.
- [std::string](http://en.cppreference.com/w/cpp/string/basic_string.html) **clientId** — ClientID as configured in the OAuth2 server.
- [std::string](http://en.cppreference.com/w/cpp/string/basic_string.html) **scope** — Desired scope of the access token.
