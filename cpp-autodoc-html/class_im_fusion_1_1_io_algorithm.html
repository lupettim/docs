Detailed Description

Interface for describing IO algorithms.

IO algorithms constructed with an empty DataList in createCompatible() will read data. IO algorithms constructed with a data set matching the IO algorithm capabilities will write data. If the data does not match the capabilities of the IO algorithm the creation will fail. Images are loaded and saved using the compute() method. Before calling compute the IO location has to be set using the setLocation() method. If the algorithm is a reader the loaded data will be available from the takeOutput() method. To find out whether the IO operation succeeded call status(). The supported file extensions (in case of a file-based IO algorithm) can be queried from the supportedFileExtensions() method. When implementing your algorithm you should not limit the locations to the ones with the correct extension. The extensions are only intended as help when displaying the file IO dialog.

◆ userOptions()

std::unique_ptr< Properties > ImFusion::IoAlgorithm::userOptions ( bool includeReadOptions,
bool includeWriteOptions ) const

Convenience method to retrieve all properties from this IoAlgorithm that are tagged as user options to be configured during load/save.

Retrieves the configuration of this algorithms and filters only those parameters that have the 'useroption' attribute set according to includeReadOptions and includeWriteOptions. If such parameters are present (i.e. set in configure()) they will be shown during the load/save dialog of the default IoController, as well as in the global settings dialog.
