Compute projection matrix for source-detector geometry p1, p2, p3 and p4 are the detector corner points in the order top-left, top-right, bottom-right and bottom-left as seen from the source.

I.e. the outer border of the image. The points are assumed to lie on a plane. C is the position of the source. w and h is the detector width and height in pixels. K, R and t contain the computed projection matrix components.

Computes the camera pose (transformation from world to camera) from 3D-2D point correspondences, camera intrinsics and distortion coefficients.

This overload of the function provides all camera pose candidates found, together with the RMSE reprojection errors associated with each of the poses. The case for using this function is the ambiguity of the camera-to-target pose for nearly planar targets. It is not always possible to select the "true" pose of the camera based on reprojection only. Having all poses at hand, the user can use external information about the visual system they consider to make the choice.

Parameters
X3D points in world coordinates.
x2D points in image coordinates.
KIntrinsic camera calibration matrix.
distCoeffsCamera distortion coefficients.
reprojectedPointsOptional output parameter containing reprojected 2D points for each pose.
Returns
A vector of tuples, where each tuple contains a root-mean-square reprojection error and a 4x4 camera pose matrix. The result is sorted in non-descending order by the value of the reprojection error.

Convert a projection matrix to OpenGL representation from OpenCV convention In OpenCV convention the y-axis is flipped compared to OpenGL, and pixel coordinates are used.

Note
We use OpenCV convention for K, R, t. I.e. this function assumes that (0,0) is the pixel center of the top-left corner pixel of the image. Consequently the corner of the image is at (-0.5,-0.5).
