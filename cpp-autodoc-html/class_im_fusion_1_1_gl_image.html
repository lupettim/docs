Detailed Description

Image representation stored as OpenGL texture.

This class is a wrapper around a low-level GL::Texture that additionally inherits from Image to hold additional metadata in the ImageDescriptor and make it compatible with SharedImage.

Furthermore, it provides a couple of member functions such as conversion from/to MemImages. However, for the majority of use cases you will likely not need to call any of them directly:

  • When doing low-level OpenGL processing the GL::Texture class and its API is usually sufficient and more explicit.
  • When doing high-level image processing you will use the SharedImage[Set] API to implement automatic conversion from/to MemImages or other representations. When receiving a GlImage instance from you will rarely need to do more with it than passing it to a GL::Program or assigning it to a different SharedImage[Set].

By default, unless explicitly selected during construction, GlImage will select the internal OpenGL texture format automatically based on the provided input PixelType:

  • Byte and UByte -> R[GBA]8
  • Short and UShort -> R[GBA]16
  • HFloat -> R[GBA]16F
  • Int, UInt, Float, Double -> R[GBA]32F

Platform-specific caveats when using OpenGL ES:

  • 3-channel images use the corresponding 4-channel internal format (RGB* -> RGBA*) because 3-channel formats are not color-renderable. The padded values are initialized to normalized 1.
  • 16-bit integer textures require an extension and use use R[GBA]16_EXT.
Warning
32-bit integer data in normalized texture formats and double-precision input may lose precision, because not all values are exactly representable as 32-bit float. This limitation is due to OpenGL not supporting such types as normalized texture format.

GlImage performs the following value domain conversions during upload/download: For non-integer (normalized) texture formats, signed integer uploads (Byte, Short, Int) are converted to the corresponding unsigned type by applying a half-range offset ([-2^(N-1) .. 2^(N-1)-1] <-> [0 .. 2^N-1]). This maps to unsigned normalized GPU range [0..1] without collapsing distinct codes. The inverse transform is applied on download. You can use GlUtils::storageToNormalized() and GlUtils::normalizedToStorage() to perform the same rescaling on the CPU for any input value.

Note
Be advised that OpenGL supports a maximum of 4 channels per element. If you try to create a GlImage with more than 4 channels, any additional channel will be discarded and the resulting GlImage will have 4 channels (i.e. you lose data). This will also happen implicitly when synchronizing through a SharedImage.

ImFusion::GlImage::GlImage ( const GlImage & image)
explicit

Creates a deep copy of the GlImage, with a whole new texture containing the same image data as the original.

Note that the copy constructor is explicit, so you cannot call it implicitly. For example, GlImage original{...}; GlImage copy1 = original // does not compile GlImage copy2 = GlImage{original}; // compiles GlImage copy3{original}; // compiles
