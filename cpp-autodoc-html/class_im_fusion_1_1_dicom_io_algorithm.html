Detailed Description

IoAlgorithm for reading and writing DICOM files.

The specified location can be either a file/folder path or a URI of the following syntax:

  • Local file system: file://path/to/file/or/folder
  • PACS instance: pacs://<Hostname>:<Port>/<HostAE>?<seriesUID>
  • DicomWeb instance: https://<Hostname>:<Port>/<dicomweb-endpoint>/studies/<studyUID>/series/<seriesUID>

Depending on the mode passed during construction, this algorithm operate in the following fashion:

  • IoMode::Read: The algorithm will use the DicomLoader class to read DICOM data from the specified location. Multiple input locations are supported by separating each item by '##'. Furthermore, an optional filter query ?<seriesUID> can be appended in order to only load a specific series from a folder. Thus, a valid input location has the following format: <path> [?<seriesUID>] [##<path> [?<seriesUID>] ]* Alternatively, the query can also be a key-value pair with the following format: <path> [?series=<seriesUID>[&key=<value>]* ] [##<path> [?series=<seriesUID>[&key=<value>]* ] ]* Only the keys 'series', 'study', 'ignoreNonDicom' and 'recursive' are recognised. All other keys are ignored. The optional 'ignoreNonDicom' key specifies whether files without a DICOM header are ignored (defaults to true). The optional 'recursive' key determines if a <path> to a folder should be loaded recursively.
  • IoMode::Write: The algorithm will use the function defined in DicomExport.h to store the SharedImageSet passed during construction to the specified location. Depending on the image type it can be stored as set of DICOM frame or (if supported) as a single Enhanced DICOM file.

This class automatically uses the DicomPlugin::defaultAuthorizationProvider for the internal Dicom::DatasetCache. The provider can be changed (or removed).
