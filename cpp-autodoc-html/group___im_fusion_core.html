Detailed Description

Essential support library.

The ImFusionCore library provides generic core functionality that is not specific to any business logic.

  • Assertion Library provides a powerful and flexible alternative to the basic assert() of the C standard library.
  • Logging offers a lightweight logging framework used by the ImFusion SDK.
  • Resource System enables you to retrieve binary data at runtime that has been embedded into the application during compile time.
  • Object Serialization offers means to generically transform object state to strings that can be saved and restored from on disk.
  • Signals and Slots offers a simple yet powerful interface for loosely coupled communication between different objects similar to the Observer design pattern.
  • Filesystem Library provides an easy-to-use abstraction layer for accessing the local filesystem so that you do not need to fall back on OS-specific APIs or third-party libraries.
  • Geometry Library contains various geometric objects and functions to intersect them.
  • Furthermore, several namespaces group classes and functions to complement the standard library with extra functionality and/or help with everyday tasks:
    • The Container namespace provides utility functions for (STL) container access and their manipulation.
    • The Threading namespace contains a couple of classes for advanced multi-threading.
    • The Platform namespace offers a selection of platform-specific functions.
    • The String namespace contains utility functions for working with std::string.
    • The Compression namespace contains utility functions to compress arbitrary data.
    • The Cryptography namespace contains utility functions for data encryption and signing.
    • The Hashing namespace contains utility functions for non-cryptographic hashing of data, for instance to use them in std::unordered_map.
    • Finally, there are the Generic utility classes and helper functions that do not fit into any of the previous categories.
