Detailed Description

Helper class to maintain a map of mouse/keyboard combinations to an enumeration of actions.

This interface is supposed to be used by input event handlers. They often define a set of interaction metaphors and/or shortcuts that enable the user to conduct particular actions (e.g. move the camera). InputEventMapper allows for specifying mappings in the form of keyboard modifiers together with either a mouse or keyboard button press, and tie them to an action represented by an enumeration value. During event handling the match member functions can be called to conveniently check if an incoming InputEvent matches one of the defined actions.

enum class Action {
Zoom,
Pan,
Reset,
...
};
InputEventMapper<Action> iem;
// during setup:
iem.addMapping(Action::Zoom, MouseEvent::Button::Left, InputEvent::KeyboardModifier::None);
iem.addMapping(Action::Pan, MouseEvent::Button::Right, InputEvent::KeyboardModifier::None);
iem.addMapping(Action::Reset, KeyEvent::Key::R, InputEvent::KeyboardModifier::Alt);
// during eventHandling:
EventResult handleInputEvent(const InputEvent& event) {
...
// example usage of matchAny()
if (std::optional<Action> a = iem.matchAny(event)) {
switch (*a) {
case Action::Zoom:
...
break;
case Action::Pan:
...
break;
case Action::Reset:
...
break;
}
}
// example usage of matches()
if (iem.matches(event, Action::Reset) {
...
}
}

By default, InputEventMapper will not allow for duplicate mappings (i.e. that the same input event combination maps to two different actions). You can optionally enable support for this if your use case requires it and you can resolve the potential ambiguity on the caller site. Be advised that in this case you should use matchAny() with caution because it is unspecified which action it returns in case of duplicates.

This class implements the Configurable interface so that you can easily serialize and restore an existing mapping.

Note
This base class uses a plain int32_t as type for the action enumeration. Use the derived class InputEventMapper<T> for a type-safe version accepting only the specified enum type.
