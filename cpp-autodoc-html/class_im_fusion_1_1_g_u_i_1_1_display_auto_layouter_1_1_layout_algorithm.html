virtual DisplayLayoutConfig ImFusion::GUI::DisplayAutoLayouter::LayoutAlgorithm::createLayout ( const LayoutInfo & layoutInfo) const
pure virtual

Main implementation of the layouting algorithm.

This function receives the LayoutInfo record describing the current configuration of the DisplayAutoLayouter and is supposed to return a corresponding DisplayLayoutConfig. A standard layout algorithm will only place views contained in layoutInfo.visibleViews in the resulting configuration. If it chooses to omit views this should be indicated in the return value of flags() if possible.

A layouting algorithm is also allowed to include extra views in the final layout config. This way a controller could for instance enforce a specific fixed layout for a specific workflow. However, this functionality should be used with care and only for dedicated use cases because the user experience may degrade otherwise (e.g. user cannot hide certain views anymore).
