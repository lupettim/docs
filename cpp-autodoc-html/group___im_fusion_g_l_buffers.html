Detailed Description

Interfaces and wrapper classes for working with OpenGL buffers.

OpenGL buffers are used to transfer larger chunks of data between the host and the GPU. All buffer types inherit from the GL::Buffer base interface. The API for uploading/downloading of buffer data is using std::vector as storage backend on the host side. The individual buffer types are used for different applications:

  • The GL::VertexBuffer class stores per-vertex data used for rendering graphical primitives onto the screen. It can be combined with an GL::IndexBuffer to reduce memory consumption if multiple vertices share the same vertex data.
  • The GL::AtomicCounterBuffer is a specialization to represent OpenGL Atomic Counter Objects enabling atomic counters in GLSL shaders. Since their functionality is very limited it is recommended to use them only if Shader Storage Buffer Objects (SSBOs) are not available.
  • The GL::ShaderStorageBuffer represents a Shader Storage Buffer Object (SSBO), the most generic and powerful buffer type available. They allow for generic uploading/downloading any kind of data between host and GPU.
