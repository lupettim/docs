# `imfusion.computed_tomography`

ImFusion Computed Tomography (CT) Python Bindings
=================================================

Core Functionality Areas
------------------------

**Geometry:**

- Cone beam geometry representation and manipulation
- Parametric geometry generators for scanner configurations
- Geometry calibration and validation tools

**Simulation:**

- Forward projection and cone beam simulation
- GPU-accelerated projection operators
- Synthetic data generation from volumes and meshes
- Scanner-specific simulation presets

**2D/3D Registration:**

- X-ray to volume registration algorithms
- Multiple initialization strategies (manual, keypoint, point-direction)
- GPU-accelerated registration optimization

**Reconstruction:**

- Analytical reconstruction (FDK)
- Iterative reconstruction (MLEM, SART, SQS, CG)
- Advanced regularization techniques

Example Usage
-------------

**Basic cone beam geometry setup:**

&gt;&gt;&gt; import imfusion.computed_tomography as ct
&gt;&gt;&gt; ct.make_cone_beam_data(projections)
&gt;&gt;&gt; metadata = ct.ConeBeamMetadata.get(projections)
&gt;&gt;&gt; metadata.enable_modern_geometry()
&gt;&gt;&gt; param_gen = ct.ParametricGeometryGenerator()
&gt;&gt;&gt; param_gen.source_det_distance = 1200.0
&gt;&gt;&gt; param_gen.source_pat_distance = 800.0
&gt;&gt;&gt; param_gen.angle_range = 180.0
&gt;&gt;&gt; metadata.add_generator(param_gen, select=True)

**Forward projection simulation:**

&gt;&gt;&gt; projections = ct.simulate_cone_beam_projections(
...     volume,
...     geometry_preset=ct.GeometryPreset.FULL_SCAN,
...     proj_type=ct.ProjectionType.LOG_CONVERTED_ATTENUATION,
...     width=1024, height=1024, frames=360,
...     add_poisson_noise=False
... )

**2D/3D registration workflow:**

&gt;&gt;&gt; ct.register_2d_3d_xray(
...     projections, volume,
...     initialization_mode=ct.InitializationMode.KEYPOINTS,
...     num_resolution_levels=4,
...     anatomy_name="spine"
... )

**Reconstruction with iterative solver:**

&gt;&gt;&gt; volume = ct.reconstruct_cone_beam_ct(
...     projections,
...     solver_mode="MLEM",
...     max_iterations=50,
...     subset_size=10,
...     force_positivity=True
... )

For detailed documentation of specific classes and functions, use Python's
built-in help() function or access the docstrings directly.

Note: This module requires the ImFusion CT plugin to be properly installed
and licensed.

### `ConeBeamGeometry`

```python
class ConeBeamGeometry(imfusion.Configurable)
```

Legacy cone beam geometry representation.

This class provides the traditional parametric representation of cone beam
CT geometry using intrinsic and extrinsic parameters. It defines the 
geometric relationship between X-ray source, detector, and reconstruction
volume for cone beam acquisition systems.

Key Features:

- Source-detector distance and source-patient distance
- Detector size, offset, rotation, and shear parameters
- Reconstruction volume size and positioning
- Support for circular
- Matrix serialization in multiple formats

Example:
        Create and configure a cone beam geometry:
        
        &gt;&gt;&gt; geometry = ct.ConeBeamGeometry()
        &gt;&gt;&gt; geometry.source_det_distance = 1200.0  # mm
        &gt;&gt;&gt; geometry.source_pat_distance = 600.0   # mm
        &gt;&gt;&gt; geometry.det_size_x = 400.0            # mm
        &gt;&gt;&gt; geometry.det_size_y = 300.0            # mm
        &gt;&gt;&gt; geometry.recon_size = 256.0            # mm

High-level 2D/3D X-ray registration algorithm for cone beam projections.

This algorithm performs registration between 2D X-ray projections and a 3D volume

high-level interface.

        &gt;&gt;&gt; projections, *_ = imf.load("projections.dcm")

        ...     initialization_mode=ct.InitializationMode.POINT_DIRECTION,
